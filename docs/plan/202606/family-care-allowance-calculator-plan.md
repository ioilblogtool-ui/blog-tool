# 가족돌봄수당(조부모 돌봄수당) 계산기 — 기획 문서

## 1. 배경

- "가족돌봄수당 계산기" 제안 — bigyocalc의 "계산기 + 복지 + 육아 + 조건판정" 정체성과 잘 맞는 콘텐츠.
- 기존 코드베이스에 `welfareBenefitEligibility.ts`(4대 급여), `livelihoodBenefitIncomeRecognition.ts`, `housingBenefitIncomeRecognition.ts`, 그리고 직전에 만든 `basicPensionEligibilityCalculator.ts`까지 **"조건 입력 → 자가 점검 판정"** 패턴이 이미 4건 쌓여 있음. 가족돌봄수당도 같은 패턴으로 자연스럽게 확장 가능.
- 검색 수요 확인: "조부모 돌봄수당", "손주 돌봄수당", "가족돌봄수당" 관련 콘텐츠가 다수 블로그에 도배되어 있을 정도로 수요가 큼 — 다만 경쟁이 많은 만큼 **계산기(자가진단)로 차별화**할 여지가 있음.

## 2. 사실 검증 결과 (2026년 기준, 다수 출처 교차검증)

### 대상·기본 조건
- 대상 연령: **만 24개월~36개월 이하** (일부 지자체는 47개월까지 확대 운영 — 지역별 차이 큼)
- 우선 대상: 맞벌이·한부모·다문화·다자녀·장애 부모 가정
- 돌봄 제공자 범위: 조부모뿐 아니라 **4촌 이내 친인척**(이모, 삼촌 등)도 가능한 지자체 있음
- 필수 조건: **월 40시간 이상 돌봄** (하루 최대 4시간, 오전 6시~오후 10시 사이만 인정), 돌봄일지 작성 필수, 조부모 대상 사전 교육(돌봄 안전·아동발달) 이수 필요

### 소득 기준
- **기준 중위소득 150% 이하** (맞벌이 가구는 완화 적용되는 지자체 있음)
- 2026년 중위소득 150% 건강보험료 본인부담금 판정기준표(직장가입자 기준, 장기요양보험료 제외) — 다수 출처 교차검증으로 신뢰도 높음:

| 가구원수 | 기준액(원) |
|---|---|
| 1인 | 138,780 |
| 2인 | 229,357 |
| 3인 | 290,169 |
| 4인 | 360,410 |
| 5인 | 약 41만원대 (⚠️ 정확한 값 설계 단계 재확인) |
| 6인 | 약 49만원대 (⚠️ 정확한 값 설계 단계 재확인) |

> 기준일은 보통 해당 연도 특정 시점(예: 3월 말)의 건강보험료 납부 기록을 사용하므로, 신청일 당시 소득이 아니라 **기준일 시점 소득**으로 판정된다는 점이 사용자들이 자주 놓치는 부분 — FAQ에서 반드시 짚어야 함.

### 지원금액
- 손주(자녀) 1명: 월 **30만원**
- 손주(자녀) 2명: 월 **45만원**
- 손주(자녀) 3명 이상: 월 **최대 60만원**
- 일부 자치구(은평·도봉·성북 등)는 자체 예산으로 **+10~15만원 추가 지원**

### 신청 절차
- 매월 **1일~15일 신청** → 해당 월부터 지급, **16일 이후 신청** → 다음 달부터 지급
- 방문: 주민등록상 주소지 읍·면·동 주민센터 / 온라인: 서울 "몽땅정보 만능키", 경기 "경기 민원24"

## 3. ⚠️ 가장 중요한 리스크 — 지역별 운영 차이

이 제도는 **전국 통일 제도가 아니라 지자체별 자체 사업**이다. 서울시("가족돌봄수당"), 경기도("아이돌보미 가족돌봄"), 기타 지자체마다:
- 사업 참여 여부 자체가 다름 (참여 안 하는 시군구 존재)
- 지원금액·대상 나이·소득기준이 미세하게 다를 수 있음
- 같은 도(경기도) 안에서도 시군별 참여 여부가 다름 (예시에 나온 고양시·파주시처럼 직접 확인 필요)

**따라서 이 계산기는 "전국 공통 기준으로 100% 확정 판정"을 할 수 없다.** 설계 시 다음 원칙을 반드시 반영:
1. 나이·소득·돌봄시간 조건은 **일반적인 기준으로 1차 판정**
2. 거주 지역(시군구) 조건은 **"참여 여부 확인 필요"**로 별도 분리 — 확정 가능/불가능과 혼동되지 않게
3. 최종 판정 문구는 항상 "가능성이 있음 / 확인 필요"로, "100% 가능"이라는 확정적 표현 금지

## 4. 기존 자산 재사용 방향

| 재사용 대상 | 위치 | 활용 방식 |
|---|---|---|
| 조건 입력 → 자가점검 판정 UI 패턴 | `livelihood-benefit-income-recognition.astro`/`.js`, `basic-pension-eligibility-calculator.astro`/`.js` | SimpleToolShell + 프리셋 + 결과 카드 + 경고 카드 구조 그대로 재사용 |
| 가구원수별 건강보험료 기준표 개념 | `welfareBenefitEligibility.ts`의 임계값 테이블 구조 | 동일한 `Record<householdSize, threshold>` 패턴으로 신규 상수 정의 (절대 값 자체는 혼용 금지, 가족돌봄수당 전용 상수로 분리) |

## 5. 입력값 / 출력값

### 입력값
- 거주 지역(시/도) — 서울/경기/기타
- 시군구 (선택, 텍스트 또는 셀렉트 — 참여 확인 안내용)
- 아이 생년월
- 신청 예정 연월
- 자녀(돌봄대상) 수
- 가구원 수
- 건강보험료(가구 합산, 장기요양보험료 제외)
- 양육공백 사유 (맞벌이/한부모/다자녀/장애부모/없음)
- 월 예상 돌봄시간

### 출력값
- 대상 연령 충족 여부 (24~36개월 진입·종료 시점 계산)
- 소득 기준 충족 여부
- 돌봄시간 기준 충족 여부 (40시간 이상)
- 거주지역 참여 여부 — **"확인 필요"로 고정 표시**
- 종합 판정: 가능성 높음 / 일부 조건 미충족 / 확인 필요
- 예상 월 지원금 및 최대 총 지원금 (자녀 수 기준)

## 9. 리스크

- **지역별 운영 차이로 인한 오판정 리스크(최우선)**: 위 3번 섹션 원칙 준수, "확정 가능"이라는 표현 금지.
- **건강보험료 기준표 5인/6인 값 미확정**: 설계 단계에서 정확한 수치 재검증 필요.
- **최대 지원개월 수 불확실**: 사용자 초안의 "13개월"은 검증되지 않은 추정 — 설계 단계에서 실제 지자체 공고 기준으로 재확인하거나, 확정 못하면 "지자체 공고 확인 필요"로 처리.
- **소득기준 판정 시점 오해**: 신청일이 아니라 특정 기준일(예: 3월 말) 건강보험료라는 점을 FAQ에서 명확히 해야 함 — 그렇지 않으면 사용자가 잘못된 입력값으로 오판정할 위험.
- **돌봄시간 조건의 복잡성**: "하루 최대 4시간, 06~22시만 인정"이라는 세부 규정까지는 1차 버전에서 단순화(월 40시간 이상 여부만 판정)하고, 세부 규정은 안내 문구로만 고지.
