## 2. 왜 이 콘텐츠가 좋은가

### 2-1. TOP10보다 강한 이유

성과급 TOP10 순위표는 검색 유입용으로는 쓸 수 있지만 차별성이 약하다. 반면 이 계산기는 사용자가 자기 회사를 넣고 결과를 확인한다.

| 비교 | TOP10 순위표 | 성과급 체력 점수 계산기 |
|---|---|---|
| 검색 유입 | 있음 | 있음 |
| 체류 시간 | 짧음 | 김 |
| 공유성 | 보통 | 높음 |
| 재방문 | 낮음 | 실적 시즌마다 가능 |
| 고유성 | 약함 | 강함 |
| 내부 링크 | 제한적 | 성과급·연봉·기업 비교 콘텐츠로 확장 가능 |

### 2-2. bigyocalc와 맞는 이유

비교계산소의 강점은 단순 정보가 아니라 “내 상황을 넣어 계산하는 콘텐츠”다. 이 계산기는 회사 실적이라는 공개 데이터를 사용자가 자기 상황과 연결하게 만든다.

- 직장인은 자기 회사 성과급 가능성을 보고 싶어 한다.
- 취업 준비생은 돈 잘 버는 회사와 성과급 체력이 좋은 회사를 구분하고 싶어 한다.
- 투자자는 매출보다 이익률, 직원 1인당 이익, 시총 대비 영업이익을 보고 싶어 한다.
- 기존 삼성전자·SK하이닉스·현대차·기아 성과급 콘텐츠와 내부 링크가 자연스럽다.

## 10. 예상 성과급률 산식

성과급 체력 점수는 “가능성”이고, 예상 성과급률은 별도 추정치로 표시한다.

```text
예상 성과급률 = 전년도 성과급률 × 실적 개선 계수 × 업황 계수
```

### 10-1. 실적 개선 계수

| 영업이익 증가율 | 계수 |
|---:|---:|
| 300% 이상 | 1.8 |
| 100~300% | 1.5 |

## 14. FAQ

### Q1. 성과급 체력 점수는 실제 성과급 지급률과 같은가요?

아닙니다. 성과급 체력 점수는 회사가 성과급을 줄 여력이 있는지 보는 추정 지표입니다. 실제 성과급은 회사의 임금 정책, 노사 협상, 사업부별 기준, 개인 평가에 따라 달라집니다.

### Q2. 매출이 큰 회사가 성과급도 많이 주나요?

반드시 그렇지는 않습니다. 성과급은 매출보다 영업이익, 영업이익률, 현금흐름, 전년도 지급 이력의 영향을 더 크게 받습니다. 매출이 커도 이익률이 낮으면 성과급 체력은 낮을 수 있습니다.

### Q3. 직원 1인당 영업이익은 왜 중요한가요?

직원 1인당 영업이익은 회사가 직원 한 명당 얼마의 이익을 만들었는지 보여주는 지표입니다. 이 수치가 높을수록 성과급이나 임금 인상 여력이 높게 해석될 수 있습니다.

### Q4. 삼성전자와 SK하이닉스는 같은 방식으로 계산해도 되나요?

큰 틀에서는 가능하지만 사업부 구조가 다릅니다. 삼성전자는 반도체 외에도 모바일, 가전, 디스플레이 등 사업부가 나뉘고, SK하이닉스는 메모리 반도체 집중도가 높습니다. 따라서 회사 전체 실적과 사업부 성과급은 다를 수 있습니다.

### Q5. 적자 회사도 성과급이 나올 수 있나요?

가능은 합니다. 다만 적자 회사의 성과급은 실적 기반 성과급보다는 격려금, 특별 보상, 조직 유지 목적의 지급일 가능성이 큽니다. 계산기에서는 적자 회사의 성과급 기대치를 낮게 반영합니다.

### Q6. 프리셋에 없는 회사도 계산할 수 있나요?

직접 입력 모드에서 매출, 영업이익, 전년 영업이익, 직원 수, 전년도 성과급률을 입력하면 계산할 수 있습니다.

## 18-1. 검토 의견 — 반영 필요 사항 (2026-06-23 검토)

### 18-1-1. [중요] 업종 절대 기준 점수의 구조적 왜곡

영업이익률(25점)과 직원 1인당 영업이익(20점)을 업종 구분 없이 절대 금액 기준(예: 직원 1인당 10억 이상=20점)으로 매기면, 점수가 "실적이 개선됐는가"가 아니라 "이 업종이 원래 이익률이 높은가"를 측정하게 된다.

- 금융·플랫폼업은 인당 이익이 구조적으로 높아 항상 고득점 구간에 들어간다.
- 조선·방산·자동차처럼 인력집약적 업종은 실적이 아무리 개선돼도 절대 기준에서는 항상 낮은 점수대에 머문다.

**대응**: 영업이익률·직원 1인당 영업이익 점수는 "업종 평균 대비 상대값"으로 환산한다.

```text
업종 상대 이익률 = 회사 영업이익률 / 업종 평균 영업이익률
업종 상대 인당이익 = 회사 직원1인당영업이익 / 업종 평균 직원1인당영업이익
```

상대값(예: 1.0 = 업종 평균, 1.5 = 업종 평균보다 50% 높음)을 기준으로 점수 구간을 다시 설계한다. 이렇게 하면 조선업 안에서 1등 회사와 금융업 안에서 1등 회사가 같은 점수대에 위치할 수 있어, "내 회사가 같은 업종 안에서 잘하고 있는가"라는 더 정확한 질문에 답하게 된다. 영업이익 증가율(30점)과 업황 전망(10점)은 이미 회사 자체의 변화율이라 절대 기준이어도 문제가 없다.

`BonusIndustryDefault`에 업종 평균 영업이익률, 업종 평균 직원 1인당 영업이익 필드를 추가해야 한다(섹션 11 업종별 기본값 표 확장 필요).

### 18-1-2. 전년도 성과급률의 "기준" 모호성

실제 보도되는 성과급(삼성 OPI·PS, SK PS·PI 등)은 회사마다 "기본급 기준", "월급(기본급+제수당) 기준", "연봉 기준" 중 다른 방식을 쓴다. 프리셋에 `previousBonusRate`를 단순 %로만 넣으면, 어떤 회사는 기본급 기준 50%고 어떤 회사는 월급 기준 50%라 실제 금액 차이가 크게 난다.

**대응**: `CompanyBonusPreset`에 `bonusRateBasis: "base" | "monthly" | "annual"` 필드를 추가하고, 계산 시 사용자가 설정한 `기본급 비중`과 함께 기준을 환산한다. 최소한 프리셋 데이터 입력 시 출처 보도의 기준을 반드시 확인해 일관되게 변환한다.

### 18-1-3. 실명 회사 저점수 노출의 평판 리스크

이 계산기는 실명 대기업을 "낮음(0~19점)"처럼 보이는 점수로 표시한다. 회사 실적이 일시적으로 안 좋을 때 검색 결과에 "OO전자 성과급 체력 낮음"처럼 노출되면 명예훼손성 민원이나 어그로성 캡처 확산 리스크가 있다.

**대응**:
- 기본 노출 회사(첫 화면 기본값)는 실적이 양호한 회사로 설정해 "성과급 안 나오는 회사"가 첫 인상이 되지 않게 한다(현재 기획대로 삼성전자 기본값 유지 — 다만 발행 시점 실적이 부진하면 다른 회사로 교체 검토).
- 점수 라벨에 회사명을 직접 결합한 문구(`"OOO 성과급 체력 낮음"`)를 자동 생성하지 않는다. 점수와 회사명은 같은 화면에 있지만, 문장으로 단정 결합하지 않는다.
- "낮음" 라벨도 "성과급 기대가 낮은 구간" 정도로 유지하고, "이 회사는 성과급을 못 준다"처럼 들리는 표현은 쓰지 않는다(섹션 8-2 문구는 이미 양호하나 구현 시 그대로 유지할 것).

### 18-1-4. 데이터 갱신 주기 미정의

기획 의도(섹션 2-1)에는 "실적 시즌마다 재방문 가능"이 강점으로 적혀 있는데, 실제 갱신 주기·트리거가 정의되어 있지 않다.

**대응**: 운영 규칙으로 "분기 실적 발표 후 2주 내 프리셋 40개 영업이익·직원수 갱신"을 명시하고, 갱신일을 `dataAsOf` 필드로 페이지에 노출한다. 갱신을 안 하면 "재방문성"이라는 핵심 차별점이 무의미해진다.

### 18-1-5. 직접 입력 모드 0값 처리 명확화 필요

섹션 7-3에 "계산 제한"이라고만 적혀 있어 실제 동작(계산을 막는지, 0점으로 표시하는지)이 불명확하다. 설계 문서에서 매출 또는 직원 수가 0이면 결과 영역 대신 "입력값을 확인해주세요" 안내만 표시하고 점수 게이지는 렌더링하지 않는 것으로 명확히 한다.

## 18. 최종 추천

이 콘텐츠는 바로 구현할 가치가 높다.

이유:

1. 기존 성과급 계산기와 자연스럽게 연결된다.
2. 검색 키워드가 넓다.
3. 직접 입력 기능 덕분에 회사 풀이 작아도 사용성이 높다.
4. 직장인 공유성이 좋다.
5. “성과급 TOP10”보다 고유 기능이 강해 애드센스 품질에도 유리하다.

추천 출시 형태는 **40개 회사 프리셋 + 직접 입력 모드 + 성과급 체력 점수 + 예상 성과급 세후 금액**이다.

초기에는 데이터 완벽성보다 계산 구조와 사용자 경험이 중요하다. 회사별 수치는 `추정`, `공개자료 기준`, `컨센서스 기준`처럼 배지를 붙이고, 직접 입력 모드로 사용자가 자기 회사 상황을 보정할 수 있게 만드는 것이 가장 안전하다.
