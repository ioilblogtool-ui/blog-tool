# 내 연봉으로 서울 어디 살 수 있나 — 지도형 PIR 리포트 2026 기획 문서

## 1. 배경

- "서울 집값 무섭다", "전월세" 검색 흐름에서 사용자가 실제로 원하는 정보는 "시장이 어떻게 변했나"가 아니라 "나는 지금 서울 어디를 살 수 있나"라는 **본인 기준 의사결정**임.
- 기존 `seoul-housing-2016-vs-2026`(10년 비교), `seoul-apartment-jeonse-report`(전세 소멸), `seoul-jeonwolse-ratio-2026`(전월세 전환 지도)는 모두 "시장 설명형"이라 후속편은 "개인화 계산형"으로 차별화.
- `seoul-housing-2016-vs-2026`이 이미 "내 연봉 기준 체감"을 계산기로 일부 제공하지만 서울 평균 1개 값 기준이라, 이번 리포트는 **25개 구 단위로 쪼개서** 지도 인터랙션으로 보여주는 후속편 역할.

## 2. 기존 자산 재사용

| 재사용 대상 | 위치 | 활용 방식 |
|---|---|---|
| 서울 25개 구 SVG 좌표(`mapShapes`) | `src/pages/reports/local-election-seoul-2026.astro` | 동일 좌표를 새 데이터 파일에 복제해 지도 그리기 (선거 데이터와 결합 X, 좌표만 재사용) |
| 지도 클릭/검색/툴팁 인터랙션 패턴 | `public/scripts/local-election-seoul-2026.js` | 동일 패턴(클릭 시 패널 갱신, 검색, 해시 라우팅)을 재사용하되 정당색 대신 PIR 등급색으로 교체 |
| 구별 평균가·전세가율 데이터 | `src/data/seoulApartmentPrice2026.ts`(`seoulApartmentDistricts`) | 25개 구 매매 평균가, 84㎡ 평균가, 전세가율을 그대로 가져와 입력값과 결합해 등급 계산 |
| 서울 평균 PIR·연봉 체감 계산 로직 | `src/data/seoulHousing2016Vs2026.ts`의 `calcHousingFeelings` | 참고 로직, 이번 리포트는 구별로 확장한 별도 함수 작성 |

## 3. 컨셉

- 입력: **연소득(만원), 보유 현금(만원)**, 선택적으로 **대출 가정(LTV 60/70/80%)**.
- 출력: 서울 25개 구 지도를 입력값 기준 4단계 등급(매매 가능 / 대출 부담 큼 / 전세만 가능 / 진입 어려움)으로 실시간 재색칠.
- 매매와 전세를 별도 탭/토글로 전환 가능하게 해서 "전월세" 검색 의도도 같은 페이지에서 흡수.
- 정치색 지도가 아니라 "내 상황 기준 색"이라는 점이 핵심 차별화 — 사용자가 입력을 바꿀 때마다 지도가 즉시 변하는 게 체류시간을 끄는 핵심 인터랙션.

## 4. 타겟 검색 의도

| 키워드 그룹 | 예시 |
|---|---|
| 연봉 기준 매매 가능 지역 | 내 연봉으로 서울 어디 살 수 있나, 연봉 5천 서울 아파트, 연봉 대비 집값 |
| PIR | 서울 PIR 지도, 구별 PIR, 서울 집값 연봉 배수 |
| 전월세 의사결정 | 전세 매매 비교, 전세 vs 매매 어디가 유리, 서울 전세 가능한 구 |
| 예산대별 지역 추천 | 9억으로 살 수 있는 서울 아파트, 6억 이하 서울 아파트 |

## 5. 입력값 / 출력값

### 입력값
- 연소득(만원) — 슬라이더 + 숫자 입력
- 보유 현금/자산(만원) — 슬라이더 + 숫자 입력
- 대출 가정(LTV) — 60% / 70% / 80% 선택 (기본 70%)
- 매매 vs 전세 토글

### 출력값
- 25개 구 SVG 지도 (등급별 색상)
- 선택 구 상세 패널: 평균가, 84㎡가, 전세가율, 등급, 부족/여유 금액
- 등급별 구 목록 요약 카드 (매매 가능 N개 구 / 대출 부담 N개 구 / 전세만 가능 N개 구 / 진입 어려움 N개 구)
- 예산대별 추천 구 리스트 (기존 `seoulApartmentBudgetBands` 데이터 활용)

## 6. 데이터 요구사항

### Fact
- 구별 평균가, 84㎡ 평균가, 전세가율 — `seoulApartmentPrice2026.ts` 기존 데이터 재사용 (표본·추정 리포트 기준임을 동일하게 고지)

### 계산 로직 (Estimate 명시)
- 최대 매매 가능 예산 = 보유 현금 / (1 - LTV)
- 최대 전세 가능 예산 = 보유 현금 + (보유 현금 × 전세대출 가정 비율)
- 등급 분류: 매매 가능 예산이 구 평균가 이상이면 "매매 가능", 84㎡가 이상이지만 평균가 미달이면 "대출 부담 큼", 전세 예산은 충분하지만 매매 예산 부족하면 "전세만 가능", 둘 다 부족하면 "진입 어려움"
- 모든 계산은 단순화된 추정이며 실제 DSR, 신용도, 규제 지역 LTV 차등은 반영하지 않음을 명시 (기존 `home-purchase-fund` 계산기로 안내)

## 7. 리스크

- 단순 LTV 가정이 실제 대출 규제(스트레스 DSR, 규제지역별 LTV 차등)와 다를 수 있음 — InfoNotice와 FAQ에 명확히 고지
- 구별 평균가가 표본·추정 리포트 기준이라는 점을 기존 리포트와 동일하게 반복 고지
- 지도 좌표는 단순화된 SVG이며 실제 행정 경계와 다름 — local-election-seoul-2026과 동일 고지 문구 재사용
