## 2. 대메뉴 추가 배경

현재 bigyocalc.com의 상단 대메뉴는 `계산기`, `리포트` 중심이다. 하지만 실제 콘텐츠는 이미 아래처럼 “비교형” 성격이 강하다.

- 삼성전자·SK하이닉스·현대차 성과급 계산기
- 반도체·자동차·금융권·조선업 성과급 비교
- 신입 초봉·직군별 연봉 비교
- 청년미래적금·청년도약계좌·복지지원금 비교
- 전세 vs 월세, 신혼집 전세 vs 매매, 대출 갈아타기
- 월 적금 vs ETF, 배당 ETF, 수수료 비교

따라서 `비교표` 대메뉴는 새 콘텐츠를 억지로 만드는 메뉴가 아니라, 이미 있는 콘텐츠를 사용자의 의사결정 방식에 맞게 재분류하는 허브다.

## 3. 대메뉴 역할 정의

| 메뉴 | 사용자 의도 | 콘텐츠 성격 |
| --- | --- | --- |
| 계산기 | 내 값을 입력해서 결과를 보고 싶다 | 입력형 도구 |
| 리포트 | 시장·제도·트렌드를 해석하고 싶다 | 분석형 콘텐츠 |
| 비교표 | 여러 선택지를 한눈에 비교하고 싶다 | 표·카드·랭킹형 허브 |

### 비교표의 핵심 역할

1. 여러 계산기와 리포트를 같은 주제 안에서 묶는다.
2. 사용자가 “어디부터 봐야 할지” 빠르게 고르게 한다.
3. 검색 유입 페이지가 아니더라도 내부 회전율을 높인다.
4. 향후 `가이드`, `랭킹`, `데이터` 메뉴 확장의 기반이 된다.

## 4. 확정 IA

### 4-1. 상단 대메뉴 확정안

```text
계산기 | 리포트 | 비교표
```

초기에는 대메뉴를 3개까지만 확장한다. `가이드`, `랭킹`, `데이터`는 바로 대메뉴로 추가하지 않고 비교표 허브 안의 섹션명으로 먼저 실험한다.

### 4-2. 비교표 하위 구조 로드맵

| 경로 | 메뉴명 | 설명 | 1차 출시 여부 |
| --- | --- | --- | --- |
| `/compare/` | 비교표 홈 | 모든 비교표 카테고리 허브 | 1차 MVP |
| `/compare/bonus/` | 성과급 비교 | 대기업·업종별 성과급 비교 | 2차 |
| `/compare/salary/` | 연봉·초봉 비교 | 직군·기업·업종별 연봉 비교 | 권장 |
| `/compare/welfare/` | 지원금 비교 | 청년·출산·복지 지원제도 비교 | 3차 |
| `/compare/real-estate/` | 부동산·대출 비교 | 전월세·청약·대출·세금 비교 | 권장 |
| `/compare/investment/` | 투자·ETF 비교 | 적금·ETF·배당·수수료 비교 | 권장 |

### 4-3. 보류할 세부 카테고리

아래 경로는 콘텐츠가 충분히 쌓인 뒤 분리한다.

| 보류 경로 | 이유 |
| --- | --- |
| `/compare/parental-leave/` | 초기에는 `/compare/welfare/` 안에서 출산·육아 섹션으로 처리 |
| `/compare/etf/` | 초기에는 `/compare/investment/` 안에서 처리 |
| `/compare/loan/` | 초기에는 `/compare/real-estate/` 안에서 처리 |
| `/compare/tax/` | 세금 콘텐츠가 더 쌓이면 독립 |

## ④ 비교 기준 안내 섹션

비교표 페이지는 “무엇을 기준으로 비교했는지”가 중요하다. 각 카테고리에 공통 기준을 안내한다.

| 카테고리 | 비교 기준 |
| --- | --- |
| 성과급 | 연봉, 월급, 성과급률, 세후 추정, 지급 안정성 |
| 연봉 | 초봉, 평균연봉, 성과급, 복지, 성장성 |
| 지원금 | 가입 대상, 소득 기준, 지원금, 신청기간, 중복 가능성 |
| 부동산 | 초기자금, 월 부담, 세금, 대출금리, 거주기간 |
| 투자 | 수익률, 세금, 수수료, 변동성, 투자기간 |

### 고지 문구

```text
비교표는 공개자료와 계산 모델을 기준으로 정리한 참고용 콘텐츠입니다. 실제 조건은 회사 공지, 정부 공고, 금융회사 상품설명서, 계약 조건에 따라 달라질 수 있습니다.
```

## ⑤ 신규 비교표 제안 섹션

비교표 홈 하단에는 앞으로 만들 비교표를 “준비 중” 형태로 노출할 수 있다. 검색 수요 검증과 내부 기획 관리에 좋다.

| 준비 중 비교표 | 이유 |
| --- | --- |
| 대기업 복지포인트 비교 | 성과급 이후 복리후생 검색 확장 |
| 청년 주거지원 대출 비교 | 지원금·부동산 연결 |

| 신혼부부 대출·지원금 비교표 | 부동산·지원금 교차 수요 |

## 12. 결론

`비교표` 대메뉴는 bigyocalc.com의 다음 확장으로 가장 적합하다.

이 메뉴는 새 콘텐츠 카테고리를 억지로 만드는 것이 아니라, 이미 쌓인 계산기와 리포트를 “사용자가 비교하고 싶은 기준”으로 재배열하는 역할을 한다. 초기에는 `/compare/` 허브 하나로 시작하고, 성과급과 지원금처럼 수요가 검증된 영역부터 `/compare/bonus/`, `/compare/welfare/`로 확장하는 것이 가장 안전하다.
