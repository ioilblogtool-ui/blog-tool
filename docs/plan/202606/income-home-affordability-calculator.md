## 1. 기획 의도

`내집마련 자금 계산기(home-purchase-fund)`는 **"이 집을 사려면 얼마가 필요한가"** (정방향) 계산기다.
이번 신규 콘텐츠는 그 역방향인 **"내 소득과 현금으로 얼마짜리 집을 살 수 있나"** 를 답한다.

사용자가 가장 먼저 궁금해하는 3가지:

1. 내 연봉 기준으로 대출은 얼마까지 나올까? (DSR 한도)
2. 보유 현금 + 대출 한도로 살 수 있는 집값은 얼마인가?
3. 그 집값이 "내 연봉의 몇 배"인지 — 부담 수준은 적정한가?

기존 `home-purchase-fund`와 입력/출력이 반대 방향이라 **두 계산기를 서로 CTA로 연결**하면 체류시간과 내부 회유율을 동시에 높일 수 있다. (예: 이 계산기에서 적정 집값 도출 → "이 가격대 단지, 실제 필요 현금은 얼마?" → home-purchase-fund로 이동)

### 3-1. DSR 기준 (2026년 3월 기준, 참고용)

| 항목 | 기준값 | 비고 |
|---|---:|---|
| 기본 DSR 한도 | 40% | 은행권 일반 기준 |
| 스트레스 DSR 가산금리 (3단계) | +1.5%p | 변동금리 가정 시 적용 |
| 기존 부채 연간 원리금 | 사용자 입력 | 신용대출·기존 주담대 등 |

### 3-2. LTV / 정책 한도 / 취득세 / 중개보수

→ `home-purchase-fund`와 동일 데이터(`LTV_RATES`, `getPolicyLoanLimit`, `calcAcquisitionTax`, `calcBrokerageFee`) 그대로 사용. 신규 데이터 정의 불필요.

### 3-3. PIR (Price to Income Ratio) 참고 구간

| PIR 구간 | 해석 |
|---|---|
| 5배 미만 | 소득 대비 부담 낮음 |
| 5~8배 | 평균적 수준 (서울 평균 PIR 8~10배 내외 참고) |
| 8~12배 | 부담 높음 |
| 12배 초과 | 매우 높음 — 장기 상환 계획 필요 |

> PIR 구간 기준은 KB국민은행 주택가격동향, 통계청 자료를 참고한 일반적 해석 구간이며, 지역·시점별 실제 평균과 차이가 있을 수 있음을 InfoNotice에 명시.

## 7. 계산 로직

### 7-1. DSR 기준 최대 대출 가능액 (역산)

```text
연간 가용 원리금 = 연소득 × DSR한도(%) - 기존 대출 연간 원리금 상환액
월 가용 원리금 = 연간 가용 원리금 / 12

월이율 = (대출금리 + 스트레스가산) / 12
n = 대출기간(년) × 12

최대 대출 가능액(DSR기준)
  = 월 가용 원리금 × [(1+월이율)^n - 1] / [월이율 × (1+월이율)^n]
```

### 7-2. 적정 매매가 역산 (반복 계산)

LTV·취득세·중개보수가 모두 매매가의 함수이므로, 매매가를 변수로 두고 다음 식이 성립하는 지점을 이분 탐색(binary search)으로 구한다.

```text
최대 대출 가능액 = min(DSR기준 대출가능액, 매매가 × LTV비율, 정책상 한도)

필요 현금 = (매매가 - 최대 대출 가능액)
          + calcAcquisitionTax(매매가, 주택수, 지역유형)
          + calcBrokerageFee(매매가)

목표: 필요 현금 = 보유 현금 이 되는 매매가(적정 매매가)를 이분 탐색으로 산출
(매매가 1천만원 단위 반올림)
```

### 7-3. PIR / 월 상환 부담률

```text
PIR = 적정 매매가 / 연소득

월 상환액 = 최종 대출액 × [월이율 × (1+월이율)^n] / [(1+월이율)^n - 1]
월 상환 부담률 = 월 상환액 / (연소득 / 12) × 100
```

### 7-4. 부담 수준 배지

```text
PIR < 5   → "낮음"
5 <= PIR < 8  → "보통"
8 <= PIR < 12 → "높음"
PIR >= 12 → "매우 높음"
```

| 결과와 실제 대출 한도가 다른 이유는? | 신용점수, 기존 대출, 소득 증빙 방식, 스트레스 DSR 적용 여부에 따라 차이 발생 |

## 15. 우선순위 판단

| 평가 항목 | 점수 | 근거 |
|---|---:|---|
| 검색 수요 | 상 | "연봉 대비 집값", "내 연봉으로 집 살 수 있을까" 류 질문형 검색 다수 |
| 기존 클러스터 적합도 | 상 | `home-purchase-fund`와 정/역방향 쌍을 이루며 데이터·로직 재사용 가능 |
| 구현 난이도 | 중 | 이분 탐색 로직 추가 필요하나 기존 LTV/취득세 함수 재사용으로 단축 |
| 정책 변동 리스크 | 중 | DSR·LTV·취득세 기준은 home-purchase-fund와 공유되어 한 곳만 업데이트하면 동기화 가능 |
| 전환 가능성 | 상 | home-purchase-fund ↔ 본 계산기 상호 CTA로 페이지뷰·체류시간 동반 상승 기대 |

결론: `home-purchase-fund`의 데이터·계산 함수를 그대로 재사용할 수 있어 개발 비용 대비 효과가 크고, 부동산 카테고리의 핵심 진입점(연봉→집값 역산)으로서 우선 구현 가치가 높다.
