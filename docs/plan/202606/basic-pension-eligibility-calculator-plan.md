# 기초연금 수급 가능성 계산기 — 기획 문서

## 1. 배경

- "결혼/청년/고령자" 콘텐츠 확장 논의에서 1순위로 선정. 기존 코드베이스에 `national-pension-calculator`(국민연금), `pension-optimal-age`는 있지만 **기초연금**은 완전히 빈 자리.
- "부모님이 기초연금을 받을 수 있는지"는 3040 자녀 세대가 직접 검색하는 패턴이 강해 가구 내 검색 의도가 명확함 (관련 기사: "노인 단독가구, 소득인정액 247만원 이하면 기초연금 받는다" — 정책브리핑·한국경제 등 다수 보도, 2026-01).
- 기존 `livelihoodBenefitIncomeRecognition.ts`, `housingBenefitIncomeRecognition.ts`, `basicLivelihoodRecipientAssetStandard.ts`가 모두 `welfareBenefitEligibility.ts`의 소득인정액 계산 인프라(가구원수/지역/재산 입력 → 소득인정액 산출)를 공유 재사용하는 패턴을 이미 갖고 있음. 기초연금도 "소득평가액 + 재산의 소득환산액" 구조라 **개념은 같지만 계산식과 기준값이 완전히 다르므로 기존 인프라를 그대로 재사용할 수 없고, 별도 데이터 파일이 필요**.

## 2. 컨셉

- "내가 얼마 버는지"가 아니라 **"부모님이 기초연금을 받을 수 있는지"**를 자녀 입장에서 점검하는 계산기. 기존 복지 계산기들과 동일한 "자가 점검용 추정, 최종 확인은 주민센터/복지로" 원칙 적용.
- 입력값: 가구 형태(단독/부부), 거주 지역(대도시/중소도시/농어촌 — 기본재산공제액 결정), 근로소득, 기타소득(공적연금 등), 일반재산(부동산 등), 금융재산, 부채, 고급자동차·회원권 유무.
- 출력값: 소득인정액, 선정기준액 대비 격차, 수급 가능성(가능/경계/불가), 예상 기초연금액(국민연금 연계 감액·부부감액 반영 전 단순 추정).

## 3. 핵심 계산 로직 (2026년 기준, 보건복지부·정책브리핑 2026-01 보도 기준 확인됨)

### 선정기준액
| 가구 형태 | 2026년 선정기준액 |
|---|---|
| 단독가구 | 2,470,000원 |
| 부부가구 | 3,952,000원 |

### 소득평가액
```
소득평가액 = {0.7 × (근로소득 - 116만원)} + 기타소득(공적연금, 사업소득 등)
```
- 근로소득 116만원 이하면 소득평가액에 반영되는 근로소득은 0원
- "기타소득"에는 국민연금 등 공적이전소득이 그대로 포함되므로, 국민연금 수령액이 있는 부모님은 이 항목에 입력

### 재산의 소득환산액
```
재산의 소득환산액 = [{(일반재산 - 기본재산액) + (금융재산 - 2,000만원) - 부채} × 재산의 소득환산율 ÷ 12] + P
```
| 지역 | 기본재산공제액 |
|---|---|
| 대도시·특례시 | 1억 3,500만원 |
| 중소도시 | 8,500만원 |
| 농어촌 | 7,250만원 |

- 금융재산 공제 2,000만원 (전국 동일, 생계급여와 다른 기준이므로 별도 상수 필요)
- 재산의 소득환산율: 연 4% (월 환산 시 ÷12) — ⚠️ 설계 단계에서 보건복지부 고시 원문으로 재확인 필요
- P(고급자동차·회원권 등): 월 100% 환산 — 이 계산기에서는 "확인 필요" 플래그로 단순화 처리(기존 `welfareBenefitEligibility.ts`의 자동차 처리 패턴과 동일)

### 수급 가능성 판정 및 급여액
- 소득인정액 < 선정기준액 → 수급 가능 구간
- 2026년 월 최대 지급액: 단독 349,700원 (부부는 20% 감액 후 가구당 산정, 정확한 부부 감액 후 금액은 설계 단계에서 재확인)
- 국민연금 연계 감액(기초연금의 150% 초과 수급 시 단계적 감액), 소득역전방지 감액은 **이 계산기에서는 1차 버전에서 제외**하고 "국민연금 수령액이 많으면 일부 감액될 수 있다"는 안내 문구로만 처리 (정확한 감액 공식은 복잡도가 높아 과대/과소 추정 리스크가 큼)

## 4. 데이터 구조 방향

- 신규 파일: `src/data/basicPensionEligibilityCalculator.ts`
- `welfareBenefitEligibility.ts`와 유사한 타입 패턴(HouseholdRegion, HousingType 등)을 재사용하되, 임계값·환산율·공제액은 전부 기초연금 전용 상수로 새로 정의 (생계급여 상수와 절대 혼용하지 않음 — 혼용 시 잘못된 수급 판정이 나가는 게 가장 위험한 실패 모드)
- 프리셋 예시: "단독 노인, 국민연금 30만원", "부부 가구, 농어촌 자가", "도시 거주 단독, 금융재산 3천만원 경계 사례" 등 기존 LBIRC_PRESETS 패턴과 동일하게 4개 내외

## 6. 리스크

- **국민연금 연계 감액 단순화**: 실제로는 기초연금 수급자 중 국민연금 가입기간이 길거나 수령액이 기초연금의 150%를 넘으면 최대 50%까지 감액되는 복잡한 공식이 있음. 이를 생략하고 "감액 가능성 있음" 안내만 하는 것은 의도적 단순화이며, 반드시 InfoNotice·FAQ에 명시.
- **재산의 소득환산율 미확정**: 4%는 일반적으로 알려진 값이나 설계 단계에서 보건복지부 고시 원문 또는 복지로 모의계산 결과로 교차 검증 필요.
- **부부가구 급여액 산정**: 부부 감액 20% 적용 후 정확한 1인당 금액은 설계 단계에서 재확인.
- **복지 계산기 신뢰도 원칙 준수**: 기존 생계급여 계산기와 동일하게 "최종 확인은 복지로/주민센터"를 결과 화면에 반복 고지 — 확정 수급 여부로 오인되지 않도록.
