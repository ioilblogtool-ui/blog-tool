# 인터넷 TV 약정 해지 위약금 계산기 기획

> 인터넷·IPTV·결합상품을 갈아타기 전에 “지금 해지하면 손해인지”를 빠르게 판단하는 생활비 절약형 계산기. 통신사별 약관과 가입 시점에 따라 실제 위약금이 달라질 수 있으므로, 공식 확정 위약금이 아니라 입력값 기반 `예상 위약금`과 `갈아타기 손익분기점`을 보여주는 도구로 설계한다.

## 1. 배경 및 목적

- **계기**:
  - 네이버 검색에서 `인터넷 해지 위약금`, `TV 약정 위약금`, `통신사 갈아타기`, `인터넷 갈아타기 사은품`은 사용자가 실제 돈을 아끼려는 강한 의도를 가진 키워드다.
  - 비교계산소는 이미 세금, 지원금, 성과급, 생활비 계산기를 다수 보유하고 있어 “월 고정비 절약” 카테고리 확장에 적합하다.
  - 알뜰폰·통신비 절약 콘텐츠와 묶으면 `휴대폰 요금제 갈아타기 → 인터넷/TV 약정 점검 → 결합할인 재계산` 흐름을 만들 수 있다.
  - 사용자는 통신사 고객센터에 전화하기 전 대략 얼마가 나올지 먼저 알고 싶어 한다.

- **목적**:
  1. `인터넷 해지 위약금`, `TV 약정 위약금`, `인터넷 약정 위약금 계산기` 검색 유입 확보
  2. `통신사 갈아타기`, `인터넷 갈아타기 손해`, `인터넷 사은품 위약금` 검색 의도 대응
  3. 인터넷·TV·모바일 결합 할인까지 포함한 월 고정비 절약 클러스터 구축
  4. 단순 위약금 계산에서 끝나지 않고 “새 통신사로 바꾸면 몇 개월 뒤 이득인지”까지 판단 제공

## 4. 타겟 사용자

1. **인터넷 약정이 남은 사용자**
   - 약정 3년 중 1년~2년 사용
   - 이사, 품질 문제, 요금 인상, 사은품 때문에 갈아타기 고민
   - 고객센터 전화 전 대략 위약금 규모를 알고 싶음

2. **IPTV 포함 결합상품 사용자**
   - 인터넷만 쓰는 사람보다 위약금 항목이 복잡함
   - TV 셋톱박스, 채널팩, 장비 임대료, 결합할인 반환 여부가 궁금함

3. **통신사 갈아타기 사은품 비교 사용자**
   - 새 통신사 사은품이 30만~60만 원일 때 기존 위약금을 내고 바꿔도 이득인지 계산하고 싶음
   - 월요금 차이까지 포함한 손익분기점 필요

4. **알뜰폰/통신비 절약 관심 사용자**
   - 휴대폰 요금제를 바꾸면서 인터넷 결합 할인도 같이 점검하고 싶음
   - 결합 해지로 손해가 생기는지 알고 싶음

5. **이사 예정자**
   - 이사 지역에서 기존 통신사가 설치 불가하거나 품질이 낮을 때 해지 비용이 궁금함
   - 이전 설치비와 신규 설치비까지 비교하고 싶음

## 7. 계산 로직 포지션

실제 위약금은 통신사, 가입 시점, 상품, 할인 유형, 약관에 따라 다르다. 따라서 MVP는 아래 4개 항목을 분리해 예상한다.

```text
예상 위약금 =
  약정 할인반환금
  + 장비 할인반환금
  + 사은품 반환 예상액
  + 설치비/기타 반환 예상액
```

갈아타기 순손익:

```text
갈아타기 순손익 =
  새 통신사 사은품
  + 남은 기간 월요금 절감액
  - 예상 위약금
  - 신규 설치비
  - 기존 재약정 포기 혜택
```

손익분기점:

```text
손익분기 개월 =
  max(0, 예상 위약금 + 신규 설치비 - 새 통신사 사은품)
  / 월요금 절감액
```

단, 월요금 절감액이 0 이하이면 “월요금으로 회수 불가”로 표시한다.

## 8. 계산 모델 상세

### 8.1 할인반환금 단순 모델

할인반환금은 통신사 약관마다 산정 방식이 다르므로 기본 계산기는 `누적 할인액 × 반환계수` 방식으로 단순화한다.

| 사용 구간 | 기본 반환계수 | 설명 |
|---|---:|---|
| 약정 0~25% 사용 | 100% | 초반 해지라 할인 반환 부담이 큼 |
| 약정 25~50% 사용 | 70% | 일부 사용 기간 반영 |
| 약정 50~75% 사용 | 45% | 중반 이후 반환 부담 감소 |
| 약정 75~100% 사용 | 20% | 만료가 가까워질수록 부담 감소 |
| 약정 만료 이후 | 0% | 약정 위약금 없음 |

계산:

```text
누적 할인액 = 월 할인액 × 사용개월
약정 할인반환금 = 누적 할인액 × 반환계수
```

주의:

- 이 방식은 공식 통신사 산식이 아니라 예상 계산용 모델이다.
- 실제 통신사 산식은 할인 유형별로 다를 수 있다.
- 페이지 상단과 결과 카드에 `예상`, `공식 청구액 아님` 배지를 표시한다.

### 8.2 장비 할인반환금

```text
장비 할인반환금 = 장비 임대료 월 할인액 × 사용개월 × 반환계수
```

별도 옵션:

- 셋톱박스
- 공유기
- 리모컨/어댑터 미반납

장비 미반납비는 위약금과 다른 성격이므로 별도 카드로 분리한다.

### 8.3 사은품 반환 예상액

사은품 반환은 통신사/대리점/가입 경로별 조건이 다르다. 기본 모델은 아래처럼 잡는다.

| 사용개월 | 사은품 반환 예상 |
|---:|---:|
| 반환 기준개월 미만 | 사은품 전액 또는 일부 반환 위험 |
| 반환 기준개월 이상 | 기본 반환 없음으로 계산 |

기본값:

```text
사은품 반환 기준개월 = 12개월
```

계산:

```text
사은품 반환 예상액 =
  사용개월 < 반환 기준개월 ? 가입 사은품 : 0
```

고급 옵션:

- `전액 반환`
- `잔여기간 비례 반환`
- `반환 없음`

### 8.4 설치비/이전비

기존 통신사에서 할인받은 설치비가 있다면 반환될 수 있고, 새 통신사 가입 시 신규 설치비가 발생할 수 있다.

결과에서는 아래처럼 분리한다.

| 항목 | 의미 |
|---|---|
| 기존 설치비 반환 예상 | 기존 가입 때 면제받은 설치비를 돌려내는 경우 |
| 신규 설치비 | 새 통신사 설치 시 새로 내는 비용 |
| 이전 설치비 | 기존 통신사를 유지하고 이사할 때 드는 비용 |

## 9. 결과 카드 설계

상단 결과 KPI:

| 카드 | 예시 문구 |
|---|---|
| 예상 위약금 | `약 238,000원` |
| 사은품 반환 위험 | `0원` 또는 `450,000원 반환 가능성` |
| 갈아타기 순손익 | `약 182,000원 이득` |
| 손익분기점 | `약 5개월 후 이득 전환` |

판정 문구:

| 조건 | 판정 |
|---|---|
| 순손익 > 100,000원 | `갈아타기 검토 가능` |
| -100,000원 <= 순손익 <= 100,000원 | `조건 재확인 필요` |
| 순손익 < -100,000원 | `지금 해지는 손해 가능성 큼` |
| 사용개월 >= 약정기간 | `약정 만료. 위약금 부담 낮음` |
| 사용개월 < 사은품 반환 기준개월 | `사은품 반환 위험 높음` |

## 12. 계산 함수 설계

```ts
export const getReturnRate = (usedMonths: number, contractMonths: number) => {
  if (usedMonths >= contractMonths) return 0;
  const progress = usedMonths / contractMonths;
  if (progress < 0.25) return 1;
  if (progress < 0.5) return 0.7;
  if (progress < 0.75) return 0.45;
  return 0.2;
};

export const calculatePenalty = (input: PenaltyInput): PenaltyBreakdown => {
  const returnRate = getReturnRate(input.usedMonths, input.contractMonths);
  const contractDiscountReturn = input.monthlyDiscount * input.usedMonths * returnRate;
  const deviceDiscountReturn = input.deviceDiscount * input.usedMonths * returnRate;
  const giftReturn =
    input.usedMonths < input.giftReturnThresholdMonths ? input.signupGift : 0;

  return {
    contractDiscountReturn,
    deviceDiscountReturn,
    giftReturn,
    installFeeReturn: input.installFeeReturn,
    deviceNonReturnFee: input.deviceNonReturnFee,
    totalPenalty:
      contractDiscountReturn +
      deviceDiscountReturn +
      giftReturn +
      input.installFeeReturn +
      input.deviceNonReturnFee,
  };
};

export const calculateSwitchProfit = (
  input: PenaltyInput,
  penalty: PenaltyBreakdown,
): SwitchProfitResult => {
  const remainingMonths = Math.max(input.contractMonths - input.usedMonths, 0);
  const monthlySaving = input.monthlyFee - input.newMonthlyFee;
  const remainingTermSaving = monthlySaving * remainingMonths;
  const immediateNetProfit =
    input.newSignupGift -
    penalty.totalPenalty -
    input.newInstallFee -
    input.forfeitedRetentionBenefit;
  const totalSwitchProfit = immediateNetProfit + remainingTermSaving;
  const breakEvenBase = penalty.totalPenalty + input.newInstallFee - input.newSignupGift;
  const breakEvenMonths = monthlySaving > 0 ? Math.max(0, Math.ceil(breakEvenBase / monthlySaving)) : null;

  return {
    monthlySaving,
    immediateNetProfit,
    remainingTermSaving,
    totalSwitchProfit,
    breakEvenMonths,
    status: getResultStatus(totalSwitchProfit, input.usedMonths, input.contractMonths),
    headline: getResultHeadline(totalSwitchProfit, breakEvenMonths),
  };
};
```

## 22. 최종 판단

이 콘텐츠는 검색 의도가 강하고 사용자가 바로 계산 행동을 할 가능성이 높다. 특히 `인터넷 해지 위약금`은 단순 정보성보다 “내 상황을 넣어 보고 싶다”는 계산기 적합도가 높다.

MVP는 통신사별 약관을 정확히 재현하는 방향이 아니라, **약정기간·사용개월·할인액·사은품을 입력해 예상 위약금과 갈아타기 손익분기점을 계산하는 생활비 절약 도구**로 출시하는 것이 적절하다. 이후 네이버 유입에서 `KT`, `SK`, `LG`, `사은품`, `이사` 키워드가 잡히면 통신사별 프리셋과 이사/재약정 비교 콘텐츠로 확장한다.
