# 청년월세지원 계산기 — 기획 문서

## 1. 배경

- "유사 콘텐츠 추천" 논의에서 1순위로 선정 — 검색량이 크고, 기존 `livelihood-benefit-income-recognition.ts`의 소득인정액 로직을 확장 재사용할 수 있어 구현 효율이 가장 좋음.
- **가족돌봄수당과 결정적으로 다른 점**: 가족돌봄수당은 지자체별 자체 사업이라 매번 "확인 필요"를 반복해야 했지만, **청년월세지원은 국토교통부가 운영하는 중앙정부 통일 제도**다. 그래서 이 콘텐츠는 가족돌봄수당보다 훨씬 확신 있는 판정을 줄 수 있어, 직전 우려했던 "또 확인 필요네" 피로감 문제를 피할 수 있음.
- 2026년부터 한시 사업이 **상시 사업으로 전환**되어 연중 언제든 신청 가능해진 것도 콘텐츠 시의성이 좋은 변화 포인트.

## 2. 사실 검증 결과 (2026년 기준, 다수 출처 교차검증)

### 대상·기본 조건
- 연령: 만 19세~34세 이하 (서울 등 일부 지자체는 19~39세까지 확대 — 지자체 자체 추가사업과 혼동 주의)
- 무주택 청년이어야 함
- 부모와 주민등록상·실거주상 모두 분리되어 있어야 함
- 임대차계약을 맺고 월세를 내며 거주 중이어야 함

### 소득 기준 (이중 구조 — 본인 청년가구 + 원가구 둘 다 충족해야 함)
| 구분 | 기준 |
|---|---|
| 청년가구(본인) 소득 | 기준 중위소득 60% 이하 (2026년 1인 가구 기준 약 153만 8,543원 ≈ 154만원) |
| 원가구(부모 등) 소득 | 기준 중위소득 100% 이하 |

### 재산 기준
- 총재산가액 1.22억원 이하 (⚠️ 청년 본인 가구 기준으로 확인됨. 원가구 재산기준은 별도/다른 값일 가능성 있어 설계 단계 추가 확인 필요)

### 지원금액
- 월 최대 20만원, **실제 납부 월세 범위 내**에서만 지원 (월세가 20만원보다 적으면 그 금액만 지원)
- 최대 24개월(회) 지원 → 최대 총 480만원
- 임차보증금·관리비는 지원 대상에서 제외, 월세만 인정

### 제외 대상 (중요 — 계산기 결과에 반드시 반영)
- 주택 소유자(분양권·입주권 포함)
- 직계존속·형제자매 등 2촌 이내 혈족(배우자의 2촌 이내 혈족 포함) 주택에 임차
- 공공임대주택(공무원임대주택 포함) 거주자
- 한 방에 여러 명이 거주하는 전대차 형태
- **지자체 시행 월세지원 사업을 이미 받고 있는 경우** (중복 수급 불가 — 서울형 등 지자체 자체 청년 월세지원과는 양자택일)

### 제도 변경
- 2026년부터 한시 사업 → **상시 사업 전환**, 연중 신청 가능 (기존엔 특정 모집 기간에만 신청 가능했던 점과 차별화 포인트)

## 3. 컨셉

- "나도 청년월세지원 받을 수 있을까?"를 본인 소득, 부모님(원가구) 소득, 재산, 거주 형태 입력으로 자가 점검.
- 가족돌봄수당과 달리 **중앙정부 통일 기준**이므로, 지역 조건을 별도 "확인 필요"로 분리할 필요가 없음 — 대신 **"지자체 자체 월세지원과 중복 수급 불가"**라는 다른 종류의 주의사항이 핵심 변수.
- 결과에 "월세 20만원 미만이면 실제 월세만큼만 지원"이라는 점을 명확히 보여줘서, 단순히 "20만원 받는다"고 오인하지 않게 설계.

## 4. 기존 자산 재사용 방향

| 재사용 대상 | 위치 | 활용 방식 |
|---|---|---|
| 가구원수별 중위소득·소득인정액 계산 패턴 | `welfareBenefitEligibility.ts`, `livelihoodBenefitIncomeRecognition.ts` | 같은 구조의 `Record<householdSize, threshold>` 패턴 재사용, 단 본인가구(중위60%)·원가구(중위100%) **이중 기준표**를 새로 정의 |
| 조건입력→자가점검판정 UI 패턴 | `family-care-allowance-calculator.astro`/`.js`, `basic-pension-eligibility-calculator.astro`/`.js` | SimpleToolShell + 프리셋 + 종합판정카드 + 체크리스트 구조 동일 재사용 |
| "확인 필요" 분리 표시 패턴 | `family-care-allowance-calculator.ts`의 지역 조건 처리 방식 | 이번엔 지역이 아니라 **"지자체 자체 월세지원 중복 수급 여부"**를 같은 방식으로 별도 확인 항목 처리 |

## 5. 입력값 / 출력값

### 입력값
- 만 나이 (또는 생년월)
- 무주택 여부
- 부모와 별거 여부(주민등록·실거주 분리)
- 임대차계약 기반 월세 거주 여부
- 본인 가구원 수, 본인 월소득(또는 건강보험료)
- 원가구(부모 등) 가구원 수, 원가구 월소득
- 총재산가액(본인)
- 월세 금액
- 주택 소유 여부, 2촌 이내 혈족 임차 여부, 공공임대주택 거주 여부, 지자체 월세지원 중복수급 여부 (제외요건 체크)

### 출력값
- 종합 판정: 가능성 높음 / 일부 조건 미충족 / 제외 대상
- 조건별 체크리스트: 연령·무주택·별거·본인소득·원가구소득·재산·제외요건
- 예상 월 지원금(실제 월세와 20만원 중 작은 값) × 최대 24개월 = 예상 총 지원금

## 8. 리스크

- **원가구 재산기준 미확정**: 1.22억원이 청년 본인 가구 기준인지 원가구도 동일 적용인지 설계 단계에서 추가 확인 필요. 확정 못하면 "원가구 재산기준은 별도 확인 필요"로 보수적 처리.
- **지자체 자체 사업과의 혼동 리스크**: "서울형 청년월세지원"처럼 지자체 자체 추가사업이 별도로 존재하고, 이 계산기가 다루는 건 국토부 중앙정부 사업이라는 점을 명확히 구분해야 함 — 혼동하면 사용자가 잘못된 판단을 할 수 있음.
- **월세 지원액 오인 리스크**: "월 20만원 받는다"고 단정하면 안 됨 — 실제 월세가 20만원보다 적으면 그만큼만 지원된다는 점을 결과 화면에 명확히 표시.
- **나이 상한 지역차**: 일부 지자체(서울 등) 자체 사업은 39세까지 확대하지만, 이 계산기가 다루는 국토부 중앙 사업은 34세 기준 — 사용자가 지자체 사업과 헷갈리지 않도록 FAQ에서 구분.
