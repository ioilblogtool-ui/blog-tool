# 공기업 성과급 비교 2026 기획

## 1. 배경 및 목적

- **계기**: 비교계산소는 교사·경찰·군인·공무원·교수 등 직업별 연봉/월급 콘텐츠에서 이미 검색 유입 기반이 있고, 성과급 계산기 클러스터도 삼성·SK하이닉스·현대차·LG·조선·반도체 등으로 확장되어 있음. 그러나 공기업/공공기관 성과급을 한 번에 비교하는 콘텐츠는 아직 부족함.
- **검색 의도**:
  1. "한전 성과급", "코레일 성과급", "LH 성과급"처럼 특정 기관 성과급을 알고 싶은 의도
  2. "공기업 성과급 순위", "공공기관 성과급 비교"처럼 기관별 차이를 비교하려는 의도
  3. "공기업 연봉 성과급 포함", "공기업 경영평가 성과급"처럼 성과급 산정 구조를 이해하려는 의도
- **목적**:
  1. 공기업/공공기관 연봉·성과급 검색 클러스터 신규 확보
  2. 기존 `teacher-salary-2026`, `police-salary-2026`, `public-servant-salary-2026`, `military-salary-2026`, `new-employee-salary-2026` 등 직업 연봉 콘텐츠와 내부 링크 연결
  3. 기존 성과급 도구(`bonus-after-tax-calculator`, `corporate-bonus-comparison-2026`, `compare/bonus`)와 공공부문 성과급 수요를 연결
  4. 네이버에서 클릭 유도 가능한 "어디가 더 많이 받나"형 비교 콘텐츠 확보

## 6. 데이터 신뢰성 원칙

공기업 성과급은 실제 지급액이 기관별 공시, 경영평가 등급, 내부 지급률, 직급, 근속, 직군에 따라 달라진다. 따라서 아래 원칙을 반드시 지킨다.

1. **공식/공시/추정 구분**
   - 공식 공시값: 알리오·기관 공시·정부 발표 등 출처가 있는 값
   - 추정값: 연봉 대비 성과급률 또는 기준연봉 입력으로 계산한 값
   - 설명값: 경영평가 등급이 성과급에 영향을 준다는 구조 설명

2. **"실제 수령액" 단정 금지**
   - "A기관은 성과급 000만원 지급"처럼 단정하지 않음
   - "공시 평균 기준", "입력 연봉 기준 추정", "경영평가 결과에 따라 변동" 표현 사용

3. **기관 유형 설명**
   - 공기업, 준정부기관, 기타공공기관이 섞일 수 있음
   - 제목은 검색 친화적으로 "공기업"을 쓰되, 본문에서 "공공기관 전체 비교" 성격임을 설명

4. **세전/세후 분리**
   - 성과급 비교 표는 세전 기준
   - 실수령액은 `bonus-after-tax-calculator` CTA로 연결하거나 간이 세후 추정만 제공

5. **날짜 명시**
   - "2026년 기준"이라고 쓰는 경우 사용한 공시/평가/보수 기준연도를 명확히 표시
   - 데이터 업데이트 일자를 페이지 상단에 표시

## 8. 데이터 설계 초안

파일명: `src/data/publicEnterpriseBonusComparison2026.ts`

```ts
export type PublicEnterpriseType = "marketPublic" | "quasiMarketPublic" | "quasiGovernment" | "otherPublic";
export type PublicEnterpriseIndustry = "energy" | "soc" | "housing" | "finance" | "welfare" | "transport" | "airport" | "water";

export interface PublicEnterpriseBonusEntry {
  id: string;
  name: string;
  shortName: string;
  type: PublicEnterpriseType;
  industry: PublicEnterpriseIndustry;
  defaultSalaryPercent: number;
  defaultSalary: number;
  evaluationSensitivity: "high" | "medium" | "check";
  summary: string;
  caution: string;
  sourceNote: string;
}

export const PUBLIC_ENTERPRISE_BASE_SALARY = 60_000_000;
export const PUBLIC_ENTERPRISE_SIMPLE_TAX_RATE = 0.22;

export const PUBLIC_ENTERPRISE_BONUS_ENTRIES: PublicEnterpriseBonusEntry[] = [
  {
    id: "kepco",
    name: "한국전력공사",
    shortName: "한전",
    type: "marketPublic",
    industry: "energy",
    defaultSalaryPercent: 8,
    defaultSalary: 60_000_000,
    evaluationSensitivity: "high",
    summary: "에너지 대표 공기업으로 경영평가와 재무 상황에 따라 성과급 체감 차이가 클 수 있습니다.",
```

- 경영평가 등급별 실제 지급률을 일반화하기 어렵다면, "성과급률" 대신 "시뮬레이션 기본값"으로 표현.

## 18. 최종 판단

이 콘텐츠는 신규 검색어 확보와 기존 사이트 구조 강화가 모두 가능하다. 특히 "성과급" 클러스터와 "직업별 연봉" 클러스터 사이를 이어주는 브릿지 역할을 한다.  

구현 우선순위는 높음. 다만 실제 개발 전에는 1차 10개 기관의 데이터 기준을 먼저 확정해야 한다. 숫자 확정이 어렵다면 첫 버전은 "성과급 구조 비교 + 입력 연봉 기준 시뮬레이션"으로 출시하고, 공식 공시 확인 후 표를 업데이트하는 방식이 안전하다.
