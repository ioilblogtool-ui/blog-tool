## 1. 한 줄 요약

`부부 월 현금흐름 계산기`는 연봉 계산기보다 한 단계 더 생활에 가까운 계산기다. 남편 연봉, 아내 연봉, 성과급, 주거비, 육아비, 생활비, 투자금을 한 화면에 넣으면 월 실수령 합계, 고정비, 변동비, 투자 가능액, 저축률, 목표자산 도달 기간을 즉시 보여준다.

이 페이지의 승부처는 "우리 집은 월급이 많은데 왜 돈이 안 남지?", "아이 낳고도 매달 얼마를 모을 수 있지?", "10억까지 몇 년 걸리지?"라는 질문을 한 번에 잡는 것이다.

## 3. 기획 의도

### 3-1. 왜 bigyocalc다운 콘텐츠인가

비교계산소의 강점은 사용자가 자기 숫자를 넣고 결과를 확인하는 계산형 콘텐츠다. 기존 연봉 계산기, 육아비 계산기, 주거비 계산기는 각각 강하지만 사용자의 실제 고민은 보통 하나로 합쳐져 있다.

- 연봉은 높은데 통장 잔고가 늘지 않는다.
- 대출, 관리비, 보험료, 카드값, 육아비가 섞이면 어디서 새는지 모르겠다.
- 맞벌이에서 외벌이 또는 육아휴직으로 전환하면 월 현금흐름이 얼마나 바뀌는지 궁금하다.
- 매달 투자 가능한 금액으로 목표자산까지 몇 년 걸리는지 알고 싶다.

이 계산기는 기존 계산기들을 묶는 허브 역할을 한다. 사용자가 한 번 계산한 뒤 세부 항목을 더 자세히 보려고 연봉 계산기, 육아비 계산기, 분유비 계산기, 대출 계산기, FIRE 계산기로 자연스럽게 이동하게 만든다.

### 3-2. 사용자에게 주는 핵심 가치

1. 부부 합산 실수령액을 월 단위로 이해한다.
2. 고정비와 변동비를 나눠 돈이 새는 구간을 본다.
3. 육아비와 주거비를 함께 넣어 현실적인 잉여현금을 계산한다.
4. 저축률을 계산해 현재 가계 체력을 파악한다.
5. 현재 투자 가능액으로 목표자산 도달 기간을 추정한다.
6. 맞벌이, 외벌이, 육아휴직, 대출 증가 같은 시나리오를 비교한다.

### 6-2. 결과 해석 문장


## 7. 계산 로직

### 7-1. 세후 월소득 추정

1차 구현은 기존 연봉 계산기 로직을 최대한 재사용한다. 정확한 급여명세서 재현이 아니라 콘텐츠형 추정 계산기이므로 `추정` 배지를 명확히 붙인다.

```text
개인별 연간 총소득 = 세전 연봉 + 연간 성과급
개인별 월 실수령 = 연간 총소득 기준 4대보험 + 근로소득세 간이 추정 후 / 12
부부 월 실수령 합계 = 남편 월 실수령 + 아내 월 실수령 + 기타 월 소득
```

### 7-2. 주거비

```text
월 대출이자 = 대출잔액 * 금리 / 12
월 원리금 직접 입력값이 있으면 직접 입력값 우선
주거 고정비 = 월 원리금 또는 월 대출이자 + 월세 + 관리비
```

1차 구현에서는 원금상환까지 정확히 계산하려면 사용자가 월 원리금을 직접 넣을 수 있게 한다. 기본값은 이자 기준으로 계산하고, UI에서 "원리금 직접 입력"을 제공한다.

### 7-3. 지출 분류

```text
육아비 = 어린이집 + 분유/이유식 + 기저귀 + 병원 + 자녀보험 + 돌봄
생활비 = 식비 + 교통/차량 + 통신/구독 + 의료/미용/의류 + 경조사 + 기타카드비
고정비 = 주거 고정비 + 부부 보험료 + 투자/저축 중 강제 납입 성격 항목
변동비 = 육아비 + 생활비
총지출 = 고정비 + 변동비 + 월 투자/저축 입력액
월 잉여현금 = 월 실수령 합계 - 고정비 - 변동비
투자 가능액 = max(월 잉여현금, 0)
저축률 = 투자 가능액 / 월 실수령 합계 * 100
```

### 7-4. 목표자산 도달 기간

```text
현재 투자자산 = 사용자가 입력한 현재 투자자산
월 투자액 = 투자 가능액 또는 사용자가 이미 입력한 월 투자/저축 합계 중 선택
목표자산 = 기본 10억 원
월 기대수익률 = (1 + 연 기대수익률)^(1/12) - 1

반복 계산:
  자산 = 현재 투자자산
  매월 자산 = 자산 * (1 + 월 기대수익률) + 월 투자액
  자산 >= 목표자산이 되는 월 수를 찾음
```

최대 계산 기간은 60년으로 제한한다. 60년 내 도달하지 못하면 "현재 조건으로는 목표자산 도달이 어렵습니다"로 표시한다.

## 8. 시나리오 기능

### 8-1. 기본 프리셋

| 프리셋 | 소득 | 지출 특징 | 목적 |
|---|---|---|---|
| 신혼 맞벌이 | 6,000만 + 5,000만 | 자녀 없음, 주거비 중심 | 결혼 직후 저축률 |
| 영유아 1명 | 6,000만 + 5,000만 | 분유, 기저귀, 어린이집 | 육아비 반영 |
| 외벌이 전환 | 7,000만 + 0원 | 육아비 일부 감소 | 육아휴직/퇴사 시나리오 |
| 대출 많은 부부 | 7,000만 + 5,000만 | 주담대 5억, 관리비 높음 | 주거비 압박 확인 |
| FIRE 집중형 | 8,000만 + 6,000만 | 투자 월 400만 원 | 목표자산 기간 확인 |

### 8-2. 비교 모드

1차 MVP는 현재 입력값만 계산한다. 2차 확장에서 다음 비교 모드를 붙인다.

- 맞벌이 vs 외벌이
- 자녀 없음 vs 영유아 1명
- 월세 vs 대출이자
- 투자 월 100만 원 vs 250만 원
- 기대수익률 3% vs 5% vs 7%

## 12. 애드센스/품질 기준

- `추정`, `참고`, `가정` 배지를 명확히 표시한다.
- 세금, 보험료, 투자수익률을 공식 보장값처럼 표현하지 않는다.
- 미완성 상태를 암시하는 문구나 빈 링크를 남기지 않는다.
- 본문은 계산기 사용법만 얇게 쓰지 말고 현금흐름 개선 순서, 지출 분류 기준, 저축률 해석을 충분히 제공한다.
- 계산 결과가 금융 조언처럼 보이지 않도록 "개인별 상황에 따라 다르며 의사결정 참고용" 문구를 둔다.
- 투자는 원금 손실 가능성이 있다는 문구를 기대수익률 입력 근처에 배치한다.

## 15. 최종 판단

이 계산기는 비교계산소의 핵심축이 될 수 있다. 단일 계산 결과가 아니라 소득, 주거, 육아, 생활비, 투자라는 여러 카테고리를 한 화면에 묶기 때문에 체류시간과 내부 회전율이 높다. 특히 분유 가격 비교표, 육아비 계산기, 연봉 계산기, FIRE 계산기를 자연스럽게 연결할 수 있어 SEO와 애드센스 관점에서도 우선순위가 높다.
