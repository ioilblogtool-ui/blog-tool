## 1. 왜 새로 만들까

`/tools/livelihood-benefit-income-recognition/`(생계급여 소득인정액 계산기)이 적은 노출에서도 높은 CTR을 보였다. 같은 패턴의 검색어 클러스터 중 아직 비교계산소가 정확매칭 페이지를 갖지 못한 키워드가 `주거급여 소득인정액 계산기`다.

현재 `/tools/welfare-benefit-eligibility/`가 생계·의료·주거·교육급여를 한 번에 보여주는 통합 계산기 역할을 하고 있지만, 통합형 페이지는 "주거급여만 빠르게 확인하고 싶은" 검색 의도와 제목·H1 정확도가 맞지 않는다. 또한 주거급여는 2024년 이후 **소득인정액 통과 여부와 별개로 기준임대료(임차가구) 또는 수선비용(자가가구) 산정 로직**이 추가로 필요하다는 점에서, 생계급여 계산기보다 결과 화면 설계가 한 단계 더 필요하다.

따라서 이번 기획은 다음 두 가지를 동시에 한다.

1. 기존 소득인정액 계산 엔진(생계급여 계산기와 동일 로직)을 재사용해 **주거급여 선정기준(중위소득 48%) 통과 여부**를 빠르게 계산한다.
2. 통과 시에는 **임차가구 기준임대료 등급(1~4급지 × 가구원 수)** 또는 **자가가구 수선비용(경/중/대보수)**까지 이어서 계산해, "얼마 받는지"까지 한 화면에서 끝나게 한다.

| 구분 | 기존 통합 계산기 | 생계급여 계산기(선례) | 신규 주거급여 계산기 |
|---|---|---|---|
| URL | `/tools/welfare-benefit-eligibility/` | `/tools/livelihood-benefit-income-recognition/` | `/tools/housing-benefit-income-recognition/` |
| 핵심 의도 | 4대 급여 가능성 전체 확인 | 생계급여 가능 여부 + 예상액 | 주거급여 가능 여부 + 기준임대료/수선비 예상액 |
| 검색어 | 기초생활수급자 자격 계산기 | 생계급여 소득인정액 계산기 | 주거급여 소득인정액 계산기, 주거급여 얼마 |
| 추가 난이도 | - | 소득인정액만 계산 | 소득인정액 + 거주 형태별(임차/자가) 2차 계산 |
| 내부 역할 | 모 계산기 | 검색 유입용 하위 계산기 | 검색 유입용 하위 계산기, 생계급여 계산기와 형제 페이지 |

## 2. 목표

### 비즈니스 목표

- `주거급여 소득인정액 계산기`, `주거급여 계산기`, `주거급여 얼마`, `주거급여 기준임대료` 검색어의 정확매칭 랜딩 확보
- 생계급여 계산기 출시 효과(낮은 노출에도 높은 CTR)를 같은 복지 클러스터의 다음 키워드로 확장
- `/tools/livelihood-benefit-income-recognition/` ↔ `/tools/housing-benefit-income-recognition/` 상호 링크로 복지 계산기 클러스터의 체류·재방문 강화
- `/tools/welfare-benefit-eligibility/`, `/compare/welfare/`로 이어지는 내부 이동 증가

### 사용자 목표

- 우리 집 소득인정액이 주거급여 기준(중위소득 48%)보다 낮은지 빠르게 확인
- 임차가구라면 내 지역(급지)과 가구원 수 기준으로 월 기준임대료가 얼마인지 확인
- 자가가구라면 주택 노후도에 따른 수선비용 지원 등급(경/중/대보수)을 확인
- 실제 월세보다 기준임대료가 적을 때 차감되는 자기부담분 개념 이해
- 주거급여가 어렵다면 생계급여, 교육급여, 에너지바우처 같은 다음 선택지를 확인

## 7. 계산 로직

### 기본 구조 (소득인정액, 생계급여 계산기와 동일 엔진 재사용)

```text
소득인정액 = 소득평가액 + 재산의 소득환산액
주거급여 선정기준 대비 비율 = 소득인정액 / 주거급여 선정기준(중위소득 48%)
```

소득평가액, 재산의 소득환산액 산식은 `livelihood-benefit-income-recognition-calculator.md`의 6~7번 항목과 동일한 간이 모델을 그대로 재사용한다(중복 설계 방지, 같은 유틸 함수 공유).

### 주거급여 통과 시 2차 계산 — 임차가구

```text
기준임대료 = HOUSING_BENEFIT_RENT_TABLE[급지][가구원수]
실제 월 임차료 반영액 = min(실제 월 임차료, 기준임대료)

소득인정액이 생계급여 기준 이하인 경우:
  지급액 = 실제 월 임차료 반영액 (자기부담 없음)

소득인정액이 생계급여 기준 초과 ~ 주거급여 기준 이하인 경우:
  자기부담분 = (소득인정액 - 생계급여 선정기준) × 자기부담비율(약 30%)
  지급액 = max(실제 월 임차료 반영액 - 자기부담분, 0)
```

자기부담비율은 실제 제도(2024년 개편 기준 약 30%)를 단순화한 값이며 결과에 `추정` 배지를 명시한다.

### 주거급여 통과 시 2차 계산 — 자가가구

```text
수선비 지원 등급 = 주택 사용연수 기준 (경: 0~14년차 보수 필요 추정 / 중: 15~19년 / 대: 20년 이상, UI는 직접 선택도 허용)
기준 지원금액 = HOUSING_BENEFIT_REPAIR_TABLE[등급]
지원 비율 = 소득구간별 차등(생계급여 이하 100% / 그 외 주거급여 통과 90%)
예상 지원금액 = 기준 지원금액 × 지원 비율
```

### 판정 단계

| 단계 | 조건 | 메시지 |
|---|---|---|
| 가능성 높음 | 소득인정액 <= 선정기준 95% | 주거급여 신청 가능성이 있는 구간입니다. |
| 경계 | 선정기준 95% 초과~105% 이하 | 입력값과 공제 반영에 따라 결과가 달라질 수 있습니다. |
| 초과 | 선정기준 105% 초과 | 주거급여 기준은 초과한 것으로 보입니다. |
| 확인 필요 | 자동차·재산 특이값 존재 | 주민센터 상담이 필요한 항목이 있습니다. |

## 12. 결론

생계급여 계산기가 검증한 패턴 — "통합 계산기와 경쟁하지 않고, 정확매칭 검색어 전용 입구 역할을 한다" — 을 주거급여로 그대로 확장한다.

다만 주거급여는 소득인정액 통과 여부만으로 끝나지 않고 **거주 형태(임차/자가)에 따른 2차 계산**이 핵심 차별점이므로, 기존 소득인정액 엔진은 재사용하되 기준임대료·수선비용 테이블과 결과 분기 로직을 새로 설계해야 한다.

구현은 **소득인정액 엔진 재사용 + 거주 형태별 결과 분기 신설 + 정확매칭 SEO**가 핵심이다.
