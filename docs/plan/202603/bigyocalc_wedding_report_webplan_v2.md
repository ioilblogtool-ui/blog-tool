# 1. 페이지 목적

## 1.1 핵심 목적
이 페이지는 단순 정보 제공형 리포트가 아니라, 사용자가 **“결혼 비용이 10년 동안 얼마나 부담스러워졌는지”**를 직관적으로 이해하고, 바로 다음 행동으로 이어지도록 만드는 콘텐츠 페이지다.

핵심 목적은 다음 4가지다.

1. `결혼비용 2016 vs 2026` 키워드로 검색 유입 확보
2. 숫자 비교를 통해 공감과 공유 포인트 형성
3. 결혼 준비 예산 계산기로 자연스럽게 이동 유도
4. 웨딩/허니문/답례품/재테크 관련 제휴 링크 클릭 유도

## 1.2 사용자 행동 목표

| 구분 | 목표 행동 |
|---|---|
| 정보 탐색형 사용자 | 10년 비교 데이터를 빠르게 확인 |
| 예비 신혼부부 | 내 연봉 기준 체감 부담 계산 |
| 예산 고민 사용자 | 계산기로 이동해 개인 예산 시뮬레이션 |
| 구매 전환 가능 사용자 | 웨딩홀/허니문/답례품/ISA 링크 클릭 |

# 3. 페이지 포지셔닝

## 3.1 콘텐츠 성격
이 페이지는 뉴스성 글도 아니고, 계산기 단독 페이지도 아니다.

중간 포지션은 아래와 같다.

> **“비용 비교형 리포트 + 개인 상황 연결형 계산 유도 페이지”**

## 3.2 차별점

| 일반 블로그 글 | 비교계산소 방식 |
|---|---|
| 텍스트 위주 설명 | 숫자 비교 + 체감 계산 |
| 평균 비용만 나열 | 10년 변화와 부담률 강조 |
| 끝나면 이탈 | 계산기/제휴/관련 콘텐츠로 연결 |
| 공감 위주 | 의사결정 보조 위주 |

# 4. 핵심 메시지

## 4.1 메인 메시지

> 2016년에는 연봉의 70% 수준이던 결혼 비용이, 2026년에는 연봉 1년치를 넘기기 시작했다.

## 4.2 서브 메시지

- 결혼 비용 상승은 단순 물가 상승 이상이다.
- 특히 웨딩홀, 식대, 스드메 상승폭이 크다.
- 평균값만 보면 막막하지만, 항목별로 보면 줄일 수 있는 구간이 보인다.
- 실제 내 연봉 기준으로 체감하면 의사결정이 쉬워진다.

# 6. 섹션별 상세 기획

## 6.1 S1. 히어로

### 목표
첫 화면 3초 안에 사용자가 “이 콘텐츠가 내 상황과 관련 있다”고 느끼게 한다.

### 구성
- 배지: `리포트 1 · 결혼비용`
- 제목: `결혼비용 2016 vs 2026`
- 서브카피: `10년 만에 결혼 총비용은 얼마나 올랐나 — 연봉 1년치를 초과한 시대`
- 보조문구: 출처/기준 시점/추정 방식
- CTA 1: `내 예산 계산하기`
- CTA 2: `항목별 비교 보기`

### 실무 포인트
- 첫 화면에서 숫자 2개 이상 노출 필요
- CTA 1개만 두면 계산기 유입은 좋지만, 스크롤 유도는 약해질 수 있으므로 2개 권장

## 6.6 S6. 내 연봉 기준 체감 계산

### 목표
사용자가 “평균값”을 “내 문제”로 인식하게 만든다.

### 입력값

| 입력 항목 | 타입 | 기본값 |
|---|---|---|
| 연봉(만원) | number | 4000 |
| 월 저축액(선택) | number | 50 |
| 부모 지원금(선택) | number | 0 |

### 출력값

| 출력 항목 | 의미 |
|---|---|
| 2016 결혼비용 부담률 | 연봉 대비 몇 %인지 |
| 2026 결혼비용 부담률 | 현재 기준 부담률 |
| 추가 부담액 | 10년간 증가분 |
| 추가 저축 기간 | 월 저축액 기준 |
| 실질 필요 자기자금 | 부모 지원금 반영 |

### 실무 포인트
- 현재 HTML에는 연봉 기준만 있으므로, MD에서는 `월 저축액`과 `지원금`까지 확장 제안
- 이 섹션은 계산기 전환 전 예열 역할을 한다.


# 10. 수익화 전략

## 10.1 1차 수익화

| 방식 | 위치 | 설명 |
|---|---|---|
| 웨딩홀 견적 링크 | 하단 카드 | 실질 전환 가능성 높음 |
| 허니문 링크 | 하단 카드 | 결혼 콘텐츠와 직접 연결 |
| 쿠팡 답례품/청첩장 | 하단 카드 | 구매 장벽 낮음 |
| 결혼 자금 ISA/예적금 | 하단 카드 | 금융 전환 가능 |

## 10.2 2차 수익화
- 관련 계산기 이동 후 내부 체류시간 증가
- 다른 비교 리포트 시리즈 소비 유도
- 향후 메일 구독/리드 수집 가능성 검토
