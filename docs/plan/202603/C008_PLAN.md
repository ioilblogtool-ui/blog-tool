## 2. 페이지 목표

### 2.1 핵심 질문
- 결혼 준비 총예산은 대략 얼마가 드는가?
- 웨딩홀, 스드메, 신혼여행 중 어디서 가장 많이 드는가?
- 서울, 수도권, 지방에 따라 얼마나 차이가 나는가?
- 가성비 / 평균 / 고가 세트 기준으로 총액이 얼마나 달라지는가?
- 신랑과 신부는 각각 얼마를 부담하면 되는가?

### 2.2 사용자 행동 목표
- 사용자는 지역과 세트를 선택해 빠르게 총예산을 확인한다.
- 사용자는 세부 항목을 직접 수정해 본인 상황에 맞는 예산표를 만든다.
- 사용자는 신랑·신부 부담 비율을 조정한다.
- 사용자는 계산 결과를 URL로 공유한다.

### 2.3 비즈니스 목표
- “결혼 준비 예산 계산기”, “결혼식 비용 계산기”, “웨딩홀 비용 계산”, “스드메 비용 계산” 키워드 유입 확보
- 계산기 체류시간 증가
- URL 공유를 통한 자연 유입 확산
- 향후 혼수/예물/신혼집/결혼자금 계산기로 시리즈 확장

## S4. 웨딩홀 입력

### 목적
- 전체 예산에서 가장 큰 비중을 차지하는 영역 계산
- 식대를 신랑/신부측으로 나눠 현실적으로 계산

### 항목 구성
| 대분류 | 소분류 |
|---|---|
| 대관 | 웨딩홀 대관료 |
| 식대 | 신랑측 하객 식대, 신부측 하객 식대 |
| 꽃/장식 | 부케, 부토니아 |
| 양가 | 한복, 메이크업 |
| 기록 | 본식 스냅, 서브스냅, 식전 백그라운드 영상, 식중 백그라운드 영상 |
| 감사비 | 사회자, 축가 사례 |
| 추가비용 | 헬퍼, 포토테이블 장식, 플라워샤워, 웨딩카, 기타, 웨딩 컨시어지 |

### 계산식
```text
웨딩홀 식대 합계 = (신랑측 하객 수 × 신랑측 1인 식대) + (신부측 하객 수 × 신부측 1인 식대)

웨딩홀 총액 = 대관료 + 식대 합계 + 부케/부토니아 + 양가 비용 + 스냅/영상 + 감사비 + 추가비용
````

### UI 포인트

* 식대는 하객 수와 단가를 곱하는 방식
* 추가비용은 접기/펼치기 아코디언
* “평균값 채우기” 버튼 제공

## S8. 분담 계산

### 목적

* 사용자 실사용성 극대화
* 커플 간 조율 포인트를 숫자로 보여줌

### 분담 모드

| 모드       | 설명                        |
| -------- | ------------------------- |
| 반반       | 총예산 50:50                 |
| 비율 직접 입력 | 예: 55:45                  |
| 항목별 부담   | 하객 식대, 스드메, 신혼여행 등 항목별 분리 |

### 계산식

```text
반반 모드:
신랑 부담 = 총예산 × 50%
신부 부담 = 총예산 × 50%

비율 입력 모드:
신랑 부담 = 총예산 × 신랑 비율
신부 부담 = 총예산 × 신부 비율
```

### 항목별 부담 예시

* 신랑측 하객 식대 = 신랑 부담
* 신부측 하객 식대 = 신부 부담
* 웨딩밴드 = 공동 부담
* 스드메 = 신부 중심 또는 공동
* 신혼여행 = 반반

### 결과 예시

| 항목        |      금액 |
| --------- | ------: |
| 신랑 부담     | 1,850만원 |
| 신부 부담     | 1,650만원 |
| 신랑측 하객 식대 |   630만원 |
| 신부측 하객 식대 |   540만원 |

# 2) 지역 프리셋 실데이터

기획서의 `서울 / 수도권 / 지방` 3구조에 맞춰 매핑하면 아래가 가장 현실적입니다.

## `wedding_region_presets`

| regionCode | regionLabel | hallRentalBase | mealPriceBase | sdmBase | honeymoonBase | note                                  | updatedAt  |
| ---------- | ----------: | -------------: | ------------: | ------: | ------------: | ------------------------------------- | ---------- |
| seoul      |          서울 |        7000000 |         72000 | 3650000 |       7000000 | 서울(강남 외) 기준 중심. 대관+기본장식+식대 수준이 가장 현실적 | 2026-03-30 |
| metro      |         수도권 |        4100000 |         58000 | 3000000 |       6000000 | 경기도/수도권 평균 반영                         | 2026-03-30 |
| local      |          지방 |        2800000 |         50000 | 2600000 |       5000000 | 비수도권 평균 반영                            | 2026-03-30 |

## 왜 이 값으로 잡았나

### 서울

서울은 기획상 `강남 포함 평균`으로 잡으면 너무 높아져서 실사용자가 부담을 느낄 수 있습니다. 그래서 **서울 기본값은 “서울(강남 외)”**를 중심값으로 두는 게 더 적합합니다. 소비자원 참가격 기준 서울(강남 외) 평균 계약금액은 약 2,892만 원이며, 구성은 대관료 약 686만 원, 장식비 약 118만 원, 식비 약 1,723만 원, 스드메 약 365만 원입니다. 1인당 식대는 최근 조사에서 7.2만 원 수준입니다. ([가격정보시스템][1])

### 수도권

수도권은 참가격의 `경기도`와 `수도권 평균` 사이에서 잡는 게 자연스럽습니다. 경기도 평균 총액은 약 1,909만 원이며 대관료 약 407만 원, 식비 약 1,213만 원, 스드메 약 241만 원입니다. 수도권 전체 평균은 총 2,742만 원이지만 서울 포함이라 다소 높으므로, 비교계산소 기본 프리셋은 **경기권 체감가 기준**으로 두고 스드메를 약간 올린 300만 원으로 운영하는 편이 더 무난합니다. ([가격정보시스템][1])

### 지방

비수도권 평균 총 계약금액은 약 1,574만 원이며 대관료 약 263만 원, 식비 약 1,000만 원, 스드메 약 293만 원입니다. 다만 지방은 지역 편차가 커서 스드메를 너무 낮게 두면 사용자 체감과 어긋날 수 있어 260만 원 수준을 기본값으로 두는 것이 안정적입니다. ([가격정보시스템][1])

# 6) 운영 기본값 근거 요약

## 세부 항목에서 특히 많이 쓰는 값

* 웨딩밴드는 공개 시장 사례에서 커플 기준 **100만~200만 원대(실속형)**, 평균 **300만~400만 원대**가 반복적으로 보입니다. 그래서 기본값은 300만 원이 적절합니다. ([아크레도 코리아][4])
* 최근 실제 결혼비용 공개 사례에서는 본식스냅 65만 원, 아이폰스냅 22만 원, 본식 헬퍼 25만 원, 청첩장 44만 원, 웨딩밴드 235만 원 등이 확인됩니다. 이 값들을 너무 그대로 쓰지 않고 중앙값 쪽으로 보정해 운영 기본값을 잡았습니다. ([Blind][5])
* 스드메는 최근 공개 자료에서 평균 약 350만~450만 원, 저가형 약 270만 원, 프리미엄은 600만 원 이상 사례가 반복됩니다. ([갓생 말고 중생][6])

# 9) 실무 적용 체크리스트

## 추천 적용 방식

* 지역 기본값은 **서울=강남외 / 수도권=경기도 / 지방=비수도권 평균**
* 세트는 금액 고정값보다 **배수 방식** 사용
* 웨딩홀 식대는 **하객 수 × 식대 단가**로 별도 계산
* 스드메/허니문은 프리셋 선택 후 세부 옵션 가감
* 작은 항목은 `optional=true`로 두고 초기엔 일부만 노출

## 바로 구현할 때 주의

* `hallRentalBase`에는 **대관 중심값**만 두고, `hall_basic_decor`는 item catalog에서 더하는 게 UI/계산식이 깔끔합니다.
* 서울은 강남 포함 평균으로 잡으면 첫 화면 총액이 너무 높아질 수 있습니다.
* 허니문은 변동성이 커서 “기본값 + 여행지 프리셋” 구조가 필수입니다.
