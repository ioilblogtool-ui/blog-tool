````md
## 2. 페이지 목표

### 2.1 핵심 질문
- 보험사 중 평균 연봉이 높은 곳은 어디인가?
- 성과급이 강한 보험사는 어디인가?
- 연봉만 볼지, 총보상 기준으로 봐야 하는지?

- 사용자는 회사별 보상 수준을 빠르게 비교한다.

## S3. 보험사 비교 테이블

### 목적
- 페이지의 핵심 데이터 비교 영역
- 데스크탑에서는 표, 모바일에서는 카드로 전환

### 컬럼 설계
| 컬럼명 | 설명 |
|---|---|
| companyName | 회사명 |
| insuranceType | 생명보험 / 손해보험 |
| avgSalary | 평균 연봉 |
| newHireSalary | 신입 초봉 |
| managerSalary | 과장급 추정 연봉 |
| bonusRate | 성과급률 또는 성과급 수준 |
| estTotalComp | 추정 총보상 |
| employeeCount | 임직원 수 |
| companyTag | 한줄 특징 |
| hiringLink | 채용 바로가기 |
| sourceDate | 기준 시점 |

### 기본 표시 방식
- 숫자는 `만원` 단위 통일
- 정렬 가능
- `연봉순 / 성과급순 / 초봉순` 정렬 탭 제공
- 2개~3개 회사 선택 비교 기능

### 실무 예시
| 회사 | 구분 | 평균 연봉 | 신입 초봉 | 성과급 수준 | 추정 총보상 | 특징 |
|---|---:|---:|---:|---:|---:|---|
| 삼성화재 | 손보 | 14,000+ | 4,000+ | 높음 | 높음 | 업계 상위권 |
| 삼성생명 | 생보 | 13,000+ | 4,000+ | 높음 | 높음 | 생보 대표주 |
| 메리츠화재 | 손보 | 8,000~10,000+ | 3,000+ | 매우 높음 | 매우 높음 | 성과급 강세 |
| 현대해상 | 손보 | 10,000+ | 3,000+ | 중상 | 높음 | 전통 대형사 |
| DB손해보험 | 손보 | 9,000+ | 5,000+ | 중상 | 높음 | 최근 상승 주목 |
| KB손해보험 | 손보 | 8,000+ | 3,000+ | 중 | 중상 | 안정형 |

> 실제 운영 시 숫자는 데이터 시트 기준으로 관리

## S5. 총보상 계산기

### 목적
- 비교계산소의 핵심 UX 요소
- 단순 정보 제공을 넘어 사용자 참여 유도

### 입력값
| 입력 항목 | 타입 | 설명 |
|---|---|---|
| 회사 선택 | select | 보험사 선택 |
| 기본 연봉 | number | 본인 연봉 입력 |
| 성과급 시나리오 | select | 보수적 / 기준 / 공격적 |
| 기타 보너스 포함 | checkbox | 일시금 포함 여부 |

### 출력값
| 출력 항목 | 설명 |
|---|---|
| 예상 성과급 | 연봉 × 성과급률 |
| 예상 총보상 | 연봉 + 성과급 + 보너스 |
| 업계 평균 대비 | 상 / 중 / 하 |
| 간단 코멘트 | 회사별 보상 체계 해석 |

### 계산식
```text
예상 성과급 = 기본연봉 × 성과급률
예상 총보상 = 기본연봉 + 예상 성과급 + 기타 보너스
````

### 결과 메시지 예시

* “입력하신 조건 기준으로 업계 평균 대비 총보상 수준이 높은 편입니다.”
* “연봉은 중간 수준이지만 성과급 반영 시 총보상 경쟁력이 높아질 수 있습니다.”

### 체크포인트

* 계산 로직은 단순해야 함
* 지나치게 복잡한 세전/세후 계산은 1차 범위에서 제외
* 결과값은 추정치임을 명시

```text
→ 채용/공식 링크로 이동
```
