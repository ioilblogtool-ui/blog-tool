## 1. v3 → v4 전환 배경

### v3의 한계
- 타겟: "직군별 AI 툴 추천" → 블로그/유튜브와 동일한 정보 수준
- 차별화 없음: 고정 추천 테이블을 계산기처럼 포장한 구조
- SEO 경쟁: "ChatGPT Claude 비교" 키워드는 대형 사이트가 점령

### v4 핵심 전환
| 항목 | v3 | v4 |
|------|----|----|
| 타겟 | 직장인 일반 | 1인 빌더 / 사이드 프로젝트 운영자 |
| 질문 | "내 직군엔 뭐가 맞나?" | "단계별로 AI 쓰면 월 얼마지?" |
| 구조 축 | 직군 × 툴 | **업무 단계 × 예산** |
| 차별화 | 없음 | 공백 포지션 |

## 2. 서비스 정의

### 한 줄 정의
> 기획부터 배포까지, 단계별로 AI 툴을 조합하면 월 얼마인지 바로 계산해주는 1인 빌더용 비용 계산기

### 해결하는 질문
- 사이드 프로젝트 만들 때 AI에 월 얼마 써야 하지?
- 기획·분석·개발·배포 중 어느 단계에 AI 비용이 집중되나?
- 예산 $50이면 어떤 단계까지 AI로 커버할 수 있나?
- 지금 쓰는 조합, 과하게 지출하고 있는 건 아닌가?

### 포지셔닝
- **타겟**: 혼자 뭔가를 만들거나 운영하는 사람 (사이드 프로젝트, 1인 SaaS, 프리랜서, 소규모 스타트업 초기)
- **경쟁 공백**: "사이드프로젝트 AI 비용", "1인 개발 AI 스택" — 현재 전문 계산기 없음

## 3. 업무 단계 정의 (핵심 축)

v4는 직군이 아니라 **업무 단계**를 기준으로 설계합니다.

| 단계 | 설명 | 대표 AI 활용 |
|------|------|------------|
| 기획 | 아이디어 구체화, 시장 조사, 기능 정의 | 문서 초안, 리서치, 경쟁 분석 |
| 분석 | 데이터 해석, 사용자 인터뷰 정리, 수요 검증 | 데이터 요약, 인사이트 추출 |
| 개발 | 코드 생성, 리뷰, 디버깅, 테스트 | 코드 자동완성, 리팩토링 |
| 디자인 | 시안, 배너, UI 초안 | 빠른 시안, 카피 보조 |
| 배포 | 인프라, CI/CD, 문서화 | 설정 자동화, 문서 생성 |
| 운영 | 고객 응대, 콘텐츠, 모니터링 | 메일/공지 초안, 분석 보조 |

> 사용자는 자신이 **지금 어떤 단계에 집중하는지** 선택합니다.
> 전 단계 선택도 가능 (MVP 전체 흐름 시뮬레이션).

## 5. 입력값 정의

### 5-1. 필수 입력 (최대 4단계)

**Step 1. 어떤 단계에 집중하나요?** (multi-select)
- 기획 / 분석 / 개발 / 디자인 / 배포 / 운영
- 기본값: 기획 + 개발 (MVP 기준)

**Step 2. 사용 강도는?** (single-select)
- 가볍게 (월 10시간 미만)
- 보통 (월 10~40시간)
- 집중적으로 (월 40시간 이상)

**Step 3. 월 예산 범위는?** (single-select 또는 slider)
- $30 이하 / $30~$60 / $60~$100 / $100 이상
- 선택 시 "이 예산으로 커버 가능한 단계" 역산 표시

**Step 4. 지금 쓰고 있는 툴이 있나요?** (optional multi-select)
- 현재 구독 중인 툴 선택 → "이미 커버된 단계" 표시

### 5-2. 제거 항목 (v3 대비)
- 직군 선택 (단계 선택으로 대체)
- 주 사용 목적 (단계에 내포)
- API 사용 여부 (v4.1에서 옵션 추가 예정)

2. 선택 단계 수 3~4: 중복 역할 제거 후 최적 조합
3. 선택 단계 수 5~6: 프리셋 "풀 자동화 스택" 기본 제시
4. 예산 초과 시: 절약형 대안 자동 표시

## 10. 데이터 구조 (TS 파일 기준)

### 10-1. 파일 구조
```
src/data/
  tools.ts          # 툴 목록, 요금, 단계 커버리지
  stages.ts         # 업무 단계 정의
  bundles.ts        # 프리셋 스택
  faq.ts            # FAQ
  calculator.ts     # 추천 로직 함수
```

### 10-2. tools.ts 핵심 타입
```typescript
type Stage = 'planning' | 'analysis' | 'development' | 'design' | 'deployment' | 'operation';

interface Tool {
  id: string;
  name: string;
  monthlyPriceUsd: number;
  category: 'general' | 'dev' | 'research' | 'design' | 'docs';
  coverageStages: Stage[];       // 커버하는 단계
  primaryStage: Stage;           // 주력 단계
  tagline: string;               // 한 줄 설명
  isActive: boolean;
}
```

### 10-3. bundles.ts 핵심 타입
```typescript
interface Bundle {
  id: string;
  name: string;
  tools: string[];               // tool id array
  targetStages: Stage[];
  estimatedMonthlyCostUsd: number;
  type: 'budget' | 'balanced' | 'full';
  reason: string;
  alternativeBundleId: string;
}
```

## 13. 외부 참고 링크 기획

### 13-1. 목적
이 페이지는 단순 계산 결과만 보여주는 것이 아니라,  
사용자가 실제로 **공식 가격 / 공식 기능 / 공식 사용 가이드**까지 바로 확인할 수 있도록 돕는 구조로 설계합니다.

핵심 목적은 다음과 같습니다.

- 가격 정보의 신뢰도 보강
- 사용자의 다음 행동 연결
- SEO/정보성 품질 강화
- 계산 결과의 근거 보완

### 13-2. 링크 유형

| 링크 유형 | 예시 | 목적 |
|---|---|---|
| 공식 요금제 | ChatGPT, Claude, Cursor 가격 페이지 | 가격 기준 확인 |
| 공식 제품 소개 | ChatGPT 기능 소개, Claude 제품 소개, Cursor 기능 페이지 | 서비스 이해도 보강 |

### 13-3. 배치 위치
- 툴 비교표 내: 각 툴별 `공식 요금 보기`
- 결과 카드 하단: `이 툴 자세히 보기`
- FAQ 하단: `공식 안내 링크`
- 페이지 하단: 외부 참고 링크 모음

### 13-4. 운영 원칙
- 공식 사이트 링크를 우선 사용
- 가격 링크와 소개 링크를 구분
- 새 탭으로 열기
- 링크 변경 가능성을 고려해 주기적으로 점검
- 가격/기능 변경 가능성 안내 문구 포함

### 13-5. 안내 문구 예시
> 가격 및 기능 정보는 각 서비스의 공식 페이지를 기준으로 참고합니다.  
> 실제 과금 정책과 제공 기능은 시점에 따라 변경될 수 있습니다.

## 14. 콘텐츠 연계 제휴 상품 기획

### 14-1. 목적
계산기 사용자가 AI 툴 비용만 확인하고 끝나는 것이 아니라,  
해당 콘텐츠와 자연스럽게 연결되는 **실사용 작업 도구**도 함께 볼 수 있도록 설계합니다.

핵심은 광고성 배너가 아니라,  
**“이 작업을 할 때 함께 쓰는 도구”** 형태의 정보형 추천입니다.

### 14-2. 콘텐츠 연계 방향

| 콘텐츠/사용 맥락 | 추천 제휴 상품 예시 | 연결 목적 |
|---|---|---|
| AI 툴 비용 비교 | 노트북 거치대, 무선 키보드, 손목 받침대 | 장시간 작업 환경 개선 |
| 기획/문서 작업 중심 | 태블릿 거치대, 전자노트, 블루투스 키보드 | 문서 작성/아이디어 정리 효율화 |
| 개발/코딩 중심 | 기계식 키보드, 모니터 조명, 듀얼 모니터 액세서리 | 몰입도/생산성 향상 |
| 디자인/콘텐츠 제작 | 웹캠, USB 마이크, 외장 SSD | 제작/녹화/파일 관리 보조 |
| 1인 운영/배포 | 멀티탭, USB 허브, 외장 저장장치 | 작업 환경 안정화 |
| 재택/사이드프로젝트 | 독서대, 자세 교정 방석, 데스크 조명 | 장시간 개인 작업 보조 |

### 14-3. 노출 위치
- 계산 결과 하단
- 프리셋 스택 카드 하단
- FAQ 아래 보조 섹션
- 페이지 하단의 `함께 보면 좋은 작업 도구`

### 14-4. 노출 방식
- 한 번에 2~4개 정도만 노출
- 상품만 바로 보여주지 않고, 맥락 설명 후 카드 노출
- “추천 상품”보다 “함께 쓰는 작업 도구” 표현 사용
- 과도한 광고형 배너 대신 정보형 카드 구조 사용

### 14-5. 섹션 제목 예시
- 이 조합과 함께 많이 쓰는 작업 도구
- 1인 빌더 작업환경 추천
- 같이 보면 좋은 생산성 장비

### 14-6. 안내 문구 예시
> 일부 상품 링크에는 제휴 링크가 포함될 수 있습니다.  
> 가격 및 재고는 판매처 사정에 따라 달라질 수 있습니다.
