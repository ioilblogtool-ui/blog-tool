## 2. 페이지 목표

### 2.1 핵심 질문
- 대형 건설사 중 평균 연봉이 높은 곳은 어디인가?
- 성과급이 강한 건설사는 어디인가?
- 신입 초봉과 과장급 연봉은 어느 회사가 유리한가?
- 단순 연봉 외에 시공능력, 규모, 성장성까지 같이 봐야 하는가?

### 2.2 사용자 행동 목표
- 사용자는 회사별 보상 수준을 빠르게 비교한다.
- 사용자는 본인 연봉 기준 예상 총보상을 계산한다.
- 사용자는 관심 건설사 2~3곳을 비교한다.
- 사용자는 채용/공식/IR 링크로 추가 탐색한다.

### 2.3 비즈니스 목표
- “건설사 연봉”, “대형 건설사 성과급”, “건설사 초봉”, “건설사 연봉 순위” 계열 검색 유입 확보
- 비교형 UI를 통한 체류시간 증가
- 건설/엔지니어링/플랜트 업종 비교 시리즈 템플릿 확보
- 추후 EPC, 정유화학, 플랜트, 인프라 기업 비교 콘텐츠로 확장

## S5. 총보상 계산기

### 목적
- 비교계산소의 핵심 UX 요소
- 단순 정보 제공을 넘어 사용자 참여 유도

### 입력값
| 입력 항목 | 타입 | 설명 |
|---|---|---|
| 회사 선택 | select | 건설사 선택 |
| 기본 연봉 | number | 본인 연봉 입력 |
| 성과급 시나리오 | select | 보수적 / 기준 / 공격적 |
| 기타 상여 포함 | checkbox | 연말 상여, 프로젝트 인센티브 포함 여부 |

### 출력값
| 출력 항목 | 설명 |
|---|---|
| 예상 성과급 | 연봉 × 성과급률 |
| 예상 총보상 | 연봉 + 성과급 + 기타 상여 |
| 업계 평균 대비 | 상 / 중 / 하 |
| 간단 코멘트 | 회사별 보상 체계 해석 |

### 계산식
```text
예상 성과급 = 기본연봉 × 성과급률
예상 총보상 = 기본연봉 + 예상 성과급 + 기타 상여
````

### 결과 메시지 예시

* “입력하신 조건 기준으로 대형 건설사 평균 대비 총보상 수준이 높은 편입니다.”
* “기본 연봉은 중간 수준이지만 성과급 반영 시 총보상 경쟁력이 높아질 수 있습니다.”

### 체크포인트

* 계산 로직은 단순해야 함
* 세전/세후 계산은 v1 범위에서 제외
* 결과값은 추정치임을 명시

## 11. UX 원칙

### 우선 원칙

* 숫자는 한눈에 보여야 함
* 모바일에서 표가 깨지지 않아야 함

* 계산기는 입력이 3~4개 이내로 단순해야 함
* 건설업 특성상 “규모/브랜드/성장성”도 같이 보이도록 설계

* 카드형 섹션 분리
* 비교표는 sticky first column 고려
* 숫자는 우측 정렬
* 핵심 지표는 굵게
* 순위/상중하 태그로 시각적 스캔성 강화

## 17. 최종 요약

### 이 페이지의 핵심

* 건설사 연봉 비교 콘텐츠를 단순 블로그가 아닌 **비교형 계산 페이지**로 만든다.
* 사용자는 **연봉, 성과급, 총보상, 규모**를 한 번에 이해할 수 있어야 한다.
* 비교계산소 성격에 맞게 **데이터 + 계산 + 해설** 구조로 설계한다.

### v1 구현 기준

* 건설사 8개
* 평균 연봉 / 초봉 / 과장급 연봉 / 성과급 / 추정 총보상
* 비교표 + 랭킹 + 계산기 + FAQ
