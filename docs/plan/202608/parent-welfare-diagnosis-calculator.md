## 1. 페이지 목적

부모님(또는 본인)의 나이, 부부 여부, 소득, 국민연금, 금융재산, 부동산, 자동차, 거주지역, 장기요양등급 여부를 한 번 입력하면 **기초연금, 장기요양보험, 노인일자리, 기초생활보장, 주거급여, 고령자 임대주택, 각종 감면** 등 여러 복지 제도의 신청 가능성을 **한 화면에서 동시에** 보여주는 진단형 계산기다.

핵심 차별점은 개별 계산기(기초연금 계산기, 장기요양등급 계산기, 고령자 임대주택 계산기 등)를 하나씩 따로 들어가야 했던 기존 구조를 하나로 묶어, **"부모님 조건 한 번 입력 → 전체 복지 체크리스트"**로 제공하는 것이다. 이미 서비스 중인 개별 계산기들의 판정 로직과 데이터를 그대로 재사용해 이중 관리를 피한다.

> 이 계산기는 신청 전 자가 점검용 통합 진단이며, 실제 수급 여부는 각 제도별 공식 심사 결과에 따라 달라질 수 있다.

## 3. 데이터 기준

이 계산기는 새로운 기준을 만들지 않고, **이미 서비스 중인 계산기의 데이터/로직을 그대로 재사용**한다.

| 연계 제도 | 기존 계산기(재사용 대상) | 기준 |
|---|---|---|
| 기초연금 | `basicPensionEligibilityCalculator.ts` | 2026년 보건복지부 선정기준액, 소득인정액 산식(재산 소득환산율 4%, 금융재산 공제 2,000만 원, 지역별 기본재산공제) |
| 장기요양등급 | `ltci-grade-benefit-calculator-2026` (데이터 파일 확인 필요) | 등급별 급여 한도·본인부담률 |
| 고령자 임대주택 | `senior-rental-housing-eligibility-calculator-2026` | 나이·주택보유·소득수준·거주기간·수급자여부 기준 |
| 노인일자리 | `senior-job-salary-calculator-2026`, `senior-job-comparison-2026` | 유형별 급여·근무시간 |
| 기초생활보장(생계·의료·주거·교육급여) | `welfare-benefit-eligibility`, `livelihood-benefit-income-recognition`, `housing-benefit-income-recognition` | 2026년 기준 중위소득 대비 소득인정액 |
| 가족돌봄수당 | `familyCareAllowanceCalculator.ts` | 아이 돌봄 중심이라 부모님 케이스에서는 참고용으로만 링크 |

> 데이터 재사용 원칙: 이 계산기는 **자체 판정 상수를 새로 정의하지 않고**, 위 계산기들이 이미 검증해 배포 중인 상수·산식을 import해서 재사용한다. 각 제도의 최신 기준액(2026년 기준 중위소득 6,494,738원/4인 가구 등)은 기존 데이터 파일에서 그대로 가져온다.

### 3-1. 확인 필요 항목

- `ltci-grade-benefit-calculator-2026`, `senior-job-salary-calculator-2026` 등의 실제 데이터 파일 경로와 export하는 판정 함수/상수명을 구현 착수 전 코드베이스에서 재확인한다(현재 `src/data/` 디렉터리에 슬러그와 동일한 파일명이 없을 수 있어 실제 파일명 매핑이 필요).
- 노인일자리·감면 제도는 지자체별 편차가 커서 "전국 공통 기준"과 "지자체 확인 필요" 항목을 명확히 분리해야 한다.

## 5. 출력값 설계

전체를 하나의 **체크리스트 매트릭스**로 표시한다.

| 제도 | 판정 | 근거 요약 |
|---|---|---|
| 기초연금 | 가능성 높음 / 경계 / 어려움 | 소득인정액 vs 선정기준액 |
| 장기요양보험 | 등급 보유 시 자동 표시 / 미보유 시 "등급 신청 안내" | 등급별 급여한도 요약 |
| 노인일자리 | 나이 기준 충족 여부 | 65세 이상 기본 대상 |
| 기초생활보장(생계·의료·주거·교육) | 급여별 가능성 | 기준 중위소득 대비 소득인정액 |
| 고령자 임대주택 | 당첨 가능성 점수 | 나이·주택보유·소득·거주기간 |
| 교통비·통신비·에너지 감면 | 기초연금·수급자 여부 연동 자동 판정 | 기초연금 수급 시 자동 대상인 경우 많음 |

각 행 클릭 시 해당 세부 계산기로 이동하는 "자세히 계산하기" 링크를 제공한다(딥 계산은 기존 개별 계산기에 위임).

## 6. 계산 로직

이 계산기는 **자체 계산 엔진을 새로 만들지 않는다**. 대신:

```text
1. 공통 입력값(나이/소득/재산/지역)을 받는다.
2. 각 제도별 기존 계산기의 판정 함수를 호출하듯 동일한 산식을 인라인으로 재사용한다.
   - 기초연금: 소득인정액 = 소득평가액 + 재산의 소득환산액 (basicPensionEligibilityCalculator 로직)
   - 기초생활보장: 소득인정액 vs 기준 중위소득 32/40/48/50% (welfare-benefit-eligibility 로직)
   - 임대주택: 나이/주택보유/소득/거주기간 가중 점수 (senior-rental-housing-eligibility 로직)
3. 제도별 판정 결과를 하나의 배열로 합쳐 매트릭스에 렌더링한다.
```

각 제도 판정에 필요한 입력이 공통 Step 1~3에서 커버되지 않는 경우(예: 거주기간, 세부 서류 조건)는 "간단 확인 불가 — 상세 계산기에서 확인" 배지로 처리한다.

## 11. FAQ 예시

### Q1. 여러 복지를 동시에 받을 수 있나요?
제도별로 다릅니다. 기초연금과 노인일자리처럼 동시에 받을 수 있는 조합이 있는 반면, 기초생활보장 생계급여처럼 다른 급여와 함께 조정되는 제도도 있습니다.

### Q2. 이 진단 결과로 바로 신청하면 되나요?
아닙니다. 이 계산기는 자가 점검용 통합 진단이며, 실제 신청은 복지로 또는 주소지 주민센터·국민연금공단에서 정식 심사를 거쳐야 합니다.

### Q3. 왜 "확인 필요"로 나오는 항목이 있나요?
거주기간, 지자체별 조례, 서류 조건처럼 이 계산기의 공통 입력값만으로는 판단하기 어려운 조건이 있는 제도는 "확인 필요"로 표시하고 상세 계산기로 연결합니다.

### Q4. 자동차가 있으면 불리한가요?
기초연금·기초생활보장 등 일부 제도에서는 자동차 가액이 재산에 반영될 수 있습니다. 정확한 반영 방식은 각 제도 상세 계산기에서 확인하세요.

### Q5. 장기요양등급이 없으면 손해인가요?
장기요양등급이 없어도 기초연금, 노인일자리 등 다른 복지는 별도로 신청할 수 있습니다. 등급이 필요할 것 같다면 등급 신청 절차를 먼저 확인하는 것을 권장합니다.

## 13. 구현 주의사항

- 개별 제도 판정 상수는 반드시 기존 계산기 데이터 파일을 import해서 재사용하고, 별도로 복제하지 않는다(개정 시 이중 수정 방지).
- "가능성 높음/경계/확인 필요/어려움" 4단계 라벨을 모든 제도에 통일 적용해 사용자가 매트릭스를 한눈에 읽을 수 있게 한다.
- 결과 화면 상단에 "이 진단은 여러 제도를 한 번에 훑어보는 자가 점검용이며, 정확한 심사는 각 기관에서 진행된다"는 안내를 고정 배치한다.
- 지자체별로 크게 갈리는 항목(노인일자리 유형, 감면 폭)은 "전국 공통 vs 지자체 확인 필요"를 명확히 구분 표시한다.

## 14. 최종 기획 요약

기존에 개별로 흩어져 있던 노인 복지 계산기 6~7종을 **하나의 입력 흐름으로 묶어주는 허브형 진단 계산기**다. 새로운 판정 로직을 만드는 대신 이미 검증된 개별 계산기의 데이터를 재사용하므로 구현 리스크가 낮고, 완성 시 개별 계산기들의 상호 유입도 함께 늘어나는 효과가 있다.

차별화 포인트는 두 가지다.

1. "부모님 조건 한 번 입력 → 전체 복지 매트릭스"라는, 현재 어떤 개별 계산기도 제공하지 않는 통합 경험.
2. 판정이 애매한 제도는 숨기지 않고 "확인 필요"로 투명하게 표시해 신뢰도를 확보.
