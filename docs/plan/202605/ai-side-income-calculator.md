## 6. 계산 공식

### 6-1. 월 예상 매출

```text
월 예상 매출 = 건당 단가 × 월 예상 작업 건수
```

### 6-2. 월 순수익

```text
월 순수익 = 월 예상 매출 - AI 도구 월 구독비
```

### 6-3. 월 투입 시간

```text
월 투입 시간 = 주당 투입 시간 × 4.345
```

또는 작업 건수 기준으로 계산할 경우:

```text
월 투입 시간 = 건당 소요 시간 × 월 예상 작업 건수
```

### 6-4. 시간당 실질 수익

```text
시간당 실질 수익 = 월 순수익 ÷ 월 투입 시간
```

### 6-5. 손익분기 작업 수

```text
손익분기 작업 수 = AI 도구 월 구독비 ÷ 건당 단가
```

소수점은 올림 처리합니다.

```text
예: 구독비 29,000원 ÷ 건당 단가 10,000원 = 2.9건
→ 최소 3건 필요
```

### 6-6. 연간 예상 순수익

```text
연간 예상 순수익 = 월 순수익 × 12
```

## 10. 본문 섹션 구성

## 10-1. AI 부업 수익 계산기는 무엇인가?

AI 부업 수익 계산기는 ChatGPT, Claude, Midjourney, Perplexity, Canva AI 등 유료 AI 도구를 사용해 부업을 할 때 실제로 남는 수익을 계산하는 도구입니다.

단순히 “월 매출이 얼마인가”가 아니라, 다음 요소를 함께 계산합니다.

| 고려 요소     | 이유                 |
| --------- | ------------------ |
| AI 도구 구독비 | 매달 고정비로 빠지는 비용     |
| 투입 시간     | 실제 노동 시간 대비 수익성 확인 |
| 건당 단가     | 부업의 수익 구조 결정       |
| 작업 건수     | 월 매출 규모 결정         |
| 시간당 수익    | 최저시급·본업 시급과 비교 가능  |

---

## 10-2. AI 부업은 매출보다 순수익이 중요하다

AI 부업을 시작하면 “월 30만 원 벌었다”는 식으로 매출만 보기 쉽습니다. 하지만 실제 판단 기준은 순수익입니다.

예를 들어 월 30만 원을 벌었더라도 AI 도구 구독비, 이미지 생성 비용, 자동화 툴 비용, 플랫폼 수수료 등을 제외하면 실제 남는 금액은 줄어듭니다.

```text
월 매출 300,000원
- AI 도구비 29,000원
= 월 순수익 271,000원
```

따라서 AI 부업은 반드시 **매출, 비용, 시간, 순수익**을 함께 봐야 합니다.

---

## 10-3. 손익분기점 계산이 중요한 이유

AI 도구를 유료로 구독하고 있다면 가장 먼저 확인해야 할 것은 “몇 건만 해도 본전인가?”입니다.

예를 들어 ChatGPT Plus와 Claude Pro를 함께 사용해 월 6만 원 정도의 도구비가 발생한다고 가정해보겠습니다.

|    건당 단가 | 본전까지 필요한 작업 수 |
| -------: | ------------: |
|  10,000원 |            6건 |
|  20,000원 |            3건 |
|  30,000원 |            2건 |
|  50,000원 |            2건 |
| 100,000원 |            1건 |

이처럼 단가가 높아질수록 AI 도구비 회수는 쉬워집니다.

## 10-5. 개발자에게 가장 유리한 AI 부업 유형

개발자라면 단순 블로그 대필보다 **자동화 외주, 업무 효율화 세팅, 사내 문서 자동화, 크롤링/리포트 자동화** 쪽이 수익성이 높을 수 있습니다.

| 개발자형 AI 부업    | 예시                    |
| ------------- | --------------------- |
| n8n 자동화 구축    | 구글시트 → 이메일 → 슬랙 자동 발송 |
| 업무 리포트 자동화    | 매출 데이터 자동 요약 리포트      |
| 사내 문서 템플릿 자동화 | 회의록, 기획서, 보고서 자동 생성   |
| 간단한 챗봇 제작     | FAQ 챗봇, 상담 응답 봇       |
| API 연동 자동화    | 외부 서비스 데이터 수집/정리      |

특히 개발자는 AI를 단순 사용자가 아니라 **자동화 시스템 구축 도구**로 활용할 수 있기 때문에 시간당 수익을 높이기 좋습니다.

## 17. 개발 구현용 계산 로직 예시

```ts
type SideIncomeInput = {
  monthlyToolCost: number;
  weeklyHours: number;
  pricePerTask: number;
  monthlyTaskCount: number;
  hoursPerTask: number;
};

type SideIncomeResult = {
  monthlyRevenue: number;
  monthlyNetProfit: number;
  monthlyHoursByWeek: number;
  monthlyHoursByTask: number;
  effectiveHourlyRate: number;
  breakEvenTaskCount: number;
  annualNetProfit: number;
};

export function calculateAISideIncome(input: SideIncomeInput): SideIncomeResult {
  const monthlyRevenue = input.pricePerTask * input.monthlyTaskCount;
  const monthlyNetProfit = monthlyRevenue - input.monthlyToolCost;

  const monthlyHoursByWeek = input.weeklyHours * 4.345;
  const monthlyHoursByTask = input.hoursPerTask * input.monthlyTaskCount;

  const baseHours = monthlyHoursByTask > 0 ? monthlyHoursByTask : monthlyHoursByWeek;

  const effectiveHourlyRate =
    baseHours > 0 ? monthlyNetProfit / baseHours : 0;

  const breakEvenTaskCount =
    input.pricePerTask > 0
      ? Math.ceil(input.monthlyToolCost / input.pricePerTask)
      : 0;

  const annualNetProfit = monthlyNetProfit * 12;

  return {
    monthlyRevenue,
    monthlyNetProfit,
    monthlyHoursByWeek,
    monthlyHoursByTask,
    effectiveHourlyRate,
    breakEvenTaskCount,
    annualNetProfit,
  };
}
```
