## 1. 페이지 목적

병원 진료비 영수증의 `급여 본인부담금`, `비급여 항목별 금액`, `입원/통원 구분`, `실손보험 세대`, `자기부담률`을 입력하면 실손보험으로 청구 가능한 금액과 공제 후 예상 환급액을 계산한다.

이 계산기의 핵심은 보험금을 확정하는 것이 아니라, **청구 전 사용자가 대략 얼마를 돌려받을 수 있는지**와 **청구하면 연간 한도·4세대 비급여 누적액에 어떤 영향이 있는지**를 한 화면에서 보여주는 것이다.

---

## 2. 콘텐츠 한 줄 정의

> 병원 영수증의 급여·비급여 금액을 입력하면 실손보험 세대별 자기부담금과 통원 공제를 반영해 예상 환급액과 남은 연간 한도를 계산하는 도구

## 4. 기획상 중요한 기준

실손보험은 가입 시기, 보험회사, 특약 가입 여부, 표준화 이전 상품 여부에 따라 약관이 다르다. 따라서 v1에서는 다음 원칙을 적용한다.

| 원칙 | 설명 |
| --- | --- |
| 예상 계산 중심 | 실제 지급 보험금 확정 표현 금지 |
| 세대별 기본값 제공 | 1~4세대 선택 시 대표 자기부담률 자동 입력 |
| 직접 수정 허용 | 사용자가 약관상 자기부담률·회당 한도를 직접 바꿀 수 있게 설계 |
| 3대 비급여 별도 처리 | 도수치료·체외충격파·증식치료, 비급여 주사, MRI/MRA는 별도 한도/횟수 안내 |
| 4세대 누적액 안내 | 4세대는 비급여 보험금 누적액이 갱신 보험료에 영향을 줄 수 있음을 표시 |

손해보험협회 안내에 따르면 4세대 실손의료보험은 급여와 비급여를 구분하고, 급여는 보장대상의료비에서 자기부담금을 제외한 금액을 보상하는 구조다. 4세대 비급여 통원은 회당 한도와 최소 자기부담금이 별도로 적용된다. ([손해보험협회][1])

## 8. 계산 로직

### 8-1. 기본 계산식

```text
급여 환급 예상액 = 급여 본인부담금 - 급여 자기부담금
비급여 환급 예상액 = 비급여 보장대상 금액 - 비급여 자기부담금

자기부담금 = max(보장대상 금액 × 자기부담률, 통원 최소 공제금액)

통원 최종 환급액 = min(급여/비급여 환급 예상액 합계, 통원 회당 한도, 연간 잔여 한도)
입원 최종 환급액 = min(급여/비급여 환급 예상액 합계, 연간 잔여 한도)
```

### 8-2. 3대 비급여 계산

```text
도수치료 환급액 = min(도수치료 금액 - max(도수치료 금액 × 30%, 최소 공제), 도수치료 잔여 한도)
비급여 주사 환급액 = min(주사료 - 주사료 × 30%, 주사료 잔여 한도)
MRI/MRA 환급액 = min(MRI/MRA 금액 - MRI/MRA 금액 × 30%, MRI/MRA 잔여 한도)
```

3대 비급여는 3세대 이후 특약 가입 여부가 중요하다. v1에서는 특약 미가입 시 해당 항목 환급액을 0원으로 표시하고, "특약 가입 여부 확인 필요" 안내를 노출한다.

### 8-3. 4세대 비급여 누적액 안내

```text
이번 청구 후 비급여 보험금 = 기존 올해 비급여 보험금 누적액 + 이번 비급여 예상 환급액
```

| 누적 비급여 보험금 | 안내 문구 |
| ---: | --- |
| 0원 | 비급여 보험금 미수령 구간입니다. 갱신 시 할인 대상 가능성이 있습니다. |
| 100만 원 미만 | 유지 구간으로 안내하되 보험회사 조회 필요 |
| 100만 원 이상~150만 원 미만 | 할증 가능 구간 진입 주의 |
| 150만 원 이상~300만 원 미만 | 높은 할증 가능 구간 |
| 300만 원 이상 | 최대 할증 가능 구간 |

4세대 실손보험의 비급여 보험료 차등제는 2024년 7월 1일부터 적용되며, 비급여 보험금 수령액에 따라 갱신 보험료가 할인 또는 할증될 수 있다. 보도자료 기반 기사에서는 비급여 보험금 미수령자는 할인 가능, 100만 원 이상 수령자는 할증 구간에 들어갈 수 있다고 설명한다. ([파이낸셜뉴스/뉴시스][2])

### 섹션 2. 계산기 본체

| 영역 | 구성 |
| --- | --- |
| Step 1 | 통원/입원, 실손보험 세대, 의료기관 유형 |

## 14. FAQ 예시

### Q1. 이 계산기의 결과가 실제 보험금과 같나요?

아니요. 이 계산기는 입력한 금액과 세대별 대표 자기부담률을 바탕으로 한 예상 계산입니다. 실제 보험금은 가입한 보험회사, 약관, 특약, 면책사항, 병원 서류 심사 결과에 따라 달라질 수 있습니다.

### Q2. 병원 영수증에서 공단부담금도 입력해야 하나요?

아니요. 실손보험은 일반적으로 환자가 실제 부담한 금액을 기준으로 계산합니다. 공단부담금은 국민건강보험공단이 부담한 금액이므로 계산기 입력 대상에서 제외합니다.

### Q3. 통원 진료비가 적으면 환급액이 0원이 될 수도 있나요?

네. 통원은 자기부담률뿐 아니라 최소 공제금액이 적용될 수 있습니다. 병원비가 공제금액보다 작거나 비슷하면 실제 환급액이 없거나 매우 적을 수 있습니다.

### Q4. 도수치료와 MRI는 왜 별도로 입력하나요?

3세대 이후 실손보험에서는 도수치료·체외충격파·증식치료, 비급여 주사료, MRI/MRA가 3대 비급여 특약으로 분리되어 한도와 자기부담률이 다르게 적용될 수 있기 때문입니다.

### Q5. 4세대 실손은 비급여를 청구하면 보험료가 오르나요?

4세대 실손보험은 비급여 보험금 수령액에 따라 갱신 시 비급여 보험료가 할인 또는 할증될 수 있습니다. 다만 실제 적용 여부와 금액은 보험회사 조회시스템과 약관 기준을 확인해야 합니다.

### Q6. 1세대 실손도 계산할 수 있나요?

가능하지만 1세대 실손은 보험회사별 약관 차이가 큽니다. 계산기에서는 대표 기본값을 제공하되, 사용자가 자기부담률과 한도를 직접 수정하도록 설계하는 것이 안전합니다.

## 17. 개발 구현 메모

### 17-1. 데이터 타입 예시

```ts
type InsuranceGeneration = 'gen1' | 'gen2' | 'gen3' | 'gen4' | 'custom';
type VisitType = 'outpatient' | 'inpatient';

type SilsonRefundInput = {
  generation: InsuranceGeneration;
  visitType: VisitType;
  hospitalType: 'clinic' | 'general' | 'custom';
  coveredPatientPaid: number;
  nonCoveredGeneral: number;
  manualTherapy: number;
  nonCoveredInjection: number;
  mriMra: number;
  prescriptionCost: number;
  coveredCoinsuranceRate: number;
  nonCoveredCoinsuranceRate: number;
  specialNonCoveredCoinsuranceRate: number;
  outpatientDeductible: number;
  outpatientLimit: number;
  annualLimit: number;
  claimedThisYear: number;
  nonCoveredBenefitThisYear: number;
  hasSpecialNonCoveredRider: boolean;
};

type SilsonRefundResult = {
  totalPaid: number;
  eligibleAmount: number;
  deductibleAmount: number;
  estimatedRefund: number;
  remainingAnnualLimit: number;
  nonCoveredBenefitAfterClaim: number;
  warningMessages: string[];
};
```

### 17-2. 계산 함수 예시

```ts
function calculateRefund(input: SilsonRefundInput): SilsonRefundResult {
  const specialAmount =
    input.hasSpecialNonCoveredRider
      ? input.manualTherapy + input.nonCoveredInjection + input.mriMra
      : 0;

  const generalNonCovered = input.nonCoveredGeneral;
  const coveredRefundBase =
    input.coveredPatientPaid * (1 - input.coveredCoinsuranceRate);
  const nonCoveredRefundBase =
    generalNonCovered * (1 - input.nonCoveredCoinsuranceRate);
  const specialRefundBase =
    specialAmount * (1 - input.specialNonCoveredCoinsuranceRate);

  const rawRefund =
    coveredRefundBase + nonCoveredRefundBase + specialRefundBase + input.prescriptionCost;

  const outpatientCapped =
    input.visitType === 'outpatient'
      ? Math.min(Math.max(rawRefund - input.outpatientDeductible, 0), input.outpatientLimit)
      : rawRefund;

  const remainingLimit = Math.max(input.annualLimit - input.claimedThisYear, 0);
  const estimatedRefund = Math.min(outpatientCapped, remainingLimit);

  return {
    totalPaid:
      input.coveredPatientPaid +
      input.nonCoveredGeneral +
      input.manualTherapy +
      input.nonCoveredInjection +
      input.mriMra +
      input.prescriptionCost,
    eligibleAmount: input.coveredPatientPaid + generalNonCovered + specialAmount,
    deductibleAmount: Math.max(rawRefund - estimatedRefund, 0),
    estimatedRefund,
    remainingAnnualLimit: Math.max(remainingLimit - estimatedRefund, 0),
    nonCoveredBenefitAfterClaim:
      input.nonCoveredBenefitThisYear + nonCoveredRefundBase + specialRefundBase,
    warningMessages: [],
  };
}
```

> 구현 시 위 함수는 설명용 초안이다. 실제 로직에서는 급여/비급여별 공제 순서, 통원 최소 공제 적용 방식, 처방조제비 한도, 3대 비급여 한도·횟수를 별도 함수로 분리한다.
