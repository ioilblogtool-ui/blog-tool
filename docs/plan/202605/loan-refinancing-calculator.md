## 1. 페이지 목적

현재 대출의 잔액, 금리, 남은 기간, 월 납입금과 신규 대출의 금리, 기간, 중도상환수수료, 부대비용을 입력하면 대출 갈아타기 시 예상 손익을 계산한다.

이 계산기의 핵심은 단순히 "신규 금리가 낮다"를 보여주는 것이 아니라, **중도상환수수료와 신규 대출 부대비용을 회수하려면 몇 개월이 필요한지**를 보여주는 것이다. 사용자는 월 납입금 절감액, 총 이자 절감액, 순절감액, 손익분기 시점을 한눈에 보고 갈아타기 여부를 판단할 수 있다.

---

## 2. 콘텐츠 한 줄 정의

> 기존 대출과 신규 대출 조건을 비교해 월 납입금 절감액, 총 이자 절감액, 중도상환수수료 회수 기간을 계산하는 대환대출 손익 계산기

## 3. 콘텐츠 포지셔닝

| 일반 금리 비교 글 | 비교계산소 계산기 방향 |
| --- | --- |
| 은행별 최저금리 나열 | 내 대출 잔액 기준 실제 절감액 계산 |
| 금리가 낮으면 갈아타기 권유 | 중도상환수수료와 부대비용 차감 후 판단 |
| 월 이자만 비교 | 월 납입금, 총 이자, 총비용, 손익분기점 동시 계산 |
| 주담대 중심 | 주담대·전세대출·신용대출 공통 구조 |
| 정답형 추천 | 조건부 판단: 유리 / 보류 / 추가 확인 |

대환대출은 금리 차이만으로 결정하면 안 된다. 실제 손익은 `기존 대출 총비용 - 신규 대출 총비용 - 중도상환수수료 - 신규 대출 부대비용`으로 계산해야 한다.

## 7. 계산 로직

### 7-1. 원리금균등 월 납입금

```text
월이자율 = 연이자율 ÷ 12
개월 수 = 남은 기간(년) × 12

월 납입금 =
  대출잔액 × 월이자율 × (1 + 월이자율)^개월 수
  ÷ ((1 + 월이자율)^개월 수 - 1)
```

### 7-2. 원리금균등 총이자

```text
총 상환액 = 월 납입금 × 개월 수
총이자 = 총 상환액 - 대출잔액
```

### 7-3. 원금균등 총비용

```text
월 원금상환액 = 대출잔액 ÷ 개월 수
매월 이자 = 남은 원금 × 월이자율
월 납입금 = 월 원금상환액 + 매월 이자
총이자 = 매월 이자의 합계
```

### 7-4. 만기일시 총비용

```text
월 이자 = 대출잔액 × 월이자율
총이자 = 월 이자 × 개월 수
월 납입금 = 월 이자
```

### 7-5. 중도상환수수료

```text
수수료 대상 금액 = max(대출잔액 - 수수료 면제 금액, 0)
잔여 부과기간 비율 = 수수료 부과 남은 기간 ÷ 전체 수수료 부과 기간

중도상환수수료 =
  수수료 대상 금액 × 중도상환수수료율 × 잔여 부과기간 비율
```

2025년 1월 13일부터 신규 대출은 중도상환수수료를 실제 발생 비용 범위 안에서 부과하도록 제도가 개편되었다. 기존 대출 실행일, 상품 약관, 금융회사별 기준에 따라 실제 수수료가 달라질 수 있으므로 결과에는 반드시 약관 확인 안내를 붙인다. ([금융위원회][1])

### 7-6. 갈아타기 손익

```text
총 갈아타기 비용 = 중도상환수수료 + 신규 대출 부대비용
총 이자 절감액 = 기존 대출 잔여 총이자 - 신규 대출 총이자
갈아타기 순절감액 = 총 이자 절감액 - 총 갈아타기 비용
월 납입금 절감액 = 기존 월 납입금 - 신규 월 납입금
수수료 회수 기간 = 총 갈아타기 비용 ÷ max(월 납입금 절감액, 1)
```

### 7-7. 판단 라벨

| 조건 | 라벨 | 메시지 |
| --- | --- | --- |
| 순절감액 > 0 AND 회수 기간 ≤ 12개월 | 지금 유리 | 비용을 1년 안에 회수할 수 있습니다. |
| 순절감액 > 0 AND 회수 기간 ≤ 36개월 | 조건부 유리 | 장기 유지할 계획이면 유리할 수 있습니다. |
| 순절감액 > 0 AND 회수 기간 > 36개월 | 보류 | 회수 기간이 길어 금리 변동과 이사 계획을 확인해야 합니다. |
| 순절감액 ≤ 0 | 불리 | 수수료와 부대비용을 빼면 현재 조건 유지가 나을 수 있습니다. |

## 8. 고정금리 vs 변동금리 시나리오

### 시나리오 구조

| 시나리오 | 신규 금리 반영 | 설명 |
| --- | --- | --- |
| 기준 | 입력한 신규 금리 | 현재 제안받은 금리 기준 |
| 금리 하락 | 신규 금리 - 하락 가정값 | 변동금리 선택 시 유리한 경우 |
| 금리 상승 | 신규 금리 + 상승 가정값 | 변동금리 리스크 확인 |

### 출력

| 시나리오 | 신규 월 납입금 | 총이자 | 순절감액 | 판단 |
| --- | ---: | ---: | ---: | --- |
| 고정금리 유지 | 계산값 | 계산값 | 계산값 | 안정성 |
| 변동금리 하락 | 계산값 | 계산값 | 계산값 | 낙관 |
| 변동금리 상승 | 계산값 | 계산값 | 계산값 | 위험 |

결과 메시지는 "변동금리가 무조건 유리"가 아니라 "금리 상승 시 절감액이 사라지는 구간"을 보여주는 방식으로 작성한다.

## 14. FAQ 예시

### Q1. 금리가 얼마나 낮아져야 갈아타는 게 유리한가요?

정답은 대출 잔액, 남은 기간, 중도상환수수료, 신규 대출 부대비용에 따라 달라집니다. 대출 잔액이 크고 남은 기간이 길수록 작은 금리 차이도 절감액이 커질 수 있습니다.

### Q2. 월 납입금이 줄면 무조건 이득인가요?

아닙니다. 신규 대출 기간을 길게 늘리면 월 납입금은 줄어도 총이자는 오히려 늘어날 수 있습니다. 이 계산기는 월 납입금과 총이자를 함께 비교합니다.

### Q3. 중도상환수수료가 있으면 갈아타지 않는 게 낫나요?

꼭 그렇지는 않습니다. 월 납입금 절감액이나 총 이자 절감액이 수수료보다 크고, 수수료 회수 기간 이후에도 대출을 유지할 계획이라면 갈아타기가 유리할 수 있습니다.

### Q4. 고정금리에서 변동금리로 갈아타도 괜찮을까요?

변동금리는 초기 금리가 낮을 수 있지만 향후 금리 상승 위험이 있습니다. 기준 시나리오뿐 아니라 금리 상승 시나리오에서도 순절감액이 남는지 확인해야 합니다.

### Q5. 대출 갈아타기 계산 결과가 실제 은행 승인 결과와 같나요?

아니요. 이 계산기는 비용과 이자 절감액을 추정하는 도구입니다. 실제 승인 여부, 금리, 한도는 DSR, LTV, 신용점수, 소득, 담보가치, 은행 심사 기준에 따라 달라질 수 있습니다.

### Q6. 전세대출도 이 계산기로 비교할 수 있나요?

가능합니다. 전세대출은 만기일시 상환 구조가 많으므로 상환 방식을 만기일시로 선택하면 월 이자 절감액 중심으로 비교할 수 있습니다. 다만 보증료와 보증기관 조건도 함께 확인해야 합니다.

### 17-2. 계산 함수 예시

```ts
function calculateMonthlyPayment(
  principal: number,
  annualRate: number,
  months: number,
  repaymentType: RepaymentType
) {
  const monthlyRate = annualRate / 12;

  if (repaymentType === 'bullet') {
    return principal * monthlyRate;
  }

  if (repaymentType === 'annuity') {
    if (monthlyRate === 0) return principal / months;
    return (
      principal *
      monthlyRate *
      Math.pow(1 + monthlyRate, months) /
      (Math.pow(1 + monthlyRate, months) - 1)
    );
  }

  const firstMonthPrincipal = principal / months;
  return firstMonthPrincipal + principal * monthlyRate;
}

function calculatePrepaymentPenalty(input: LoanRefinancingInput) {
  const penaltyBase = Math.max(input.currentBalance - input.penaltyExemptAmount, 0);
  const remainingRatio =
    input.penaltyTotalMonths > 0
      ? Math.min(input.penaltyRemainingMonths / input.penaltyTotalMonths, 1)
      : 0;

  return penaltyBase * input.prepaymentPenaltyRate * remainingRatio;
}
```

> 구현 시 금리는 `4.8%`가 아니라 `0.048` 형태로 내부 계산한다. 화면 입력/출력에서만 `%` 포맷을 적용한다.
