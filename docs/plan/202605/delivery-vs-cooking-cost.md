## 1. 페이지 목적

사용자가 배달음식과 직접 요리의 실제 비용 차이를 빠르게 계산하도록 돕는 생활비 절약형 계산기다.

단순히 음식값만 비교하지 않고 `배달 음식값 + 배달비 + 팁 + 최소주문/쿠폰 효과`와 `식재료비 + 조리 에너지 비용 + 시간 기회비용`을 함께 반영한다. 결과는 한 끼 차이보다 월간·연간 절약액으로 보여줘 사용자가 생활비 개선 효과를 즉시 체감하게 만든다.

## 2. 콘텐츠 한 줄 정의

> 주당 배달 횟수와 직접 요리 비용을 입력하면 월 배달 총지출, 월 직접요리 총비용, 연간 절약 예상액, 손익분기 주문 횟수를 계산하는 생활비 절약 계산기

## 6. 계산 로직

### 6-1. 배달 1회 비용

```text
배달 1회 총비용 =
  max(음식값 + 배달비 + 팁 + 서비스수수료 - 쿠폰할인, 0)
```

### 6-2. 직접 요리 1회 비용

```text
식재료 낭비비용 = 식재료비 × 식재료 낭비율

직접요리 1회 현금비용 =
  식재료비 + 조리 에너지 비용 + 소모품 비용 + 식재료 낭비비용 + 장보기 배송비
```

### 6-3. 시간 비용

```text
요리 시간비용 =
  (장보기 시간 + 조리 시간 + 설거지 시간) / 60 × 본인 시급

시간 포함 직접요리 비용 =
  직접요리 현금비용 + 요리 시간비용
```

배달 대기 시간을 반영할 경우:

```text
배달 대기 시간비용 = 배달 대기 시간 / 60 × 본인 시급
시간 포함 배달 비용 = 배달 1회 총비용 + 배달 대기 시간비용
```

기본 결과는 현금 지출 기준을 우선 보여주고, 시간 비용은 토글 결과로 별도 표시한다.

### 6-4. 월간·연간 절약액

```text
월 배달 횟수 = 주당 배달 횟수 × 월 계산 주수

월 배달 총지출 = 월 배달 횟수 × 배달 1회 총비용
월 직접요리 총비용 = 월 배달 횟수 × 직접요리 1회 비교비용

월 절약 가능 금액 = 월 배달 총지출 - 월 직접요리 총비용
연간 절약 예상액 = 월 절약 가능 금액 × 12
```

### 6-5. 손익분기 주문금액

```text
손익분기 음식값 =
  직접요리 1회 비교비용 - 배달비 - 팁 - 서비스수수료 + 쿠폰할인
```

손익분기 음식값이 0원 이하라면 현재 조건에서는 배달 부대비용만으로도 직접요리보다 비싸다는 메시지를 표시한다.

## 12. FAQ 예시

### Q1. 직접 요리가 항상 배달보다 저렴한가요?

아닙니다. 1인 가구에서 식재료 낭비율이 높거나 시간 비용을 크게 잡으면 직접요리의 절약액이 줄어들 수 있습니다. 그래서 현금 지출 기준과 시간 포함 기준을 나눠 보여줘야 합니다.

### Q2. 시간 비용은 꼭 넣어야 하나요?

필수는 아닙니다. 생활비 절약 관점에서는 현금 지출 기준이 직관적이고, 맞벌이·야근이 많은 직장인은 시간 비용까지 포함해 보는 것이 더 현실적입니다.

### Q3. 배달 쿠폰은 어떻게 반영하나요?

1회 평균 할인액으로 입력합니다. 특정 날짜의 큰 할인보다 한 달 동안 실제로 자주 받는 평균 할인액을 넣어야 결과가 현실적입니다.

### Q4. 밀키트는 직접요리 비용에 넣나요?

MVP에서는 직접요리 프리셋 중 하나로 처리합니다. 밀키트는 식재료비는 높지만 조리 시간이 짧기 때문에 시간 비용 포함 결과에서 유리해질 수 있습니다.

### Q5. 손익분기 주문금액은 무슨 뜻인가요?

직접요리 비용과 배달 비용이 같아지는 1회 음식값입니다. 이 금액보다 비싸게 주문하면 직접요리가 비용상 유리하고, 이 금액보다 낮으면 배달도 합리적일 수 있습니다.

### Q6. 가족 수가 많으면 직접요리가 더 유리한가요?

대체로 유리해지는 경향이 있습니다. 배달은 인원수만큼 음식값이 늘지만 직접요리는 식재료를 묶음으로 쓰면서 1인당 비용이 낮아지는 경우가 많습니다.

### 데이터 구조 예시

```ts
type HouseholdSize = 1 | 2 | 3 | 4;

type DeliveryVsCookingInput = {
  householdSize: HouseholdSize;
  weeklyDeliveryOrders: number;
  mealsPerOrder: number;
  weeksPerMonth: number;

  deliveryFoodCost: number;
  deliveryFee: number;
  riderTip: number;
  serviceFee: number;
  couponDiscount: number;

  cookingIngredientCost: number;
  energyCost: number;
  consumableCost: number;
  ingredientWasteRate: number;
  groceryDeliveryFeePerMeal: number;

  includeTimeCost: boolean;
  hourlyWage: number;
  groceryMinutes: number;
  cookingMinutes: number;
  cleanupMinutes: number;
  includeDeliveryWaitTime: boolean;
  deliveryWaitMinutes: number;
};

type DeliveryVsCookingResult = {
  deliveryCostPerMeal: number;
  cookingCashCostPerMeal: number;
  cookingTimeCostPerMeal: number;
  cookingTotalCostPerMeal: number;
  monthlyDeliveryCost: number;
  monthlyCookingCost: number;
  monthlySaving: number;
  annualSaving: number;
  breakEvenFoodCost: number;
  decisionLabel: "직접요리 유리" | "조건부 유리" | "배달도 합리적";
};
```

### 계산 함수 예시

```ts
function calculateDeliveryCost(input: DeliveryVsCookingInput) {
  return Math.max(
    input.deliveryFoodCost +
      input.deliveryFee +
      input.riderTip +
      input.serviceFee -
      input.couponDiscount,
    0
  );
}

function calculateCookingCost(input: DeliveryVsCookingInput) {
  const wasteCost = input.cookingIngredientCost * input.ingredientWasteRate;
  const cashCost =
    input.cookingIngredientCost +
    input.energyCost +
    input.consumableCost +
    wasteCost +
    input.groceryDeliveryFeePerMeal;

  const timeMinutes = input.groceryMinutes + input.cookingMinutes + input.cleanupMinutes;
  const timeCost = input.includeTimeCost ? (timeMinutes / 60) * input.hourlyWage : 0;

  return { cashCost, timeCost, totalCost: cashCost + timeCost };
}
```

## 17. 운영 체크리스트

### 기획

- [ ] 배달비·팁·쿠폰을 음식값과 분리해서 보여준다.
- [ ] 시간 비용은 기본 꺼짐으로 두고, 켰을 때 결과가 어떻게 바뀌는지 명확히 보여준다.
- [ ] 직접요리 비용을 과소평가하지 않도록 식재료 낭비율을 포함한다.
- [ ] 1인 가구와 가족 가구의 결과 해석을 분리한다.
- [ ] `추정` 또는 `시뮬레이션` 배지를 결과 영역에 표시한다.

### 개발

- [ ] 금액 입력값은 음수 방지 처리한다.
- [ ] URL 파라미터로 입력값 공유를 지원한다.
- [ ] 모바일에서 결과 카드가 먼저 보이도록 배치한다.
- [ ] 비용 차트는 배달/직접요리 항목별 분해가 가능해야 한다.
- [ ] 계산 결과가 0 또는 음수일 때도 자연스러운 판정 메시지를 제공한다.

### 운영

- [ ] 배달비·플랫폼 수수료 구조 변화 시 본문 업데이트

- [ ] 쿠폰·할인 과대 반영 방지 문구 점검
- [ ] 제휴 링크에는 광고/제휴 안내 표시

## 18. 최종 판단

배달 vs 직접 요리 비용 계산기는 검색 의도가 명확하고 반복 사용 가능성이 높은 생활비 절약형 계산기다. 배달비 인상, 식재료 물가 상승, 1인 가구 증가, 맞벌이 시간 부족이라는 현실적인 고민을 한 화면에서 비교할 수 있어 체류 시간과 공유 가능성이 높다.

핵심 차별점은 음식값만 비교하지 않고 **배달 부대비용, 식재료 낭비율, 시간 기회비용**을 함께 계산하는 것이다. 결과는 "집밥이 무조건 싸다"가 아니라 "어떤 조건에서 직접요리가 유리하고, 어떤 조건에서는 배달도 합리적인지"를 판단하게 만드는 방향이 적합하다.
