## 1. 배경 및 목적

고령자 공공임대주택은 "운빨 청약"처럼 보이지만 실제로는 **무주택기간·소득·거주기간·수급자 여부·청약저축 납입횟수** 같은 배점 조건 싸움이다. 문제는 유형이 영구임대·국민임대·통합공공임대·고령자 매입임대·고령자 전세임대·행복주택 고령자형까지 6가지로 나뉘어 있고, 유형마다 대상·경쟁구조·배점이 달라 부모님을 대신 알아보는 자녀 입장에서 "우리 부모님이 어디에 넣어야 유리한지"를 스스로 판단하기 어렵다는 점이다.

**목표**
- 나이·주택보유·소득수준·거주지역·거주기간·장애인여부·청약통장·자동차·세대분리 여부를 입력하면 → 당첨 가능성 점수와 4단계 판정을 즉시 계산
- 점수 구성 요소를 그대로 노출해 "무엇을 보완하면 점수가 오르는지" 확인 가능하게 설계 (청약통장 유지, 세대분리 등 실행 가능한 항목 강조)
- 점수와 조건에 따라 6개 유형 중 **추천 순서**를 함께 제시하고, 유형별 준비서류·신청처까지 한 화면에서 안내
- 기존 고령자·복지 계산기 클러스터(장기요양, 기초연금, 복지급여, 주거급여)와 내부 링크로 연결

### 타겟 독자
- 부모님의 공공임대주택 신청 가능성을 대신 알아보는 40~60대 자녀 (원본 자료가 명시한 핵심 수요)
- 만 65세 이상 본인이 직접 신청을 준비하는 고령자
- 사회복지사·주거복지 상담 실무자 참고용

### 데이터 신뢰도 메모

배점 항목·판정 기준선은 사용자가 제공한 가이드 문서(LH 통합공공임대 우선공급 배점 기준 등 공개 자료 기반 요약)를 그대로 반영한 것으로, LH·마이홈 공식 배점표의 단순화된 추정치다. 실제 배점은 공고별·지자체별로 세부 기준이 다를 수 있어 `확인 필요`/`추정` 배지와 "실제 배점은 공고문 기준" 안내 문구를 결과 화면에 고정 노출한다.

### 계산 로직 메모
```js
// 소득 수준은 상호 배타적 단일 선택으로 처리 (중복 가산 방지)
const INCOME_SCORE = {
  livelihood: 30,      // 기초생활수급자
  nearPoverty: 25,     // 차상위계층
  basicPensionOnly: 0, // 기초연금만 — 별도 가산 없음, 유형 추천에는 반영
  veryLowIncome: 20,   // 소득이 매우 낮음 (수급자·차상위 아님)
  general: 0,
};

function calcScore(input) {
  let score = 0;
  if (input.age >= 65) score += 10;
  if (input.age >= 70) score += 5;
  score += INCOME_SCORE[input.incomeLevel] ?? 0;
  if (input.isDisabled) score += 15;
  if (input.homelessYears === "10plus") score += 20;
  if (input.residenceYears === "5plus") score += 15;
  if (input.subscriptionCount === "24plus") score += 10;
  if (input.carOwnership === "none-or-cheap") score += 10;
  if (input.isSeparatedFromChildren) score += 10;
  if (input.strategyPopularAreaOnly) score -= 20;
  if (input.strategyNewBuildOnly) score -= 15;
  if (input.strategySingleApplicationOnly) score -= 20;
  return score;
}

function judge(score) {
  if (score >= 80) return "적극 신청 권장 — 영구임대·매입임대 가능성 높음";
  if (score >= 50) return "국민임대·매입임대·통합공공임대 병행 추천";
  if (score >= 30) return "지역 확대 필요 — 전세임대도 검토";
  return "조건 재점검 필요 — 소득·세대·자산부터 확인";
}

// 추천 유형 순서: 기본 순서(6절)에서 소득수준에 따라 재정렬
function rankTypes(input) {
  const base = ["seniorPurchase", "permanent", "national", "integrated", "jeonse", "happy"];
  if (input.incomeLevel === "livelihood") return ["permanent", "seniorPurchase", ...base.filter(t => !["permanent","seniorPurchase"].includes(t))];
  if (input.incomeLevel === "basicPensionOnly") return ["national", "integrated", "seniorPurchase", ...base.filter(t => !["national","integrated","seniorPurchase"].includes(t))];
  return base;
}
```
