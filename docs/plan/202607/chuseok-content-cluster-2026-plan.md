# 추석·8월 시즌 콘텐츠 클러스터 2026 — 기획 문서

> 작성일: 2026-07-21
> 유형: 계산기 4개 (전부 `/tools/`)
> 담당: swchoi
> 배경: "8월 컨텐츠 추천" 논의에서 1순위로 선정한 4개 항목을 클러스터로 묶어 기획

## 1. 배경 및 목적

2026년 추석은 9월 25일(금), 연휴는 9월 24일(목)~26일(토) 3일이다. 귀성 기차표 예매·벌초·명절 상여금·장마철 침수 피해 대응은 매년 8월 중순~9월 초에 검색 수요가 집중되는 반복 시즌 키워드다. 8월에 미리 발행해두면 예매 오픈·명절 상여 지급·장마철 사고 시점에 자연 유입을 받을 수 있다.

4개 항목은 검색 의도가 서로 다르지만 "명절 준비 비용·실수령·리스크를 미리 계산해본다"는 공통 프레임으로 묶인다.

| 우선순위 | 슬러그 | 핵심 질문 | 발행 목표 시점 |
|---|---|---|---|
| 1 | `holiday-bonus-after-tax-calculator` | 명절 상여금 실수령액이 얼마나 될까? | 8월 초 (지급 시즌 전) |
| 2 | `hometown-trip-cost-calculator` | 귀성길 KTX·고속버스·자차 중 뭐가 더 쌀까/빠를까? | 8월 말 (예매 오픈 직전) |
| 3 | `beolcho-cost-calculator` | 벌초 대행 맡기면 얼마 드나? | 8월 중순 |
| 4 | `flood-car-insurance-calculator` | 침수차, 보험 처리하면 얼마 받을까? | 8월 초 (장마·태풍 시즌) |

## 2. 데이터 신뢰성 원칙 (공통)

이 클러스터는 4개 모두 "시세·제도 참고 계산기"이며, 실제 사고·거래는 개별 조건에 따라 크게 달라진다. 착수 전 확인한 사실관계는 다음과 같다.

| 항목 | 확인 결과 | 처리 방침 |
|---|---|---|
| 2026년 KTX/SRT 추석 예매 오픈일 | 2026-07-18 기준 미발표. 과거 사례상 명절 3~4주 전이 많았으나 확정 아님 ([노랗IT월드](https://yellowit.co.kr/%EC%A7%81%EC%9E%A5%EC%9D%B8-%EC%97%B0%EB%A7%90%EC%A0%95%EC%82%B0-%EA%B5%AD%EB%AF%BC%EB%82%B4%EC%9D%BC%EB%B0%B0%EC%9B%80%EC%B9%B4%EB%93%9C-%EB%85%B8%EB%8F%99%EB%B2%95/2026%EB%85%84-%EC%B6%94%EC%84%9D-%EA%B8%B0%EC%B0%A8%ED%91%9C-%EC%98%88%EB%A7%A4-ktx-srt-%EC%98%88%EC%95%BD%EC%9D%BC%EC%A0%95-%EB%B0%8F-%EC%B7%A8%EC%86%8C-%EC%9C%84%EC%95%BD%EA%B8%88/)) | 예매 오픈일을 **날짜로 단정하지 않는다.** "통상 3~4주 전, 코레일·SR 공지 확인 필요"로만 안내하고, 계산기는 예매일이 아니라 **비용·소요시간 비교**에 집중한다 |
| KTX 서울-부산 요금 | 2026년 4월 수서 경유 KTX 요금 인하로 일반실 48,800원 (SRT와 동일 수준) ([info.koreacharts.com](https://info.koreacharts.com/traintime/10012/10020/contents.html?date=2026-07-13)) | 확정 요금으로 사용 가능 |
| KTX 서울-대구/광주 요금 | 검색상 "최저가 기준" 20,900원(대구)·22,300원(광주) 노출됨 — 얼리버드 등 할인 특가일 가능성이 있어 표준 운임으로 단정 불가 | 계산기 기본값은 **보수적인 왕복 대표값**으로 두고 사용자가 직접 수정하게 하며, 구현 직전 코레일/SR 공식 요금표로 재확인 |
| 고속버스 서울-부산 요금 | 일반 약 23,000원, 우등 약 34,000원, 프리미엄 약 40,000원 ([bus.koreacharts.com](https://bus.koreacharts.com/express-bus-terminal/NAEK010/NAEK700.html)) | 참고값으로 사용, "시점별 변동" 고지 |
| 벌초 대행 비용 | 기당 6만~20만 원, 지역·난이도별 편차 큼. 추석 전후 10~30% 할증 흔함 ([조상님복덕방](https://blog.bokdeokbang.kr/%EB%AC%B4%EB%8D%A4-%EC%9D%B4%EC%95%BC%EA%B8%B0/2025%EB%85%84-%EB%B2%8C%EC%B4%88%EB%8C%80%ED%96%89-%EA%B0%80%EA%B2%A9%C2%B7%EB%B9%84%EC%9A%A9-%ED%95%9C%EB%88%88%EC%97%90-%EB%B3%B4%EA%B8%B0-%EA%B2%AC%EC%A0%81-%EB%8B%A4%EB%A5%B8-%EC%9D%B4%EC%9C%A0/), [greenfarmers](https://greenfarmers.co.kr/blog/?bmode=view&idx=21270188)) | 표준 단가가 없으므로 "범위(최저~최고) + 할증 배지"로 표현하고 **견적 참고용**임을 강조 |
| 침수차 전손/분손 기준 | 손해액+인정비용 합계가 보험가액 이상이면 전손, 미만이면 분손(수리) 처리. 자기부담금 공제 여부는 상품마다 다름 ([DB손해보험](https://www.idbins.com/pc/bizxpress/ask/cc/FWCLAV1146.shtm), [정보알리미](https://dnublog.co.kr/2026/07/06/%EC%B9%A8%EC%88%98%EC%B0%A8-%EC%A0%84%EC%86%90-%EC%B2%98%EB%A6%AC-%EA%B8%B0%EC%A4%80%EA%B3%BC-%EC%B0%A8%EB%9F%89%EA%B0%80%EC%95%A1%C2%B7%EB%B3%B4%ED%97%98%EA%B8%88-%ED%99%95%EC%9D%B8-%EB%B0%A9/)) | 계산기는 **전손 여부 추정**과 **자기부담금 반영 예상 보험금**만 제공하고, "최종 판정은 보험사 손해사정 결과가 우선"이라고 매 결과에 고지 |
| 명절 상여금 과세 | 설·추석 상여·떡값·선물세트(현물 포함)는 전부 과세 대상 근로소득. 비과세 항목 아님 ([삼쩜삼](https://help.3o3.co.kr/hc/ko/articles/10223074227225--%EC%B6%94%EC%84%9D-%EB%AA%85%EC%A0%88-%EC%83%81%EC%97%AC%EA%B8%88-%EC%84%B8%EA%B8%88%EC%9D%80-%EC%96%BC%EB%A7%88%EB%82%98-%EB%97%84%EA%B9%8C-ft-%ED%8F%89%EA%B7%A0-%ED%86%B5%EA%B3%84)) | "떡값은 비과세" 같은 통설 오해를 FAQ에서 명시적으로 정정 |

## 3. 원안 대비 조정 사항

| 원안 아이디어 | 조정 |
|---|---|
| "추석 귀성 기차표 예매 전쟁 대응 가이드/체크리스트" (정보성) | 계산기 중심 사이트 포맷에 맞춰 **"귀성길 교통수단 비교 계산기"**로 전환 — KTX/SRT/고속버스/자가용 비용+소요시간을 사용자가 직접 비교. 예매 타임라인은 계산기 하단 안내 섹션으로 축소 |
| "명절 상여금·떡값 실수령 계산기" | 기존 `/tools/bonus-after-tax-calculator/`(성과급 세후 실수령액 계산기)가 이미 동일한 세금 계산 구조(간이세액표, 4대보험, 지급월 반영)를 갖추고 있음을 확인. **완전 신규 엔진을 만들지 않고, 같은 계산 로직 구조를 재사용**해 지급 방식(할부 없음, 단일 지급 중심)과 프리셋만 명절에 맞게 단순화한 전용 페이지로 설계 (§5) |
| "벌초·성묘 대행비용 비교" | 표준 단가가 없어 "비교표"보다 **입력형 견적 계산기**(기수·난이도·지역·부가작업 → 예상 총액 범위)가 사용자 의도에 더 맞음 |
| "태풍·장마철 침수차 보험처리 계산기" | 그대로 채택. 다만 전손/분손 판정과 자기부담금 처리가 보험사·약관마다 달라 **"추정" 프레임을 강하게 유지** |

## 4. 항목별 요약 기획

### 4-1. 명절 상여금 실수령 계산기 (`holiday-bonus-after-tax-calculator`)

| 항목 | 내용 |
|---|---|
| 핵심 메시지 | "명절 상여금도 전액 과세 근로소득입니다. 연봉·부양가족을 입력하면 세후 실수령액을 계산합니다." |
| 입력값 | 상여금 총액, 연봉, 부양가족 수, 20세 이하 자녀 수, 지급월(9월 기본값), 4대보험 포함 여부 |
| 출력값 | 세전 상여금, 소득세·지방소득세, 4대보험(국민연금·건강보험·장기요양·고용보험), 세후 실수령액, 실수령률(%) |
| 차별점 | 기존 `bonus-after-tax-calculator`와 계산 구조는 동일하되, 지급월 프리셋을 추석·설로 고정하고 "떡값은 비과세"라는 흔한 오해를 정면으로 반박하는 FAQ·설명 섹션 강화 |
| 참고 계산기 | `/tools/bonus-after-tax-calculator/` (계산 로직 구조 재사용, §5 설계 문서에서 상세) |

### 4-2. 귀성길 교통수단 비교 계산기 (`hometown-trip-cost-calculator`)

| 항목 | 내용 |
|---|---|
| 핵심 메시지 | "출발지·인원수를 입력하면 KTX·SRT·고속버스·자가용 중 뭐가 더 저렴하고 빠른지 비교합니다." |
| 입력값 | 출발지-도착지(주요 구간 프리셋: 서울-부산/대구/광주/대전), 인원수, 자차 유종·연비(자가용 계산용), 톨게이트 통행료 자동 반영 |
| 출력값 | 교통수단별 총비용(왕복, 인원수 반영), 소요시간, 1인당 비용, "가장 저렴/가장 빠름" 배지 |
| 주의 | 정확한 요금은 예매 시점·좌석 등급에 따라 달라지므로 모든 요금에 "참고 요금" 배지 필수. 예매 오픈일은 날짜 단정 금지(§2) |

### 4-3. 벌초 대행비용 계산기 (`beolcho-cost-calculator`)

| 항목 | 내용 |
|---|---|
| 핵심 메시지 | "묘지 기수와 상태를 입력하면 벌초 대행 예상 비용 범위를 계산합니다." |
| 입력값 | 묘지 기수, 현장 난이도(평지/경사/차량 진입 불가), 부가작업(진입로 정비·폐기물 처리·잡목 제거), 추석 성수기 할증 여부 |
| 출력값 | 예상 비용 범위(최저~최고), 할증 적용 후 범위, 기당 평균 단가 |
| 주의 | 표준 단가가 없는 시장이므로 결과는 반드시 범위로 표시하고 "실제 견적은 업체별로 직접 확인" 고지 |

### 4-4. 침수차 보험처리 계산기 (`flood-car-insurance-calculator`)

| 항목 | 내용 |
|---|---|
| 핵심 메시지 | "차량 시세와 예상 수리비를 입력하면 전손·분손 가능성과 예상 보험금을 추정합니다." |
| 입력값 | 차량 시세(보험가입금액), 예상 수리비, 자기부담금(정액/정률), 자기차량손해담보 가입 여부, 전손 시 자기부담금 면제 특약 여부 |
| 출력값 | 전손/분손 추정 결과, 예상 보험금(자기부담금 반영), 잔존물 가액 안내, 전손 판정 기준 설명 |
| 주의 | 최종 전손·분손 판정은 보험사 손해사정 결과가 우선이라는 고지를 결과 카드에 항상 포함. 자기차량손해담보 미가입 시 보상 불가를 첫 화면에서 강조 |

## 5. 공통 구현 원칙

- URL: `/tools/<slug>/` flat 구조, 저장소 컨벤션 그대로.
- 데이터 파일: `src/data/<slug camelCase>.ts`, 스크립트: `public/scripts/<slug>.js`, SCSS: `src/styles/scss/pages/_<slug>.scss` prefix는 각 계산기 이니셜(예: `hbc-`, `htc-`, `bcc-`, `fci-`).
- 모든 결과 카드에 `추정`/`참고` 배지 필수 (`CONTENT_GUIDE.md` 기준).
- 홈 카테고리 등록(`src/pages/index.astro` 대응 없음 — tools는 `src/pages/tools/index.astro`의 카테고리 매핑을 별도 확인).
- 4개 모두 `SimpleToolShell` 레이아웃 기준으로 설계(비교형 요소가 있는 `hometown-trip-cost-calculator`만 결과 카드 내부에 비교 그리드 추가).

## 6. 발행 순서 제안

1. `flood-car-insurance-calculator` — 장마·태풍 시즌 즉시 유입 가능, 8월 초 우선
2. `holiday-bonus-after-tax-calculator` — 9월 상여 지급 전 검색 시작, 8월 초~중순
3. `beolcho-cost-calculator` — 8월 중순, 추석 한 달 전 검색 증가 시작
4. `hometown-trip-cost-calculator` — 예매 공지가 뜨는 8월 말 직전 발행이 타이밍상 가장 유리
