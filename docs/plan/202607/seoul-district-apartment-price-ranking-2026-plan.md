## 1. 기획 의도

"서울 구별 집값 순위"는 CTR 100%로 잡힌 고전환 키워드이며, 부동산 관심이 높아지는 시기에 반복적으로 검색되는 주제다. 비교계산소에는 강남·마포·성동·용산·강서·강동·송파 등 **구별 단지 리포트**가 이미 상당수 있지만, 이를 한눈에 비교하는 **서울 전체 구 순위형 허브 리포트**는 없다.

이 리포트는 두 가지 역할을 동시에 한다:
1. **독립 콘텐츠**: "서울 구별 집값 순위"를 찾는 사용자를 직접 흡수
2. **내부링크 허브**: 기존 구별 리포트로 트래픽을 분배해 전체 부동산 클러스터의 체류시간·페이지뷰 향상

## 3. 데이터 구조

### 3-1. 커버 범위
서울 25개 구 전체. 아파트 거래가 활발한 구부터 데이터 확보 → 거래 건수가 적은 구는 "참고용" 표기.

### 3-2. 핵심 지표 (구별)
| 지표 | 기준 |
|------|------|
| 대표 매매가 | 2026년 상반기 전용 84㎡ 실거래 평균 (건수 3건 미만이면 중위값) |
| 전년 대비 상승률 | 2025년 상반기 vs 2026년 상반기 동일 평형 평균 비교 |
| 전세가율 | 동일 기간 전세 평균 ÷ 매매 평균 |
| 대표 단지 | 구 내 거래량 상위 1~2개 단지명 (독자 탐색 힌트용) |

### 3-3. 데이터 출처
- 국토교통부 실거래가 공개시스템 (rt.molit.go.kr)
- 기존 구별 리포트 데이터 파일(`gangnamApartmentPrice2026.ts`, `mapoApartmentPrice2026.ts` 등) 재사용 — 이미 구현된 단지 데이터는 그대로 참조


## 4. 화면 구성

### 섹션 순서

1. **Hero**
   - 타이틀: "서울 25개 구 아파트 집값 순위 2026"
   - 부제: "가장 비싼 구와 가장 저렴한 구의 차이는 얼마나 될까?"
   - 업데이트 날짜 표시

2. **Summary 카드 (4개)**
   - 가장 비싼 구 + 평균가
   - 가장 저렴한 구 + 평균가
   - 1위↔25위 격차 (배수)
   - 전년 대비 가장 많이 오른 구

3. **구별 순위표 (인터랙티브)**
   - 정렬 기준 탭: `매매가 높은 순` / `상승률 높은 순` / `전세가율 높은 순`
   - 컬럼: 순위 / 구 / 평균 매매가(84㎡) / 전년 대비 / 전세가율 / 대표 단지 / 상세 리포트 링크
   - 기존 구별 리포트가 있는 행에는 "상세 보기" 링크 표시
   - 거래 건수 적은 구는 "참고" 뱃지

4. **가격대별 구 분류 카드**
   - 30억+ / 20~30억 / 10~20억 / 10억 이하 구간별 그룹핑
   - 예산별 탐색 편의 제공

5. **격차 시각화**
   - 바차트: 25개 구 평균 매매가 바 (1위~25위)
   - 강조: 상위 3위·하위 3위 별도 색상

6. **주목할 구 TOP3**
   - 상승률 1위 / 2위 / 3위 구 — 이유(개발 호재, 교통 등) 1~2줄 요약
   - 단정 표현 금지, "~로 알려져 있음", "~가 영향을 준 것으로 보임" 톤

7. **구별 상세 리포트 링크 카드 그리드**
   - 기존 리포트 있는 구만 카드 표시
   - 강남·마포·성동·용산·강서(마곡)·강동·송파·분당(경기) 연결
   - "이 구 더 자세히 보기 →"

8. **데이터 기준 및 한계**
   - 84㎡ 기준, 실거래 평균, 거래 건수 적은 구 주의사항
   - 발행 시점 스냅샷, 계속 변동됨

## 8. 리스크 및 주의사항

- 거래 건수가 적은 구(종로구·중구·용산구 일부 평형 등)는 평균값이 왜곡될 수 있음 — "참고용" 뱃지 필수
- 재건축·재개발 단지의 일시적 저가/고가 거래가 평균을 왜곡할 수 있음 — 이상값 제외 처리 기준 명시
- 구별 대표 단지 선정이 주관적으로 보일 수 있어, "거래량 상위 기준" 선정 근거를 본문에 명시
- 실거래가는 분기마다 갱신 필요 (운영 부담) — 1차 발행 후 분기별 업데이트 루틴 설정

## 9. 우선순위 판단

| 평가 항목 | 점수 | 근거 |
|---------|------|------|
| 검색 수요 | 상 | CTR 100% 확인, 부동산 시즌 상시 수요 |
| 체류시간 기대 | 상 | 정렬 인터랙션 + 구별 탐색 + 상세 리포트 이동 → 멀티페이지 동선 |
| 구현 난이도 | 중 | 데이터 25개 구 수집이 핵심 병목, 컴포넌트는 기존 비교표 패턴 재사용 |
| 내부링크 효과 | 상 | 기존 구별 리포트 7개로 트래픽 분배 — 부동산 클러스터 전체 강화 |
| 데이터 변동 리스크 | 중 | 분기별 업데이트 필요 |

결론: **즉시 설계 착수 권장.** 데이터 수집(25개 구 실거래 평균)이 병목이므로, 거래량 많은 구 10~15개로 1차 발행 후 나머지 구 추가하는 점진적 확장 전략.
