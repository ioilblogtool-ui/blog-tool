## 1. 배경 및 목적

"삼성전자 최대주주", "SK㈜ SK스퀘어 차이", "지주회사 할인율" 등은 검색 수요가 꾸준하고 투자자·구직자 양쪽 관심사와 겹치는 주제다. 단순 계열사 목록 나열이 아니라, **① 누가 누구를 지배하는지(지분 연결도), ② 내가 가진 주식이 그룹 내 어떤 위치인지(간접 지분율 계산), ③ 중복상장으로 가치가 얼마나 할인되는지(지주회사 할인율 계산)**를 계산·시각화하면 기존 "계열사 목록" 콘텐츠와 차별화된다.

비교계산소 아키텍처(Astro SSG + `src/data/*.ts` 정적 데이터 + `public/scripts/*.js` 클라이언트 계산)와도 궁합이 좋다 — 지분구조는 기준일 스냅샷 데이터라 실시간 API·DB 없이 **분기 1회 수동 갱신하는 정적 데이터**로 충분하다.

### 목표
- "OO그룹 지분구조", "지주회사 할인율", "간접 지분율 계산" 검색 유입 확보
- 그룹별 리포트 ↔ 간접 지분율 계산기가 상호 연결되는 클러스터 구조
- 계열사 데이터 재사용 원칙 확립 (그룹당 데이터 파일 1개, 리포트·계산기가 공유)

### 원안 대비 조정 사항
원안은 그룹 9개(삼성·SK·LG·롯데·한화·현대차·HD현대·LS·두산) × 계산기 4개(지주회사 할인율·간접 지분율·중복상장 단계·쪼개기 상장 전후)를 제안했다. 1차 개정(2026-07-16 오전)에서는 조사 비용을 이유로 3개 그룹으로 축소했으나, **9개 그룹 전체 1차 조사를 완료한 뒤(같은 날 오후) 범위를 원안 그대로 9개 그룹으로 복원**했다. 다만:
- 시가총액이 들어가는 순간 "기준일" 관리 부담이 생긴다(지분율은 분기 단위, 시가총액은 일 단위로 변함) — **계산기는 여전히 간접 지분율 계산기 1개로 시작**하고, 지주회사 할인율 등 NAV 기반 계산기는 2단계로 미룬다(8장).
- 지분율·최대주주 데이터의 정확도 처리 방식은 2장 "수치 출처" 원칙 참고 — DART 원문 직접 대조가 불가능해(자동화 접근 차단 확인됨) 2차 출처를 명확한 고지와 함께 사용하는 쪽으로 조정했다.

## 5. 페이지 10 — 간접 지분율 계산기

### 5-1. URL·메타

```
슬러그: /tools/indirect-ownership-calculator/
타이틀: 간접 지분율 계산기 2026 | 다단계 지분 연결 실질 지분율 바로 계산
디스크립션: 모회사→자회사→손자회사 지분율 입력하면 실질 간접 지분율 바로 계산. 삼성·SK·LG 등 9개 그룹 지분구조 예시 포함.
```

### 5-2. 계산 로직 (원안 코드 그대로 채택, 검증 완료된 단순 산식)

```ts
export function calculateIndirectOwnership(rates: number[]): number {
  return rates.reduce((result, rate) => result * (rate / 100), 1) * 100;
}
```

- 입력: 단계별 지분율 (2~4단계, `+ 단계 추가` 버튼)
- 출력: 최종 실질 지분율, 단계별 계산 과정 표시
- 프리셋: 9개 그룹 리포트에서 실제로 쓰인 연결 관계 중 대표적인 것 3~4개를 예시 버튼으로 제공 (예: "삼성물산→삼성생명→삼성전자", "SK㈜→SK스퀘어→SK하이닉스", "현대모비스→현대차→기아" 등) — 리포트와 계산기를 데이터로 연결

### 5-3. 섹션 구성
1. Hero + InfoNotice ("실제 청산가치·의결권과 다를 수 있는 단순 곱셈 계산" 고지)
2. 계산기 (단계별 입력 → 실질 지분율)
3. "간접 지분율이란?" 설명 카드 (순환출자 시 유의사항 — 계산기 자체는 단순 경로 계산이며 순환출자 그래프 탐색은 미지원함을 명시. 현대차·LS 사례를 예시로 언급 가능)
4. 그룹별 리포트 허브(`group-ownership-hub`)로 CTA

### 5-4. FAQ
1. 간접 지분율은 어떻게 계산하나요?
2. 여러 경로로 지분을 보유하면 어떻게 계산하나요? (합산 원칙, 이 계산기는 단일 경로만 지원함을 안내)
3. 간접 지분율이 의결권과 같은 의미인가요?
4. 순환출자가 있으면 계산이 달라지나요? (→ 현대차그룹 사례 링크)
5. 실제 사례로 확인하고 싶어요 (→ 그룹별 리포트 허브 CTA)

```ts
  stockCode?: string;          // 비상장이면 undefined
  listed: boolean;
  market?: "KOSPI" | "KOSDAQ";
  role: "CORE_COMPANY" | "FINANCIAL" | "CORE_AFFILIATE" | "SUBSIDIARY";
}

export type SourceConfidence = "HIGH" | "MEDIUM" | "LOW";

export interface OwnershipEdge {
  from: string;        // GroupCompany.id, 또는 "OWNER_FAMILY"
  to: string;
  ratePercent: number;
  baseDate: string;          // "2026-03-31" 등 공시·기사 기준일
  sourceLabel: string;       // "삼성물산 2026 1분기보고서" 또는 "위즈리포트 지분현황(2026-05-15)"
  sourceUrl: string;
  confidence: SourceConfidence; // 조사 노트의 신뢰도 라벨을 그대로 이관 — 2차 출처 사용 원칙(2장)에 따라 필수
}

export interface GroupOwnershipMeta {
  groupId: string;
  groupName: string;
  isLegalHoldingCompany: boolean;
  holdingCompanyNote?: string;  // 두산(지위 반납)·LS(3개 지주사)처럼 단순 예/아니오로 안 끝나는 경우 설명
  updatedAt: string;
}
```

시가총액·NAV 필드는 1차 버전에 포함하지 않는다 (2장 원칙 참고). `confidence` 필드는 페이지 렌더링 시 "낮음" 항목에 시각적 표시(예: 각주 아이콘)를 붙이는 데 사용 — 2장 결정(2차 출처 사용 + 고지)의 실제 구현 방법.

## 7. 시각화 (1차: SVG 세로 트리)

원안 "2안: 그룹 구조 트리"를 1차 버전으로 채택 — 모바일 우선(`CLAUDE.md` mobile-first 원칙)이라 네트워크 그래프보다 세로 트리가 적합.

```
[오너 일가]
   │ 19.34%
   ▼
[삼성물산]
   │ 8.51%
   ▼
[삼성생명]
   │ ...%
   ▼
[삼성전자]
```

- 각 간선에 지분율 라벨 표시
- 상장/비상장 배지 (원안 "3안: 상장 단계 배지"를 트리 노드에 통합 — 별도 계산기로 분리하지 않음)
- React Flow 등 신규 라이브러리 도입은 1차에서 보류 — 기존 `SimpleToolShell`/리포트 페이지에서 쓰는 정적 SVG 패턴 재사용 (신규 의존성 추가 여부는 설계 단계에서 `docs/CODE_SKILL.md` 확인 후 결정)

**단일 트리로 표현 안 되는 그룹 주의**:
- **LS그룹**: ㈜LS·E1·예스코홀딩스가 지분으로 연결되지 않은 3개 별도 지주사다. 세 그룹을 억지로 한 트리에 넣지 말고, **"동일인(총수)만 공유하는 3개 지주사"**로 병렬 배치하거나 탭으로 분리
- **현대차그룹**: 현대모비스→현대차→기아→현대모비스로 되돌아오는 순환출자라 세로 트리가 원형에 가깝게 그려짐 — 마지막 간선을 점선으로 표시해 "출발점으로 돌아옴"을 시각적으로 구분
- **두산그룹**: ㈜두산이 법적 지주회사가 아니므로 트리 최상단 노드에 "지주회사 아님(2025년 지위 반납)" 배지 별도 표시
