## 1. 배경 및 목적

"60대 일자리", "노인 일자리", "경비 월급", "요양보호사 월급"은 검색량이 꾸준하고, 부모님 일자리를 찾는 자녀와 본인이 검색하는 수요가 겹치는 롱테일 키워드 묶음이다. 단일 글이 아니라 **입력하면 예상 월급이 나오는 계산기**로 만들어야 체류시간과 재방문이 생긴다.

### 목표
- "60대 일자리", "경비 월급", "요양보호사 월급", "노인 일자리" 검색 유입 확보
- 각 계산기가 서로 자연스럽게 연결되는 클러스터 구조 (메인 계산기 ↔ 직업별 계산기 ↔ 비교 리포트)
- 계산기 로직 중복 없이, 페이지마다 실제로 다른 입력·해석 제공

### 페이지 수 결정
계산기 5개+콘텐츠 7개짜리 원안을 그대로 다 만들면 로직이 겹치는 페이지가 늘어난다(경비/미화/주차관리가 전부 "시급×시간+주휴수당"으로 로직 동일). **로직이나 관점이 실제로 다른 페이지만 분리**해 5개로 구성한다. (상세 배경은 1차 기획 문서 참고 — 이 문서는 그 결정을 그대로 유지하고 세부 설계만 보강)

### 4-3. 계산·추천 로직

```js
function calculateJobScore(user, job) {
  let score = 0;
  if (job.maxMonthlyPay >= user.targetMonthlyPay) score += 30;
  else score -= 20;
  if (user.physicalCondition === "LOW" && job.physicalLoad.includes("HIGH")) score -= 30;
  else score += 20;
  if (job.requiresCertificate && !user.hasCertificate) score -= 15;
  else score += 10;
  if (job.nightShift && !user.canNightShift) score -= 20;
  if (user.prefersPartTime && job.partTimeAvailable) score += 15;
  if (job.jobCode === "CARE_WORKER" && user.canCareWork) score += 20;
  return score;
}
```

직업 마스터 데이터(`jobCode`, `physicalLoad`, `nightShift`, `partTimeAvailable` 등)는 아래 5-1 데이터 구조 참고. **월급 계산은 "2-4 설계 원칙"에 따라 시급×시간 기반 계산이지 시장평균 조회가 아님.**
