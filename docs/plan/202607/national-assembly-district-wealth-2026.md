## 1. 기획 의도

`국회의원 재산순위` 키워드는 트래킹 기간 내내 CTR 최상위권(최근 90%)을 유지하는 검증된 클러스터다. 다만 기존 `national-assembly-wealth-2026`은 **TOP30 랭킹** 중심이라 "내 지역구 의원은 재산이 얼마인가"라는 실제 검색 의도(정원오 재산, 추미애 재산 등 이름 기반 롱테일)를 정면으로 받지 못하고 있다.

이 리포트는 같은 데이터를 **지역구·권역 단위로 재구성**해 다음을 노린다:
1. "우리 지역 의원" 프레이밍으로 실사용자 검색 의도에 더 가깝게 접근
2. 기존 TOP30 리포트와 인터널 링크로 묶어 정치인 재산 클러스터 체류시간 확장
3. 이름 기반 롱테일(지역구 소지역명 포함 검색)을 지역구 데이터로 흡수

---

## 2. 데이터 범위 — 반드시 먼저 읽을 것

⚠️ **중요한 스코프 제약**: 현재 `src/data/nationalAssemblyWealth2026.ts`에는 **개인 단위 실명 데이터가 40명분**(TOP30 + BOTTOM10)만 있다. 나머지 260명은 정당별·직업군별·구간별 **집계 추정치**만 존재하고 개인별 지역구 데이터는 없다.

→ 따라서 이 리포트는 **"전국 253개 지역구 전체 비교"가 아니라 "TOP30·BOTTOM10 의원의 지역구·권역별 재구성 + 권역 단위 집계 비교"**로 스코프를 명확히 한다. "우리 지역 의원 순위"라는 제목의 기대치와 실제 제공 데이터(40명 한정) 사이 괴리가 생기지 않도록, Hero·FAQ에서 "공개된 TOP30·BOTTOM10 기준" 문구를 명시한다.

신규로 실명 개인의 재산 수치를 260명분 추가 추정하지 않는다 (이전 검토에서 실명 대상 대량 추정 데이터 생성은 리스크로 판단해 기각됨).

## 4. 데이터 구조

### 4-1. 재사용 (신규 수집 불필요)
- `NAW_TOP30`, `NAW_BOTTOM10` — 이미 `district` 필드 보유, 그대로 재사용
- `NAW_BY_PARTY`, `NAW_KPI` — 정당·전체 통계는 기존 데이터 그대로 인용

### 4-2. 신규 추가 (권역 집계 — 기존 40명 데이터로부터 도출, 신규 개인 추정 아님)
```ts
// src/data/nationalAssemblyDistrictWealth2026.ts
export interface RegionSummary {
  region: string;           // "수도권" | "영남" | "호남" | "충청" | "강원·제주" | "비례대표"
  memberCountInSample: number; // TOP30+BOTTOM10 중 해당 권역 인원 수 (표본, 전체 아님)
  avgBillionInSample: number;  // 표본 평균
  topName: string;
  topBillion: number;
  note: string;              // "TOP30·BOTTOM10 표본 기준, 전체 지역구 평균 아님" 등 명시
}
```
- 권역 분류 기준: 서울·경기·인천=수도권 / 부산·울산·경남=영남(동남권) / 대구·경북=영남(대경권) / 광주·전남·전북=호남 / 대전·세종·충남·충북=충청 / 강원·제주 / 비례대표는 별도 그룹
- **표본 40명 기준임을 카드·표에 반복 라벨링** ("40명 표본" 배지) — 전체 지역구 대표성 오인 방지

### 4-3. 데이터 출처
- 기존 `nationalAssemblyWealth2026.ts`의 `dataNote`와 동일한 출처·주의 문구 승계: "공직자윤리위원회 2026년 정기 재산변동 공개(관보 기준) 추정치, 투자 판단 근거로 사용 금지"

## 9. 리스크 및 주의사항

- **가장 큰 리스크**: 제목("지역구별 순위")이 "전체 지역구 커버"를 암시할 수 있음 → Hero·InfoNotice·FAQ 3곳에서 "40명 표본 기준" 반복 고지로 완화
- 권역 분류(수도권/영남/호남/충청 등)는 정치적으로 민감할 수 있어 "지리적 구분일 뿐 정치 성향과 무관" 톤 유지
- 실명 개인 재산 수치는 기존 TOP30/BOTTOM10 데이터를 그대로 인용만 하고 신규 추정하지 않음 — 새 개인 데이터 요청이 들어와도 기각 원칙 유지
- 원본 리포트(`national-assembly-wealth-2026`)와 콘텐츠 70% 이상 중복되지 않도록, 지역구 검색·권역 비교 등 **원본에 없는 인터랙션**을 핵심 차별화 요소로 유지

## 10. 우선순위 판단

| 평가 항목 | 점수 | 근거 |
|---------|------|------|
| 검색 수요 | 상 | 국회의원 재산 클러스터 CTR 최상위권 검증됨, 이름 기반 롱테일 지속 유입 확인 |
| 체류시간 기대 | 중상 | 검색·필터 인터랙션 + 원본 리포트 왕복 동선 |
| 구현 난이도 | 하 | 신규 데이터 수집 없음 — 기존 40명 데이터 재구성 + 권역 집계만 추가 |
| 내부링크 효과 | 상 | 정치인 재산 클러스터(TOP30, 지자체장 3종) 상호 연결 강화 |
| 데이터 정확성 리스크 | 중 | "40명 표본" 고지를 누락하면 완전성 오인 리스크 → 설계 단계에서 문구 고정 |

결론: **설계 착수 권장.** 신규 데이터 수집이 거의 없어 착수 비용이 매우 낮음. 단, "표본 40명" 고지를 컴포넌트 레벨에서 강제하는 설계가 선행되어야 함.
