## 1. 배경 및 목적 (원안 요약)

원안은 "2026 부동산 세제개편"을 뉴스 요약이 아니라 **"내 세금이 얼마나 바뀌는지 계산"**하는 도구로 만들자는 아이디어다. 보유세·양도세·취득세를 각각 계산기로 분리하고, "2026년에 팔까 2027년에 팔까"를 비교하는 콘텐츠를 대표 기능으로 제안했다. 확정 전 정책이므로 완화/기준/강화 3개 시나리오로 보여주자는 방향도 제시했다.

이 주제는 기존에 이미 구축된 세금 계산기 인프라(`apartment-holding-tax`, `real-estate-acquisition-tax`, `multi-house-tax-2026`, `property-tax-payment-2026`)와 자연스럽게 연결되는 **확장 클러스터**다. 완전히 새로운 주제가 아니라, 기존 계산기에 "정책 변경 시나리오" 레이어를 얹는 작업에 가깝다.

## 2. 데이터 신뢰성·표현 원칙 ★ 핵심 (착수 전 반드시 확정)

이 콘텐츠는 **아직 확정되지 않은 세법 개정안**을 다루므로, 다른 계산기보다 표현·근거 리스크가 훨씬 크다. 원안도 이 위험을 인지하고 "확정 세금", "반드시 오른다" 같은 표현을 금지했지만, 그보다 먼저 확인해야 할 전제가 있다.

### 2-1. 착수 전 필수 확인 — 완료 (2026-07-20)

웹 검색으로 실제 근거를 확인했다. **실제로 진행 중인 논의이며, 정부 발표가 예고된 상태**다.

| 확인 항목 | 결과 |

### 4-2. 신규 계산기 vs 기존 계산기 확장 — 조사 완료 (2026-07-20)

착수 전 확인 결과, 원안이 제안한 계산기 중 다수가 **이미 구현되어 있다.**

| 원안 계산기 | 확인 결과 | 판단 |
|---|---|---|
| 보유세 변화 계산기 (공정시장가액비율 조정) | `src/pages/tools/apartment-holding-tax.astro` — 재산세·종부세 공정시장가액비율을 사용자가 직접 입력하는 필드가 이미 존재(§2-1 대안 프레임과 정확히 일치) | **신규 불필요.** §2-1 확인 결과 실제 개편 근거가 있으면 이 페이지에 "개편 전/후 비교" 섹션만 추가 |
| 양도세 개편 계산기 (장특공제 시나리오) | `src/pages/tools/capital-gains-tax-calculator.astro` — 1주택 비과세·장기보유특별공제·단기세율·다주택 중과 모두 이미 반영 | **신규 불필요.** 장특공제율 자체를 바꿔보는 옵션이 없다면 그 부분만 추가 검토 |
| 취득세 변화 계산기 | `src/pages/tools/real-estate-acquisition-tax.astro` 존재 | **신규 불필요.** 개편 시나리오 옵션 추가 여부만 검토 |
| 집을 언제 팔까 (2026 vs 2027 매도 비교) | 동일 기능 페이지 없음 | **신규 필요 — 유일한 진짜 공백** |

결론: 원안의 4개 계산기 중 3개는 이미 있는 계산기에 "정책 변경 시나리오" 옵션을 얹는 수준으로 충분하고, **완전 신규 계산기는 매도 시점 비교 1개뿐**이다. 이는 유지보수 부담을 크게 줄이는 방향이므로 원안(4개 신규 계산기)보다 우선 채택한다.

### 4-3. 1차 착수 범위 — 확정 (2026-07-20, 사용자 결정)

**이번 착수분은 §6 비교표 리포트 1개만 진행한다.** §5 매도 비교 계산기와 §7 확장 항목은 리포트 발행 후 별도로 재검토한다.

## 5. 페이지 설계 — 2026년 vs 2027년 매도 비교 계산기 (1차 착수 권장)

### 5-1. URL·메타 (CLAUDE.md 타이틀 공식 적용)

```
슬러그: /tools/sell-timing-comparison-2026/
타이틀: 집 매도 시점 계산기 2026 | 2026년 vs 2027년 팔면 얼마 차이날까
디스크립션: 예상 매도가, 보유세, 대출이자, 양도세를 입력하면 올해 팔 때와 내년 팔 때 최종 손에 남는 돈을 바로 비교. 시나리오별 손익 계산 포함.
```

### 5-2. 입력값

| 입력 | 타입 | 설명 |
|---|---|---|
| 현재 매도 예상가 | number | 올해 매도 시 예상가 |
| 예상 매도가 상승률 | percent | 1년 보유 시 가격 변동 가정 (사용자 지정) |
| 취득가 | number | 양도세 계산용 |
| 보유기간 / 실거주기간 | number | 장특공제·비과세 판단 |
| 1년 추가 보유 시 보유세 증가액 | number | 기존 `apartment-holding-tax` 계산 결과 연동 가능 |
| 1년 추가 보유 시 대출이자 | number | 사용자 입력 |
| 중개수수료율 | percent | 지역별 상한요율 기본값 제공 |
| 주택 수 | select | 양도세 중과·비과세 판단 |

### 5-3. 출력값

| 항목 | 2026년 매도 | 2027년 매도 |
|---|---:|---:|
| 예상 매도가 | O | O |
| 추가 보유세 | 0 | 계산값 |
| 추가 대출이자 | 0 | 계산값 |
| 양도세 | 계산값 | 계산값(보유·거주기간 변화 반영) |
| 중개수수료 | 계산값 | 계산값 |
| 최종 실수령액 | 계산값 | 계산값 |
| 최종 차액 | — | ± 계산값 |

핵심 기능 문구: **"1년 더 보유하면 실수령액이 얼마나 달라지는가"** — 정책 개편 여부와 무관하게 성립하는 계산이므로 우선 착수 후보로 적합.

### 5-4. 계산 흐름 개요

```text
최종 실수령액 = 매도가 - 양도세 - 중개수수료 - (해당 연도까지 누적 대출이자)
연도별 차액 = 2027년 최종 실수령액 - 2026년 최종 실수령액
```

양도세는 기존 양도세 계산 로직(있다면 재사용, 없다면 §4-2 조사 결과에 따라 신규 산식 작성 — 장기보유특별공제·1세대1주택 비과세 판단 포함)을 그대로 가져다 쓴다.

## 6. 페이지 설계 — 현행 vs 개편 검토안 비교 리포트 (1차 착수 확정, 2026-07-20)

§2-1 확인 완료로 착수 조건이 충족됐다. **§5 계산기보다 먼저, 이 리포트를 단독 1차 산출물로 진행**한다(사용자 결정, 2026-07-20). 계산기(§5)와 §7 확장 항목은 이후 이 리포트의 반응을 보고 순서를 정한다.

### 6-1. URL·메타 (CLAUDE.md 리포트 타이틀 공식 적용)

```
슬러그: /reports/real-estate-tax-reform-2026/
타이틀: 2026 부동산 세제개편 검토안 정리 | 종부세·보유세 뭐가 바뀌나
디스크립션: 기획재정부가 검토 중인 종부세 공정시장가액비율 인상, 장기보유특별공제 개편 내용을 정리. 8월 말 정부안 발표 전 검토 단계 기준, 확정 아님.
```

`GOVERNMENT_REVIEW` 단계이므로 타이틀에 "확정", "발표" 등의 표현을 쓰지 않고 "검토안"으로 명시한다.

### 6-2. 상단 정책 상태 배지

```text

```

### 6-3. 비교표 (§2-1-1 출처 기반, 근거 확인된 항목만)

| 항목 | 현행 | 검토 중인 방향 | 영향 대상 | 근거 |
|---|---|---|---|---|
| 종부세 공정시장가액비율 | 60% | 단계적으로 80% 등으로 인상 검토 | 다주택자·비거주 보유자 (실거주 1주택자는 상대적 보호 방향) | 뉴스핌 2026-06-23 |

## 7. 이후 확장 후보 (§4-1 우선순위 3, 5, 6 — 착수 시점 미정)

§4-2 조사로 신규 페이지가 아니라 **기존 계산기 보강**으로 방향이 확정됐다.

- 세 계산기를 한 화면에서 연결하는 **"부동산 세금 계산 허브"** 리포트를 §6 비교표 대신 만드는 방안도 검토 가능 — §2-1 근거가 부족할 경우 이쪽이 더 안전한 대안
