## 3. 이 계산기가 강한 이유

### 사용자 문제

해외여행 준비 시 사용자는 보통 아래를 동시에 고민합니다.

1. 항공권이 얼마쯤 드는지
2. 숙박이 며칠이면 총 얼마인지
3. 식비/교통/관광비까지 포함하면 총액이 얼마인지
4. 내 예산 안에 들어오는지
5. 일본/동남아/유럽 중 어디가 더 현실적인지

### 이 계산기의 장점

* 입력 즉시 총비용이 보임
* 1인당/전체 금액을 동시에 확인 가능
* 여행지 프리셋으로 진입장벽 낮음
* 예산 초과 여부를 바로 시각화 가능
* 환율 변동에 따라 반복 사용 유도 가능

## 11. 계산 로직 설계

## 11-1. 핵심 계산식

### 항공비

`항공비 = 1인 항공권 비용 × 인원수`

### 숙박비

`숙박비 = 1박 숙박비 × 숙박일수 × 객실 수`

객실 수는 기본적으로:

* 1~2명: 1실
* 3~4명: 2실 또는 가족실 선택
* 직접 조정 가능 옵션 제공

### 식비

`식비 = 1인 1일 식비 × 여행일수 × 인원수`

### 현지교통비

`현지교통비 = 1인 1일 교통비 × 여행일수 × 인원수`

### 관광비

`관광비 = 1인 1일 관광비 × 여행일수 × 인원수`

### 보험비

`보험비 = 1인 보험료 × 인원수`

### 쇼핑비

`쇼핑비 = 사용자 입력값`

### 소계

`소계 = 항공비 + 숙박비 + 식비 + 교통비 + 관광비 + 보험비 + 쇼핑비`

### 비상금

`비상금 = 소계 × 비상금 비율`

### 총비용

`총비용 = 소계 + 비상금`

### 1인당 비용

`1인당 비용 = 총비용 ÷ 인원수`

---

## 11-2. 예산 비교 로직

### 계산

`예산차이 = 총예산 - 총비용`

### 상태 구분

* 여유: 총예산 > 총비용 + 10%
* 적정: 총예산이 총비용 ±10% 범위
* 초과: 총예산 < 총비용

### 출력 문구 예시

* “예산 안에서 여행 가능합니다”
* “예산과 거의 비슷합니다. 여유자금 확인이 필요합니다”
* “현재 입력 기준으로 예산을 초과합니다”

## 21. 개발 관점 요구사항

## 21-1. 입력/상태

* 프리셋 선택 시 입력값 자동 반영
* 직접 입력 시 프리셋 해제
* 입력값 변경 시 debounce 후 즉시 재계산

* 환율 API 붙이기 쉬운 구조

## 24. 주의사항

### 반드시 명시할 것

* 계산 결과는 참고용 추정치
* 성수기/비수기, 도시, 예약 시점에 따라 차이 큼
* 환율과 유류할증료, 세금은 변동 가능
* 실제 결제 금액은 예약 시점에 달라질 수 있음

### 피해야 할 것

* “정확한 실제 비용”처럼 단정
* 지역 평균값을 절대값처럼 표시
* 보험/환율/숙박 변동성을 과소표현
* 너무 복잡한 입력으로 초기 이탈 유발
