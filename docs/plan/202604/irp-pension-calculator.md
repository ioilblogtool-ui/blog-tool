## 2. 페이지 목표

### 사용자 목표

- 현재 적립금과 추가 납입을 바탕으로 은퇴 시점 예상 적립금을 빠르게 확인한다.
- 연금 수령과 일시금 수령의 차이를 직관적으로 비교한다.
- 월 수령액, 총 수령액, 세금/수익률 가정 차이를 한 번에 이해한다.

### 서비스 목표

- 연금·은퇴 카테고리의 핵심 유입 계산기로 사용한다.
- `국민연금 계산기`, `연금저축 vs IRP 비교 2026`, `은퇴 생활비 계산기`로 내부 링크 허브 역할을 한다.
- 반복 사용이 가능한 도구형 콘텐츠로 재방문을 유도한다.

## 3. 핵심 메시지

- IRP는 적립금 규모보다 `추가 납입`, `운용 수익률`, `수령 방식`, `수령 기간`이 결과 차이를 크게 만든다.
- 사용자는 복잡한 세법 전체를 이해하기보다, 내 조건에서 월 얼마를 받을 수 있는지 먼저 알고 싶어 한다.
- 이 계산기는 공식 확정 금액이 아니라 `현실적인 비교 추정 도구`여야 한다.

## 4. 주요 UX 시나리오

### 시나리오 A. 직장인 기본 계산

1. 사용자가 현재 나이, 은퇴 예정 나이, 현재 IRP/DC 적립금을 입력한다.
2. 월 추가 납입액과 기대 수익률을 넣는다.
3. 연금 수령 시작 나이와 수령 기간을 선택한다.
4. 결과 화면에서 은퇴 시점 적립금, 월 예상 수령액, 총 수령액을 확인한다.

### 시나리오 B. 일시금 vs 연금 비교

1. 사용자가 동일한 적립 조건으로 수령 방식을 바꿔본다.
2. 연금 수령 시 월 흐름과 총액, 일시금 수령 시 즉시 확보 금액을 비교한다.
3. 차이 해석 카피를 통해 어떤 선택이 더 맞는지 감을 잡는다.

### 시나리오 C. 추가 납입 민감도 확인

1. 월 추가 납입을 10만 원, 30만 원, 50만 원으로 바꾼다.
2. 결과 변화량을 빠르게 비교한다.
3. “매월 얼마 더 넣으면 은퇴 시점 얼마 차이”가 나는지 확인한다.

## 8. 계산 로직 설계

### 8-1. 은퇴 시점 적립금

- 월 수익률 = 연 기대수익률 / 12
- 적립 개월 수 = `(은퇴 예정 나이 - 현재 나이) * 12`
- 미래가치 = 현재 적립금 복리 성장 + 월 추가 납입의 적립 합계

```text
FV = PV * (1 + r)^n + PMT * [((1 + r)^n - 1) / r]
```

### 8-2. 총 납입원금

```text
총 납입원금 = 현재 IRP 적립금 + 현재 DC 적립금 + (월 추가 납입액 * 적립 개월 수)
```

### 8-3. 예상 수익

```text
예상 수익 = 은퇴 시점 적립금 - 총 납입원금
```

### 8-4. 월 예상 수령액

MVP에서는 단순 비교 모델을 사용한다.

```text
월 예상 수령액 = 은퇴 시점 적립금 / (수령 기간 * 12)
```

고급 옵션이 켜지면 연금 수령 중 잔액 운용수익을 반영한 완화형 모델을 추가할 수 있다.

### 8-5. 일시금 비교

- 일시금 수령액: 은퇴 시점 적립금 기준
- 연금 수령 총액: 월 예상 수령액 * 수령 개월 수
- 결과 해석: 유동성은 일시금, 현금흐름은 연금 수령이 강점이라는 문구 제공
