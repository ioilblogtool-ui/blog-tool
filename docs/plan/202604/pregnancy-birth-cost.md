## 2. 기획 배경

이 계산기는 검색 의도가 아주 선명합니다.
사용자는 보통 아래 순서로 궁금해합니다.

1. **검사비 얼마나 드는지**
2. **자연분만 vs 제왕절개 얼마나 차이 나는지**
3. **병원 급이 바뀌면 얼마나 비싸지는지**
4. **국민행복카드/임신출산 진료비 지원으로 얼마나 줄어드는지**
5. **산후조리원까지 합치면 총 얼마인지**

핵심은 단순 총액 계산이 아니라, **“지원금 차감 후 내가 실제로 얼마 내는지”**를 보여주는 것입니다.

다만 기획 시 반드시 주의할 점이 있습니다.
**임신·출산 진료비 지원(국민행복카드)은 의료비 성격의 진료·약제·치료재료에 쓰는 지원금이지, 산후조리원 기본 이용료를 자동 차감하는 구조는 아닙니다.** 임신 1회당 100만 원, 다태아는 140만 원 기본 지급이며, 태아당 100만 원이 되도록 추가 지급됩니다. 분만취약지는 20만 원이 추가됩니다. 지원 범위는 임산부 진료와 2세 미만 영유아 진료·약제 등의 본인부담금입니다. ([보건복지부 대표홈페이지][1])

즉 이 계산기는
**의료비 영역은 바우처 자동 차감**,
**산후조리원은 별도 비용 영역**,
이렇게 분리해야 정확하고 신뢰도가 높습니다. ([보건복지부 대표홈페이지][1])

## 3. 핵심 메시지

### 메인 메시지

**임신·출산 비용은 병원비만이 아니라 검사비, 분만비, 입원비, 조리비까지 합쳐 봐야 하고, 지원금은 ‘의료비에 어디까지 적용되는지’를 분리해서 계산해야 실제 본인부담액이 보인다.** ([보건복지부 대표홈페이지][1])

### 서브 메시지

* 자연분만과 제왕절개는 비용 구조가 다르다.
* 동네산부인과, 종합병원, 상급종합병원은 비용 차이가 크다.
* 다태아는 진료비 지원금 구조도 다르다. ([보건복지부 대표홈페이지][1])
* 국민행복카드는 통합 바우처 카드이지만, 모든 출산 관련 비용을 자동 차감하는 카드는 아니다. ([보건복지부 대표홈페이지][2])
* 산후조리원 비용은 의료비 지원과 별도 계산이 필요하다. ([보건복지부 대표홈페이지][1])

## 4. 데이터 기준 및 원칙

## 4-1. 공식 기준

### 임신·출산 진료비 지원

| 항목        | 기준                                        |
| --------- | ----------------------------------------- |
| 단태아 기본 지원 | 100만 원                                    |
| 다태아 기본 지원 | 140만 원                                    |
| 다태아 추가 지원 | 태아당 100만 원이 되도록 추가 지급                     |
| 분만취약지 추가  | 20만 원                                     |
| 사용 범위     | 임산부 진료, 약제·치료재료 / 2세 미만 영유아 진료·약제 등 본인부담금 |

임신·출산 진료비 지원사업 공식 기준입니다. ([보건복지부 대표홈페이지][1])

### 국민행복카드

국민행복카드는 2026년 기준 23종 국가 바우처를 한 장으로 이용할 수 있는 통합 카드입니다. 즉 카드 자체는 결제 매개이고, 실제 차감 여부는 **어떤 바우처가 어떤 업종·항목에 적용되는지**에 따라 달라집니다. ([보건복지부 대표홈페이지][2])

### 첫만남이용권

첫만남이용권은 출생아당 200만 원 이상 이용권이며, 2024년 1월 1일 이후 출생아부터 첫째 200만 원, 둘째아 이상 300만 원입니다. 다만 이 역시 사용 범위가 있고, 이 계산기의 핵심인 “의료비 차감”과는 별도 로직으로 다뤄야 합니다. ([바우처 사이트][3])

## 4-3. 가장 중요한 계산 원칙

### 반드시 분리할 3개 금액

1. **의료비 총액**

   * 산전검사비
   * 분만비
   * 입원비
   * 약제/처치 예상

2. **의료비 지원 차감 후 본인부담**

   * 임신·출산 진료비 지원 차감

3. **출산 총비용**

   * 의료비 본인부담
   * 산후조리원/산후도우미
   * 기타 준비비(optional)

이 3단 분리가 있어야 사용자가 “지원금 다 받았는데 왜 총비용은 여전히 큰지”를 이해합니다. ([보건복지부 대표홈페이지][1])

## 8-2. 계산식 예시

```ts id="3t7ooj"
medicalTotal =
  prenatalCheckupCost +
  deliveryCost +
  hospitalizationCost +
  epiduralCost +
  optionalCosts;

voucherLimit =
  singleOrMultipleBaseSupport +
  multiFetusAdditionalSupport +
  underservedAreaExtra;

medicalOutOfPocket =
  Math.max(0, medicalTotal - voucherAvailable);

finalTotal =
  medicalOutOfPocket +
  postpartumCareCost;
```
