# Agent Guide

## 목적
이 저장소에서 에이전트가 코드를 생성하거나 구조를 확장할 때 따라야 할 기준을 정리한다.

## 프로젝트 정의
비교계산소는 생활, 정책, 급여, 보상, 커리어 영역에서
숫자를 계산하고 비교해 더 잘 판단하게 만드는 도구형 웹페이지 프로젝트다.

## 현재 구조
- `src/pages/tools/*.astro`: 실제 도구 페이지
- `src/data/tools.ts`: 도구 메타 정보
- `src/components/*`: 공통 UI 컴포넌트
- `src/styles/app.scss` + `src/styles/scss/*`: 공통 스타일 진입점과 페이지별 스타일
- `public/scripts/*.js`: 클라이언트 계산 로직

## 작업 원칙
- 사용자 노출 URL에 `.md`를 포함하지 않는다.
- 새 도구 추가 시 `src/data/tools.ts`에 메타를 먼저 정의한다.
- 새 도구 추가 시 `src/pages/tools/<slug>.astro`를 생성한다.
- 공통 UI는 중복 구현보다 컴포넌트화 우선
- 공통 로직은 가능하면 `public/scripts/tools.js` 또는 전용 스크립트로 정리
- 빌드 검증 없이 마무리하지 않는다.

## 우선순위 기준
- 월 100만 원 목표에 도움이 되는가
- 검색 유입 가능성이 큰가
- 비교 페이지 / 가이드 / 랜딩으로 확장 가능한가
- 급여 / 보상 / 실수령 / 커리어 카테고리와 연결되는가

## UX/UI 원칙
- 모바일 중심 화면을 먼저 설계한다.
- 태블릿과 PC에서도 자연스럽게 확장되는 반응형 구성을 유지한다.
- 입력, 결과, 비교를 한눈에 구분할 수 있어야 한다.
- 계산 결과는 요약 카드 + 상세 비교 + 안내 박스 구조를 기본으로 한다.
- 설명은 짧고 직관적으로, 숫자는 신뢰감 있게 보여준다.
- 첫 화면에서 도구 목적과 핵심 입력이 바로 이해되어야 한다.
