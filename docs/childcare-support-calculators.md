# 출산·육아휴직 지원금 계산기 시리즈 설계/운영 문서

## 1. 문서 목적

이 문서는 현재 저장소에 추가된 출산·육아휴직 관련 계산기들의 기획 의도, 페이지 구조, 계산 기준, 정책 가정, 확장 포인트를 한 번에 정리한 기준 문서다.

이 문서를 남기는 목적은 아래와 같다.

- 추후 정책 변경 시 수정 기준점 확보
- 새 계산기 추가 시 기존 시리즈 톤앤매너 유지
- 페이지별 입력값과 계산 범위 재확인
- 블로그용 설명 글과 계산기 페이지 역할 분리

## 2. 시리즈 개요

시리즈명:

- `출산·육아휴직 지원금 계산기`

메인 메시지:

- `아이 두 돌까지 우리 집이 받을 수 있는 돈과 가구 현금흐름을 쉽고 빠르게 확인한다.`

핵심 차별점:

- 정책 설명보다 `가구 총액`과 `월별 흐름` 중심
- 단일 개인 급여가 아니라 `가구 소득` 관점 강조
- 출산 직후 지원금, 육아휴직 급여, 배우자 급여를 함께 봄
- 일반 육아휴직과 6+6 특례를 비교 가능하게 설계

## 3. 현재 사이트에 맞는 구조

이 저장소는 `블로그 본문에 링크 또는 iframe으로 붙이는 독립형 정적 도구 페이지` 구조다.
따라서 이 시리즈 역시 긴 설명형 글이 아니라 `독립 계산기 4개 + 카드형 진입 구조`로 운영하는 것이 맞다.

현재 시리즈는 아래 4개 도구로 구성한다.

1. `birth-support-total`
2. `single-parental-leave-total`
3. `parental-leave-pay`
4. `six-plus-six`

권장 사용자 흐름은 아래와 같다.

1. `출산~2세 총지원금 계산기`로 기본 지원 총액 확인
2. `한 명만 육아휴직 총수령액 계산기`로 가구 기준 총액 확인
3. `육아휴직 급여 계산기`로 개인 급여 구조 세부 확인
4. `6+6 부모육아휴직제 계산기`로 맞벌이 특례 비교

## 4. 페이지/슬러그 목록

### 4-1. 출산~2세 총지원금 계산기

- 슬러그: `/tools/birth-support-total/`
- 페이지 파일: `src/pages/tools/birth-support-total.astro`
- 스크립트 파일: `public/scripts/birth-support-total.js`

목적:

- 출생부터 18개월 또는 24개월까지 받을 수 있는 기본 지원금을 한 번에 계산

현재 포함 항목:

- 첫만남이용권
- 부모급여
- 아동수당

핵심 출력:

- 선택 구간 총지원금
- 월 평균 체감액
- 월급 몇 개월치인지
- 월급 대비 보전율
- 월별 지원금 타임라인

### 4-2. 한 명만 육아휴직 총수령액 계산기

- 슬러그: `/tools/single-parental-leave-total/`
- 페이지 파일: `src/pages/tools/single-parental-leave-total.astro`
- 스크립트 파일: `public/scripts/single-parental-leave-total.js`

목적:

- 엄마 또는 아빠 한 명만 육아휴직할 때, `육아휴직자 소득 + 배우자 월급 + 지원금`을 합쳐 두 돌까지 가구 총액과 월 평균 가구소득을 계산

이 페이지가 시리즈의 대표 계산기다.

핵심 출력:

- 두 돌까지 가구 총액
- 월 평균 가구소득
- 육아휴직자 소득 합계
- 배우자 급여 합계
- 월별 가구 소득 타임라인

### 4-3. 육아휴직 급여 계산기

- 슬러그: `/tools/parental-leave-pay/`
- 페이지 파일: `src/pages/tools/parental-leave-pay.astro`
- 스크립트 파일: `public/scripts/parental-leave-pay.js`

목적:

- 일반 육아휴직 급여를 월 통상임금 기준으로 계산

핵심 출력:

- 총 육아휴직 급여
- 월 평균 급여
- 월급 대비 보전율
- 월별 급여 테이블

### 4-4. 6+6 부모육아휴직제 계산기

- 슬러그: `/tools/six-plus-six/`
- 페이지 파일: `src/pages/tools/six-plus-six.astro`
- 스크립트 파일: `public/scripts/six-plus-six.js`

목적:

- 부모 모두 육아휴직을 쓸 때 특례 적용 여부와 일반 육아휴직 대비 차액 계산

핵심 출력:

- 6+6 적용 가능 여부
- 가구 총액
- 일반 대비 차액
- 월급 대비 보전율
- 엄마/아빠별 일반 vs 특례 비교 테이블

## 5. tools.ts 메타 운영 원칙

현재 메타는 `src/data/tools.ts`에서 관리한다.

출산·육아휴직 시리즈 관련 메타 분류 원칙:

- category: `support`
- 대표 배지:
  - `birth-support-total`: `추천`
  - `single-parental-leave-total`: `대표`, `추천`
  - `six-plus-six`: `신규`
  - `parental-leave-pay`: 배지 없음 또는 필요 시 `추천`

홈/목록 노출 목적:

- 홈에서는 `single-parental-leave-total`을 대표 카드로 밀기 좋음
- `/tools/` 목록에서는 `출산`, `육아휴직`, `6+6`, `두 돌`, `가구` 검색어 유입 대응

## 6. 정책 기준과 현재 가정

이 시리즈는 정책 안내 페이지가 아니라 `빠른 계산기`다.
따라서 엄밀한 자격요건 전체를 구현하기보다, 핵심 숫자를 단순화해 사용자의 질문에 바로 답하는 방향을 택한다.

### 6-1. 첫만남이용권

현재 반영값:

- 첫째: `2,000,000원`
- 둘째 이상: `3,000,000원`

계산 방식:

- 출생월 1회 지급 가정

### 6-2. 부모급여

현재 반영값:

- 0세 구간: 월 `1,000,000원`
- 1세 구간: 월 `500,000원`

계산 방식:

- 0~11개월: `1,000,000원`
- 12~23개월: `500,000원`

### 6-3. 아동수당

현재 반영값:

- 월 `100,000원`

계산 방식:

- 계산기는 0~24개월 구간만 단순 반영

### 6-4. 일반 육아휴직 급여

현재 반영 로직:

- 1~3개월: 통상임금 100%, 상한 `2,500,000원`
- 4~6개월: 통상임금 100%, 상한 `2,000,000원`
- 7개월 이후: 통상임금 80%, 상한 `1,600,000원`

### 6-5. 육아휴직 18개월 확장

현재 구현 가정:

- 2025년 2월 23일 시행된 육아휴직 기간 확대 흐름을 반영
- 계산기에서는 아래를 `18개월 연장 조건`으로 단순화
  - `SHARED_PARENT`: 부모 모두 3개월 이상 사용
  - `SINGLE_PARENT`: 한부모 가정
  - `DISABLED_CHILD`: 중증장애아동 양육
- `NONE`일 때는 최대 12개월
- 위 조건일 때는 최대 18개월

중요한 구현상 가정:

- 13~18개월 급여는 현재 계산기에서 `7~12개월과 같은 일반 상한 구조를 연장 적용한 추정치`로 계산
- 즉, 정책 해설보다 `비교용 시뮬레이션`에 가깝다

### 6-6. 6+6 부모육아휴직제

현재 반영 로직:

- 자녀 생후 18개월 내 부모 모두 육아휴직 사용 시 적용 가능 가정
- 첫 6개월 특례 상한:
  - 1개월: `2,500,000원`
  - 2개월: `2,500,000원`
  - 3개월: `3,000,000원`
  - 4개월: `3,500,000원`
  - 5개월: `4,000,000원`
  - 6개월: `4,500,000원`
- 7개월 이후는 일반 육아휴직 급여 구조 사용

## 7. 페이지별 입력값 정의

### 7-1. birth-support-total

현재 출력:

- 총 육아휴직 급여

현재 출력:

- 일반 대비 차액

## 8. 핵심 UX 원칙

이 시리즈에서 반드시 유지해야 하는 UX 원칙은 아래와 같다.

- 입력은 첫 화면 기준 5개 안팎에서 시작
- 결과는 큰 카드 4개로 먼저 보여줌
- 표준 시나리오 기본값 제공
- 복잡한 제도 설명은 하단 안내로 보내기
- `월별 흐름`을 항상 보여주기
- `가구 기준 총액`을 우선하기

넣지 말아야 할 것:

- 신청서류 중심 UI
- 예외조건 나열형 첫 화면
- 복잡한 세무/4대보험 계산
- 자격요건 전체 법령 재현

## 9. 현재 구현에서 중요한 해석 포인트

### 9-1. single-parental-leave-total의 의미

이 페이지는 `한 사람이 육아휴직할 때 그 사람 급여만 얼마나 줄어드는가`를 보는 페이지가 아니다.
이 페이지의 핵심 질문은 아래다.

- `배우자 월급까지 합치면 우리 집 통장에 두 돌까지 총 얼마가 들어오나?`
- `월 평균 가구소득이 어느 정도로 유지되나?`

즉, `가구 총액`과 `월 평균 가구소득`이 핵심 KPI다.

### 9-2. birth-support-total의 역할

이 페이지는 육아휴직 여부와 관계없이 `기본적으로 받을 수 있는 출산·영아기 지원금` 총량을 보여주는 역할이다.
즉, 가구 소득 계산의 바닥선을 보여주는 도구다.

### 9-3. parental-leave-pay의 역할

이 페이지는 `일반 육아휴직 급여 자체의 규칙`을 이해시키는 세부 계산기다.
메인 유입 페이지라기보다 보조 설명형 계산기에 가깝다.

### 9-4. six-plus-six의 역할

이 페이지는 맞벌이 사용자에게 `특례를 쓰면 일반 육아휴직보다 얼마나 차이가 나는가`를 보여주는 비교기다.
핵심은 총액보다도 `차액`과 `적용 가능 여부`다.

## 10. 향후 수정 우선순위

### 10-1. 우선 유지해야 할 것

- 슬러그 유지
- 페이지별 역할 분리 유지
- 가구 총액 중심 구조 유지
- 월별 타임라인 유지

### 10-2. 추후 개선 후보

- 6+6과 일반 육아휴직 결과를 카드형 시각화로 강화
- `엄마 육아휴직 vs 아빠 육아휴직` 즉시 비교 토글 추가
- 출산일 기준 실제 월별 지급 시점 보정
- 부모급여/아동수당 제도 변경 시 숫자 테이블 분리
- 법 개정 시 정책 설정값을 별도 TS 상수 파일로 분리

### 10-3. 정책 변경 시 우선 점검 파일

- `public/scripts/birth-support-total.js`
- `public/scripts/single-parental-leave-total.js`
- `public/scripts/parental-leave-pay.js`
- `public/scripts/six-plus-six.js`
- `src/pages/tools/*.astro` 안내 문구
- `src/data/tools.ts` 설명/배지/메타

## 11. 문구 정책

반드시 유지해야 할 문구 방향:

- `단순화한 계산입니다`
- `실제 지급액은 제도 요건, 회사 기준, 신청 시점에 따라 달라질 수 있습니다`
- `18개월 구간은 현재 공개 안내 흐름을 기준으로 단순 반영한 추정입니다`

피해야 할 표현:

- `정부 공식 확정 계산`
- `실제 수령액 보장`
- `예외 없이 동일 적용`

## 12. 최종 정리

현재 출산·육아휴직 계산기 시리즈는 아래 철학으로 운영한다.

- 설명보다 계산
- 개인보다 가구
- 총액보다 흐름도 함께
- 정책 안내보다 체감 금액 중심

그리고 대표 페이지는 여전히 아래다.

- `/tools/single-parental-leave-total/`

이 페이지가 사용자의 가장 현실적인 질문,
`엄마나 아빠 한 명만 육아휴직하면 우리 집 총 얼마 들어오나?`
에 가장 직접적으로 답하기 때문이다.
