# QUALITY_SCORE.md — 품질 루브릭

> 비교계산소의 모든 페이지·코드·콘텐츠에 적용되는 품질 기준입니다.
> PR 생성 전, 배포 전, 코드 리뷰 시 이 루브릭으로 자가 점검합니다.

---

## 1. 빌드 & 배포 (Pass/Fail)

| 항목 | 기준 | 판정 |
|---|---|---|
| 빌드 성공 | `npm run build` 에러 없음 | 필수 Pass |
| 라우트 생성 | `dist/` 내 해당 slug HTML 존재 | 필수 Pass |
| sitemap 등록 | `public/sitemap.xml`에 URL 포함 | 필수 Pass |
| SCSS import | `src/styles/app.scss`에 `@use` 추가 | 필수 Pass |
| tools/reports 등록 | 해당 `.ts` 메타 파일에 slug 추가 | 필수 Pass |

## 2. 콘텐츠 품질 (5점 척도)

### 2-1. 데이터 신뢰성

| 점수 | 기준 |
|---|---|
| 5 | 모든 수치에 출처 배지(`공식`/`시뮬레이션`/`참고`) 표시, 가정 명시 |
| 4 | 대부분 배지 있음, 1~2개 누락 |
| 3 | 주요 수치 배지 있음, 설명형 수치 누락 |
| 2 | 절반 이하 배지, 공식·추정 혼재 불명확 |
| 1 | 배지 없음, 출처 불명 |

**최소 기준: 4점 이상**

### 2-2. 한국어 품질

| 점수 | 기준 |
|---|---|
| 5 | 자연스러운 구어체 한국어, 오탈자 없음, 영어 혼용 최소화 |
| 4 | 약간의 어색한 표현, 오탈자 1~2개 |
| 3 | 기계번역 느낌 일부, 오탈자 3~5개 |
| 2 | 전체적으로 어색함, 오탈자 다수 |
| 1 | 영어 문장 포함, 읽기 어려움 |

**최소 기준: 4점 이상**

### 2-3. 단정 표현 제한

| 점수 | 기준 |
|---|---|
| 5 | 추정값에 "~입니다" 단정 없음, "~로 추정됩니다" 등 완화 표현 일관 |
| 4 | 대부분 완화 표현, 1~2개 단정 |
| 3 | 절반 단정, 절반 완화 |
| 2 | 대부분 단정 표현 |
| 1 | 시뮬레이션을 공식 데이터처럼 제시 |

**최소 기준: 4점 이상**
