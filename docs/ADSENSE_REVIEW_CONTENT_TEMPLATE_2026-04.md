# 애드센스 재검토용 계산기 콘텐츠 보강 템플릿 (2026-04)

기준일: 2026-04-29  
대상: `src/pages/tools/*.astro` 계산기 상세 페이지

## 목적
- `가치가 별로 없는 콘텐츠` 판정을 피하기 위해 계산기 페이지를 "도구 + 해설 + 의사결정 가이드" 구조로 통일한다.
- 작성자마다 품질 편차가 생기지 않도록 공통 섹션과 최소 기준을 고정한다.

## 페이지 공통 구조
아래 섹션 순서를 기본으로 사용한다.

1. 문제 정의 (도입)
2. 계산 기준과 가정
3. 입력값 설명
4. 결과 해석 가이드
5. 실제 사용 예시 2개 이상
6. 주의사항/면책 (추정, 시뮬레이션 명시)
7. FAQ 3개 이상
8. 관련 계산기/리포트 내부 링크 3개 이상
9. 업데이트 기준일 및 출처

## 섹션별 템플릿 문안

### 1) 문제 정의 (도입)
```md
## 이 계산기는 언제 필요한가요?
이 계산기는 [상황]에서 [비용/수익/기간]을 빠르게 비교해 의사결정을 돕기 위한 도구입니다.
단순 결과값만 보여주지 않고, 결과를 어떻게 해석하면 좋은지까지 함께 안내합니다.
```

### 2) 계산 기준과 가정
```md
## 계산 기준과 가정
- 기준일: YYYY-MM-DD
- 사용 데이터: [공식 통계/공공기관/내부 가정]
- 핵심 가정: [예: 수익률, 물가상승률, 세율]
- 계산식 요약: [핵심 수식 1~2줄]
```

### 3) 입력값 설명
```md
## 입력값은 이렇게 넣으세요
- [입력값 A]: [무엇을 의미하는지]
- [입력값 B]: [권장 범위와 주의점]
- [입력값 C]: [모를 때 보수적으로 넣는 방법]
```

### 4) 결과 해석 가이드
```md
## 결과는 이렇게 읽으세요
핵심은 총액보다 [비교 기준]의 차이를 보는 것입니다.
[시나리오 A]에서는 [해석], [시나리오 B]에서는 [해석]이 중요합니다.
```

### 5) 실제 사용 예시
```md
## 예시 1. [유형]
- 입력: [...]
- 결과: [...]
- 해석: [...]

## 예시 2. [유형]
- 입력: [...]
- 결과: [...]
- 해석: [...]
```

### 6) 주의사항/면책
```md
## 계산 전에 꼭 확인하세요
- 본 결과는 [추정/시뮬레이션]이며 최종 의사결정 전 공식 자료 확인이 필요합니다.
- 실제 금액은 개인 조건, 시점, 제도 변경에 따라 달라질 수 있습니다.
```

## 최소 품질 기준 (재검토 전)
- 본문(설명 텍스트) 1,200자 이상
- "결과 해석" 섹션 필수
- 예시 2개 이상
- FAQ 3개 이상
- 내부 링크 3개 이상
- 기준일/출처 명시
- 추정/시뮬레이션 문구 명시

## Astro 구현 체크포인트
- `SeoContent` 섹션 누락 여부 확인
- `InfoNotice`에 추정/면책 문구 포함 여부 확인
- `ToolActionBar` 또는 본문 하단에 내부 링크 배치
- 페이지 타이틀/설명에 계산 목적이 드러나는지 확인
