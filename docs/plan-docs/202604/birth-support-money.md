## 2. 공식 기준 메모

2026년 기준 계산기에서 우선 반영해야 할 국가 공통 항목은 아래 3개입니다.

| 지원 항목     | 2026 기준 핵심                                                                                                     |
| --------- | -------------------------------------------------------------------------------------------------------------- |
| 부모급여      | 2세 미만 아동 대상, 0세 월 100만 원, 1세 월 50만 원 지급. 보건복지부 페이지 최종수정일은 2026년 3월 25일입니다. ([보건복지부 대표홈페이지][1])                 |
| 아동수당      | 기존 8세 미만 월 10만 원에서, 2026년 9세 미만으로 확대되고 이후 매년 1세씩 확대 예정입니다. 비수도권·인구감소지역 추가 지급 구조도 반영 필요합니다. ([보건복지부 대표홈페이지][2]) |
| 지자체 출산지원금 | 임신육아종합포털 아이사랑에 2026년 지자체 출산지원금 업데이트가 2026년 3월 17일 완료된 것으로 공지되어 있으며, 지역별 게시글 형태로 제공됩니다. ([어린이집 정보 포털][3])       |

정부24는 정부 서비스·혜택을 통합 조회할 수 있는 포털이고, 서울시는 “탄생육아 몽땅정보통”에서 서울형 출산·육아 지원사업을 통합 안내합니다. 계산기 하단의 “공식 확인 링크” 영역에 연결하면 신뢰도를 높일 수 있습니다. ([정부24][4])

## 7. 계산 로직

### 7.1 기본 계산식

```text
총수령액 =
첫만남이용권
+ 부모급여 합계
+ 아동수당 합계
+ 지자체 출산장려금
+ 지역별 추가 바우처
+ 다자녀/다태아 추가 지원금
```

### 7.2 부모급여 계산

```text
0~11개월: 월 1,000,000원
12~23개월: 월 500,000원

부모급여 합계 =
0세 해당 개월 수 × 1,000,000
+ 1세 해당 개월 수 × 500,000
```

보건복지부 기준상 부모급여는 2세 미만 아동에게 지급되며, 0세 월 100만 원, 1세 월 50만 원 구조입니다. ([보건복지부 대표홈페이지][1])

### 7.3 아동수당 계산

```text
아동수당 합계 =
지급 대상 개월 수 × 월 지급액
```

2026년에는 아동수당 지급 연령이 9세 미만으로 확대되고, 비수도권·인구감소지역 추가 지급 구조가 있으므로 지역 테이블과 연동해야 합니다. ([보건복지부 대표홈페이지][2])

### 7.4 지자체 지원금 계산

```text
지자체 지원금 =
지역코드 + 출생순위 + 다태아여부 + 거주요건 + 신청기한 조건으로 조회
```

아이사랑 포털은 2026년 지자체 출산지원금 업데이트 완료 공지를 제공하고, 지역별 출산지원금 게시글을 운영하고 있으므로 초기 데이터 소스로 활용하기 적합합니다. ([어린이집 정보 포털][3])

## 14. 개발 메모

### 14.1 계산기 상태 구조 예시

```ts
type BirthSupportInput = {
  birthDate: string;
  sido: string;
  sigungu: string;
  birthOrder: 1 | 2 | 3;
  isMultipleBirth: boolean;
  childcareType: 'home' | 'daycare';
  calculationMonths: 12 | 24 | 95;
};

type BenefitItem = {
  name: string;
  type: 'national' | 'local';
  paymentType: 'cash' | 'voucher' | 'localCurrency';
  amount: number;
  month: number;
  sourceUrl?: string;
};

type BirthSupportResult = {
  totalAmount: number;
  firstMonthAmount: number;
  nationalTotal: number;
  localTotal: number;
  monthlyTimeline: {
    month: number;
    ageLabel: string;
    items: BenefitItem[];
    monthlyTotal: number;
  }[];
};
```

### 14.2 계산 함수 예시

```ts
function calculateParentBenefit(month: number): number {
  if (month >= 0 && month <= 11) return 1_000_000;
  if (month >= 12 && month <= 23) return 500_000;
  return 0;
}

function calculateChildAllowance(month: number, monthlyAmount = 100_000): number {
  return month >= 0 ? monthlyAmount : 0;
}

function calculateBirthSupportTotal(input: BirthSupportInput): BirthSupportResult {
  const timeline = [];

  for (let month = 0; month < input.calculationMonths; month++) {
    const parentBenefit = calculateParentBenefit(month);
    const childAllowance = calculateChildAllowance(month);

    const firstMeetingVoucher =
      month === 0
        ? input.birthOrder >= 2
          ? 3_000_000
          : 2_000_000
        : 0;

    const monthlyTotal =
      parentBenefit +
      childAllowance +
      firstMeetingVoucher;

    timeline.push({
      month,
      ageLabel: `${month}개월`,
      items: [
        { name: '첫만남이용권', type: 'national', paymentType: 'voucher', amount: firstMeetingVoucher, month },
        { name: '부모급여', type: 'national', paymentType: 'cash', amount: parentBenefit, month },
        { name: '아동수당', type: 'national', paymentType: 'cash', amount: childAllowance, month },
      ].filter(item => item.amount > 0),
      monthlyTotal,
    });
  }

  const totalAmount = timeline.reduce((sum, row) => sum + row.monthlyTotal, 0);

  return {
    totalAmount,
    firstMonthAmount: timeline[0]?.monthlyTotal ?? 0,
    nationalTotal: totalAmount,
    localTotal: 0,
    monthlyTimeline: timeline,
  };
}
```

※ 첫만남이용권 금액과 지자체 금액은 정책 데이터 테이블에서 관리하는 방식이 안전합니다. 정책 금액이 바뀔 수 있으므로 UI 문구에는 `최종 확인일`을 반드시 표시하는 구조를 추천합니다.

이 콘텐츠는 단순 리포트보다 **계산기 + 지역별 리포트 허브**로 키우는 게 좋습니다.

| ------------ | ----------------------------- |
