# exec-plans/ — 실행 계획

> 스프린트·마일스톤·우선순위 실행 계획을 관리합니다.
> 작업 단위로 파일을 생성하고, 완료 후 `done/` 하위로 이동합니다.

---

## 폴더 구조

```
docs/exec-plans/
├── README.md          ← 지금 이 파일
├── CURRENT.md         ← 현재 진행 중인 실행 계획
└── done/              ← 완료된 실행 계획 아카이브
```

---

## 실행 계획 작성 형식

```md
# [YYYY-MM-DD] <작업명> 실행 계획

## 목표
한 문장 목표

## 범위 (In Scope)
- 할 것 1
- 할 것 2

## 범위 외 (Out of Scope)
- 안 할 것 1

## 작업 순서
1. [ ] 작업 A (예상: 30분)
2. [ ] 작업 B (예상: 1시간)
3. [ ] 작업 C (예상: 20분)

## 완료 기준
- [ ] npm run build 성공
- [ ] 브라우저에서 동작 확인
- [ ] QUALITY_SCORE.md 최소 기준 충족

## 메모
```
