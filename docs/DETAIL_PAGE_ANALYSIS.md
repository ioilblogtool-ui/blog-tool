# 도구 상세 페이지 구성 분석

## 공통 판단
비교계산소의 상세 페이지는 단순 계산기 화면이 아니라 검색 유입을 받는 랜딩 페이지 역할까지 수행해야 한다.
따라서 모든 도구 페이지는 아래 4개 기능을 같이 가져야 한다.
- 계산을 바로 시작할 수 있는 입력 영역
- 핵심 결과를 빠르게 이해하게 해주는 요약 영역
- 계산 기준과 FAQ를 통해 신뢰를 높이는 설명 영역
- 다음 계산기로 이동하게 만드는 내부 링크 영역

## 페이지별 현재 역할
### salary
- 검색 의도: 연봉 인상, 실수령, 협상 체감
- 사용자의 핵심 질문: 인상액이 실제 월 체감에 얼마나 반영되는가
- 상세 페이지 포인트: 비교 케이스를 먼저 보여주고 협상용 판단을 돕는 문구가 중요함

### retirement
- 검색 의도: 퇴직금, 세전/세후, 평균임금
- 사용자의 핵심 질문: 퇴직 시점에 실제로 얼마 정도 손에 쥐는가
- 상세 페이지 포인트: 평균임금 기준과 세후 추정의 한계를 명확히 보여줘야 신뢰도가 올라감

### negotiation
- 검색 의도: 이직 연봉 비교, 목표 연봉, 체감 연봉
- 사용자의 핵심 질문: 목표 오퍼가 실제로 얼마나 나은가
- 상세 페이지 포인트: 연봉 증가액보다 월 체감 변화와 인상률을 같이 강조해야 머무름이 좋아짐

### parental-leave
- 검색 의도: 육아휴직 중 소득 공백, 복직 후 회복, 생활비 버퍼
- 사용자의 핵심 질문: 육아휴직 동안 얼마가 부족하고 얼마를 준비해야 하는가
- 상세 페이지 포인트: 버퍼 관점과 가구 총액 계산기 연결이 중요함

### parental-leave-pay
- 검색 의도: 육아휴직 급여, 상한, 18개월 확장
- 사용자의 핵심 질문: 월별로 실제 얼마를 받는가
- 상세 페이지 포인트: 월별 표와 상한 적용 개월 수를 빨리 이해하게 해야 함

### single-parental-leave-total
- 검색 의도: 한 명만 육아휴직, 가구 총액, 배우자 월급 포함
- 사용자의 핵심 질문: 두 돌까지 가구 전체 현금흐름이 어떻게 되는가
- 상세 페이지 포인트: 상단 요약과 하단 타임라인의 연결이 핵심

### birth-support-total
- 검색 의도: 출산 지원금 총액, 부모급여, 아동수당
- 사용자의 핵심 질문: 아이 출생부터 얼마를 받을 수 있는가
- 상세 페이지 포인트: 지원금 분해와 월별 흐름을 함께 보여줘야 검색 유입 후 체류가 늘어남

### six-plus-six
- 검색 의도: 6+6 특례, 일반 육아휴직 대비 차액
- 사용자의 핵심 질문: 두 사람이 같이 쓰면 얼마나 더 유리한가
- 상세 페이지 포인트: 특례 가능 여부와 차액을 가장 먼저 보여줘야 함

### bonus-simulator
- 검색 의도: 삼성전자 성과급, SK하이닉스 성과급, 현대차 성과급, 총보상 비교
- 사용자의 핵심 질문: 어느 회사가 더 유리하고, 내 직급 기준 총보상은 어느 정도인가
- 상세 페이지 포인트: 결과 대시보드와 비교 흐름이 핵심이며, 기준 메모는 접어서 신뢰를 보완하는 역할이 적절함

## 공통 개선 원칙
- 모든 상세 페이지는 계산기 하단에 `핵심 안내 -> 바로 확인할 것 -> 계산 기준 -> FAQ -> 관련 계산기` 흐름을 둔다.
- FAQ 첫 항목은 기본 펼침으로 두어 검색 유입 사용자가 바로 답을 찾게 한다.
- 관련 계산기는 단순 링크가 아니라 다음 행동 CTA로 보이게 구성한다.
- 설명형 섹션은 모바일에서도 스캔 가능하도록 짧은 카드와 리스트를 우선 사용한다.
- 검색 유입 후 체류를 늘리기 위해 내부 링크는 최소 3개 이상 유지한다.
