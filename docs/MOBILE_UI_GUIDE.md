# Mobile First UI Guide

## 목적
비교계산소의 모든 페이지를 모바일 중심으로 설계하고, 같은 규칙으로 태블릿과 PC까지 자연스럽게 확장하기 위한 공통 UI 기준 문서.

## 현재 화면에서 보이는 핵심 이슈

### 1. 모바일에서 여백과 카드 크기가 전반적으로 큼
- `global.css` 기준으로 `panel`, `tool-card`, `notice-box`의 기본 패딩이 24px로 고정되어 있음
- `radius-xl` 30px, `radius-lg` 24px가 모바일에서는 과하게 둥글고 공간을 많이 차지함
- 첫 화면에서 입력 카드와 결과 카드가 함께 보이지 않고 스크롤이 길어짐

### 2. 2열 기준 설계가 모바일 전환 시점에서 다소 늦음
- `calculator-grid`는 데스크톱 기준 `minmax(300px, 360px) minmax(0, 1fr)` 구조
- `960px 이하`에서만 1열로 내려가서 태블릿 세로, 작은 노트북, 대형 모바일에서 중간 단계 최적화가 부족함
- `tool-showcase`, `metrics`, `home-spotlight`, `scenario-grid`도 비슷하게 한 번에 크게 바뀜

### 3. 첫 화면에서 핵심 행동이 분산됨
- 상세 도구 페이지에서 입력 카드, 액션 바, 요약 카드가 페이지마다 배치 밀도가 다름
- 모바일에서는 사용자가 먼저 봐야 할 순서가 `도구 소개 -> 핵심 입력 -> 계산 버튼 -> 요약 결과`여야 하는데 일부 페이지는 정보 블록이 많아 바로 계산까지 도달하는 속도가 떨어짐

### 4. 표 중심 결과가 모바일에서 길어짐
- `result-table`은 가로 스크롤을 허용하지만, 핵심 비교 결과까지 테이블에 의존하는 페이지가 있음
- 모바일에서는 표 전체보다 핵심 결과 카드와 1차 요약 리스트가 먼저 보여야 함

### 5. 컴포넌트별 시각 밀도가 일정하지 않음
- 홈 카드, 도구 목록 카드, 상세 페이지 패널, 안내 박스의 텍스트 크기와 간격 리듬이 조금씩 다름
- 일부 페이지는 `panel-heading`, `breakdown-chip`, `scenario-card` 등 추가 패턴이 많아 통일감이 약해짐

## 비교계산소 반응형 설계 원칙

### 원칙 1. 모바일 퍼스트로 먼저 설계
- 기본 스타일은 모바일 기준으로 작성
- 태블릿과 PC는 `min-width` 기준으로 점진 확장
- 데스크톱 레이아웃을 모바일에서 접는 방식보다 모바일에서 필요한 정보만 먼저 배치

### 원칙 2. 첫 화면에서 바로 목적이 보여야 함
도구 상세 페이지의 첫 화면 기본 순서:
1. 제목과 한 줄 설명
2. 핵심 입력 2~4개
3. 계산 버튼
4. 핵심 결과 2~4개
5. 상세 비교와 설명

### 원칙 3. 표보다 요약을 먼저 보여줌
- 모바일에서는 표보다 요약 카드와 한 줄 해석이 먼저
- 표는 두 번째 단계 정보로 배치
- 가로 스크롤이 필요한 표는 유지하되, 모바일 핵심 결과를 별도 카드로 중복 제공

### 원칙 4. 카드 시스템을 통일
- 입력 카드, 결과 카드, 안내 카드의 패딩/보더/라운드 규칙을 통일
- 카드 간 간격은 12px, 16px, 20px 중 하나만 사용
- 시각적 강조는 색보다 구조로 우선

### 원칙 5. 반응형은 3단계로 설계
- 모바일: 0~767px
- 태블릿: 768~1023px
- PC: 1024px 이상

## 권장 레이아웃 기준

### 공통 컨테이너
- 모바일: 좌우 16px
- 태블릿: 좌우 24px
- PC: 최대 1120~1160px

### 공통 섹션 간격
- 모바일: 16~20px
- 태블릿: 20~24px
- PC: 24~32px

### 공통 카드 패딩
- 모바일: 16~18px
- 태블릿: 20px
- PC: 24px

### 공통 라운드
- 모바일: 16~20px
- 태블릿: 20~24px
- PC: 24~30px

## 페이지 유형별 설계 가이드

### 1. 홈
모바일 우선 구조:
1. 헤드라인
2. 서브카피
3. 대표 CTA 1개
4. 추천 도구 1개
5. 도구 카드 목록

개선 포인트:
- `home-spotlight`는 모바일에서 추천 도구가 너무 길어지지 않게 요약형 유지
- 통계 카드 2개는 모바일에서 한 줄씩 충분히 읽히게 간결화
- 카드 설명은 2~3줄 이내로 제한

### 2. 도구 목록
모바일 우선 구조:
1. 페이지 소개
2. 검색창
3. 필터 칩
4. 결과 수
5. 카드 목록

개선 포인트:
- 필터 버튼 높이와 검색창 높이를 맞춤
- 카드 메타는 모바일에서 핵심 1개만 남기고 축약 가능
- 카드 안 CTA 의미를 더 명확히 유지

### 3. 도구 상세
모바일 우선 구조:
1. 페이지 제목
2. 핵심 입력 카드
3. 계산 버튼
4. 요약 카드
5. 상세 결과
6. 안내 박스
7. 설명형 SEO 콘텐츠

개선 포인트:
- `calculator-grid`를 모바일 기준 1열 기본으로 두고 PC에서만 2열 확장
- `SummaryCards`는 모바일 1열 또는 2열 우선, PC에서만 3~4열
- `ToolActionBar`는 모바일에서 버튼 2개가 같은 폭으로 정렬되게 설계
- `InfoNotice`는 항상 결과 아래 또는 입력 아래의 같은 위치 패턴 유지

## 공통 컴포넌트 설계 기준

### CalculatorHero
- 모바일 제목 최대 2~3줄
- 설명 문구는 2줄 내외
- 상단 여백 과다 금지

### SummaryCards
- 모바일: 1열 또는 2열
- 태블릿: 2열
- PC: 3열 또는 4열
- 숫자 크기는 모바일에서 과도하게 크지 않게 제한

### ToolActionBar
- 모바일: 버튼이 줄바꿈되더라도 폭과 높이가 통일
- 태블릿 이상: 가로 정렬
- `입력값 초기화`, `링크 복사` 외 액션이 늘어나도 같은 패턴 유지

### InfoNotice
- 모바일에서도 제목과 리스트가 너무 길어 보이지 않게 문장 길이 관리
- 리스트 간격을 일정하게 유지

## CSS 리팩터링 방향

### 1. 브레이크포인트를 모바일 퍼스트로 재정리
권장 순서:
- 기본값: 모바일
- `@media (min-width: 768px)` 태블릿 확장
- `@media (min-width: 1024px)` PC 확장

### 2. 공통 spacing token 정리
예시:
- `--space-2: 8px`
- `--space-3: 12px`
- `--space-4: 16px`
- `--space-5: 20px`
- `--space-6: 24px`

### 3. 공통 radius token 축소
예시:
- 모바일 기본: 18px
- 큰 카드: 22px
- 버튼/입력: 12px

### 4. 카드 컴포넌트 계층 단순화
현재 `panel`, `tool-card`, `notice-box`, `scenario-card`, `breakdown-chip`, `metric-card`가 조금씩 다르게 생겼음

정리 방향:
- Base Card
- Soft Card
- Accent Card
- Data Card

## 우선 수정 추천 순서
1. `global.css`의 여백, 라운드, 브레이크포인트 재정리
2. `calculator-grid`, `metrics`, `tool-showcase`, `home-spotlight`를 모바일 퍼스트 구조로 재설계
3. `SummaryCards`, `ToolActionBar`, `CalculatorHero`의 공통 반응형 규칙 고정
4. `salary`, `bonus-simulator` 같은 대표 페이지 2개 먼저 맞춤
5. 나머지 도구 페이지를 같은 패턴으로 통일

## 페이지 점검 체크리스트
- [ ] 모바일 첫 화면에서 제목과 핵심 입력이 바로 보이는가
- [ ] 모바일에서 계산 버튼이 너무 아래로 밀리지 않는가
- [ ] 요약 카드가 모바일에서 읽기 쉬운가
- [ ] 표 없이도 핵심 결과 이해가 가능한가
- [ ] 태블릿에서 2열 전환이 자연스러운가
- [ ] PC에서 좌우 여백이 과하게 넓거나 비지 않는가
- [ ] 홈, 목록, 상세 페이지 카드 스타일이 같은 제품처럼 보이는가
