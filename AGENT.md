# AGENT.md

## 프로젝트 정체성
비교계산소는 **숫자를 비교해서 더 잘 판단하게 만드는 계산 플랫폼**이다.
단순 정보 제공 사이트가 아니라, 사용자가 직접 입력하고 비교하고 판단할 수 있는 도구형 웹페이지를 만든다.

## 최상위 목표
- 장기 목표: 월 100만 원 수익 달성
- 기준 문서: `docs/MONTHLY_1M_GOAL.md`
- SEO / 광고 준비 기준 문서: `docs/SEO_ADSENSE_ROADMAP.md`

## 이 저장소에서 코드를 만들 때 우선하는 것
1. 계산기 몇 개 추가가 아니라 검색형 계산 콘텐츠 플랫폼으로 확장 가능한가
2. 사용자가 바로 입력하고 비교할 수 있는가
3. 모바일 중심으로 읽기 쉽고, 태블릿과 PC까지 자연스럽게 확장되는가
4. 추후 가이드, 비교 페이지, 랜딩 페이지로 확장 가능한 주제인가

## 현재 구조
- `src/pages/tools/*.astro`: 실제 도구 페이지
- `src/data/tools.ts`: 도구 메타 정보
- `src/components/*`: 공통 컴포넌트
- `src/styles/app.scss` + `src/styles/scss/*`: 공통 스타일 진입점과 페이지별 스타일
- `public/scripts/*.js`: 도구별 동작 스크립트
- `docs/*`: 기획 / 운영 / 목표 문서

## 작업 원칙
- 사용자 노출 URL에는 `.md`를 쓰지 않는다.
- 메타 관리는 코드 설정 파일 `src/data/tools.ts`에서 한다.
- 새 도구를 추가할 때는 `src/pages/tools/<slug>.astro`와 `src/data/tools.ts`를 함께 수정한다.
- 공통 동작이 겹치면 컴포넌트 / 공통 스크립트로 먼저 흡수한다.
- 새 페이지는 항상 `npm run build`까지 확인한다.

## 제품 방향
우선순위 카테고리:
1. 급여 / 연봉 실수령 / 성과급 / 총보상
2. 대기업 초봉 / 직급별 연봉 / 회사 비교
3. 출산 / 육아휴직 / 부모급여

## UX 원칙
- 모바일 중심 화면을 기본으로 설계한다.
- 태블릿과 PC에서도 정보 밀도가 어색하지 않게 반응형으로 확장한다.
- 복잡한 설명보다 바로 써보는 느낌을 우선한다.
- 가볍지만 신뢰감 있게 보여준다.
- 입력 > 요약 > 상세 비교 구조를 유지한다.
- 숫자만 보여주지 말고 판단에 도움이 되는 비교를 제공한다.
- 첫 화면에서 도구 목적과 핵심 결과가 바로 보이게 만든다.
